## 1. Motivation

v0.1.0 shipped with a structural grounding gap acknowledged in `grounding-v0.1.0.md`:

- 10 primary sources are **all Anglo-Western** (Cialdini / Toulmin / Burke / Bhatia / Norman et al.)
- Plugin self-positions as **multi-lingual** (3-language README + glossary + JP business anchor BCG / 山口周)
- This is a **claim/method mismatch**: plugin announces multi-language relevance but methods only operate in one cultural register

User raised the question (2026-05-05): "should the lenses differentiate by language, e.g. rhetoric?" The answer is yes — and the audit revealed 4 of 9 lenses are **culturally sensitive** at high or medium level (per author's own self-audit table).

v0.2.0 closes the gap by shipping **language-specific lens variants**.

### 2.2 What v0.2.0 does NOT ship

- Lens variants for `lens-semiotic` and `lens-ux` — lower cultural sensitivity per audit; deferred to v0.3
- Lens variants for `argument-deconstruct` (lens-toulmin / lens-burke-pentad) — these are deeper splits of `artifact-deconstruct/lens-rhetoric`; v0.2.0 focuses on artifact-deconstruct's lens library
- Translation of skill SKILL.md / protocol files into JP/ZH — content stays English; only lens references get language variants
- New skills (`product-deconstruct` / `pricing-decode` from v0.1.0 roadmap) — pushed to v0.3.0

### 2.3 Hard exclusions

- ❌ AI-generated translations of Anglo lens content into JP/ZH — that's cosplay rigor, not real grounding
- ❌ Per-language fixtures that are translations of the existing fixtures — variants need fixtures from native JP/ZH artifacts
- ❌ Forcing JP/ZH artifacts to use Anglo methods only — defeats the purpose

| **v0.2.0** | product-deconstruct + pricing-decode | **cultural variants** (this proposal) |

## 4. 4 High-sensitivity Lenses — Variant Plan

### 4.1 lens-rhetoric (Toulmin + Burke)

**Cultural sensitivity**: 🔴 high

| Variant | Anchor | Content focus |
|---|---|---|
| `lens-rhetoric.md` (universal core) | meta-only | 1-page summary; routes to variant; documents that rhetoric is culturally bound |
| `lens-rhetoric-anglo.md` | Toulmin (1958) + Burke (1945) | Existing content, renamed. Universal claim originated in English |
| `lens-rhetoric-ja.md` | Hinds (1983/1987) + Oh (2025) | Kishōtenketsu 起承転結; reader-responsible language; absence of explicit claim is virtue; "ten" (turn) functions as argument unit; modern JP academic uses Western 序論・本論・結論 — **dual mode** |
| `lens-rhetoric-zh.md` | 劉勰《文心雕龍》六觀 | 位體 / 置辭 / 通變 / 奇正 / 事義 / 宮商 framework; ZH rhetoric tradition's relational orientation; emphasis on 通變 (continuity-and-change) over warrant/grounds; modern ZH academic uses 緒論-本論-結論 hybrid |

### 4.2 lens-persuasion (Cialdini + Brignull)

**Cultural sensitivity**: 🔴 high

| Variant | Anchor | Content focus |
|---|---|---|
| `lens-persuasion.md` (universal core) | meta-only | Same as 4.1 — routes to variant |
| `lens-persuasion-anglo.md` | Cialdini (2021) + Brignull (2024) | Existing content, renamed. WEIRD-sample empirical base |
| `lens-persuasion-ja.md` | Cialdini + cross-cultural research (Murayama 2017 / Hofstede / 加藤恭子) | High power-distance → authority weighting up; collectivism → social proof + unity weighting up; **建前/本音** affects how Cialdini principles get expressed; 婉曲表現 indirect persuasion as virtue |
| `lens-persuasion-zh.md` | Cialdini + ZH research (林安梧 / Hofstede CN) | High power-distance similar to JP; **關係 (guanxi)** as Cialdini #5 (liking) extension; **面子 (face)** affects #4 (authority) and #1 (reciprocity); collectivist social proof |

**Cultural sensitivity**: 🟠 medium

| Variant | Anchor | Content focus |
|---|---|---|

## 7. Router Intelligence

### 7.1 Language detection in `using-deconstruct-toolkit`

Update `using-deconstruct-toolkit/SKILL.md` to add a new step:

> **Step N: Detect artifact language and cultural register**
>
> Before dispatching to a sibling skill, determine:
> 1. What language is the artifact's primary text? (English / Japanese / Chinese-Traditional / Chinese-Simplified / Korean / mixed)
> 2. What cultural register does the artifact target? (academic / business / literary / political / consumer-marketing)
> 3. Is the artifact a translation (e.g. JP-original translated to English)?
>
> Pass these signals to the dispatched skill so it can select the appropriate lens variant.

### 7.2 New protocol `lens-variant-selection.md`

In `artifact-deconstruct/protocols/`, add a new protocol that:

1. Detects artifact language at sentence/word level
2. Detects cultural register
3. Selects: `-anglo` / `-ja` / `-zh` / universal-core based on signals
4. Documents fallback rules:
   - Default → `-anglo` if uncertain
   - Multilingual artifact → apply 2 variants and compare
   - Korean / Vietnamese / Thai → use universal core + flag low cultural-coverage

### 7.3 SKILL.md updates

`artifact-deconstruct/SKILL.md` Step 4 ("Apply selected lenses") gets updated:

> Before reading lens references, **resolve the language variant** per
> [`protocols/lens-variant-selection.md`](protocols/lens-variant-selection.md).
> The universal-core lens-*.md files contain meta-only content; the
> primary analysis happens in the per-language variant.

- **Q5 + Q6 combined**: universal-core lens files are pure meta-routers. If invoked without language signal, the lens prompts the user or applies multiple variants in parallel. **No `-anglo` default fallback** (Q6 obviates this).
- **Q1 → CHANGELOG + Roadmap**: roadmap shifted v0.2 → v0.3 → v0.4 for product/atomic skills; CHANGELOG v0.2.0 leads with "cultural variants foundational grounding fix."
