# Brief — Taiwan financial-sector iXBRL ingestion (-fh / -basi / -bd / -ins)

Branch: `feat-tw-ixbrl-fh` · base clean at origin/main `b6b2a4d4` · arc follows the
`-ci` industrial arc shipped in PR #592.

Status: brainstorm COMPLETE — scope B ratified, measurement CONVERGED (23 filers, 5
taxonomies confirmed complete, no 6th found). Awaiting final sign-off before `writing-plans`.

**Scope evolved during brainstorm via 3 live-measurement passes (21 filers total).**
The financial universe is FIVE taxonomies, not the two the arc name assumed:
`-fh` (FHC), `-basi` (standalone commercial bank — discovered round 2), `-bd`
(broker), `-ins` (insurer — itself splits into life / P&C / reinsurance sub-shapes),
alongside the shipped `-ci` (industrial). Key consequence: **a dispatcher cannot
branch on `_is_fh_taxonomy` alone** — it returns False for `-basi`/`-bd`/`-ins`
alike, so standalone banks would be wrongly lumped with brokers/insurers. A 5-way
taxonomy classifier is required.

## Problem

The user is a long-horizon equity researcher writing investment memos for himself.
When he researches a Taiwan financial-holding company (Cathay 2882, Fubon 2881,
CTBC 2891, Mega 2886, etc.), the investing-toolkit memo pipeline currently gives him
nothing usable: the canonical builder fail-closes on the `-fh` taxonomy
(`twse_ixbrl_canonical.py:~238` → `{"unsupported": "financial-fh"}`) and the memo
degrades to a yfinance stub. The job: **produce a trustworthy structured financial
data layer for TW financial-sector filers**, so a memo can be written without hand-
scraping MOPS. This is a DATA-LAYER arc — analysis/valuation shape is a separate arc
(see Out of Scope).

## Users

One user, self-serve, offline-first. Reads TW financial statements directly; wants
the same canonical + curated-notes surface the `-ci` arc gave industrials, adapted
to how banks actually report. Constraint the tooling must respect: a bank memo with
no asset-quality signal (NPL / coverage) is half-blind — three statements alone
carry far less information for a bank than for a manufacturer.

    (same wrong-answer class as the DCF total_debt gap). Union = 67 concepts, 18
    shared by all three sub-shapes; sub-shape-specific: life = separated-account
    (投資型保單) + policy loans, P&C = PAA premium-allocation, reinsurer = premium /

- **Round-4 frontier (MEASURED, 23 filers total) — NO 6th taxonomy**: bills-finance

  The fetch layer should try `C` then fall back to `A` rather than key on type.
  Balance-sheet spine present in all 23 filers across all 5 taxonomies (zero-miss).

## Smallest End State

A financial-sector canonical + curated bank-notes surface, wired into memo-fetch,
that emits trustworthy structured data and **cannot silently feed a wrong valuation
downstream**. Concretely:

1. **`-fh` canonical mapping** — three statements from measured concepts:
   assets/liabilities/equity/cash (`ifrs-full:`), deposits vs interest-bearing
   borrowings kept as DISTINCT named fields (`DepositsFromCustomers`/`DepositsFromBanks`
   vs `BondsIssued`/`OtherBorrowings`/`CommercialPapersIssuedNet`/repo), NII
   (`tifrs-bsci-fh:NetInterestIncomeExpense`), profit/EPS (`ifrs-full:`).
2. **5-way taxonomy classifier + sector marker** — replace the `_is_fh_taxonomy`
   boolean with a classifier returning one of `ci`/`fh`/`basi`/`bd`/`ins`; canonical
   carries `sector_class` (industrial vs financial, with the finer taxonomy tag). For
   financial filers the canonical does NOT emit the DCF trigger fields
   (`revenue`/`ebit`/`fcf`/`capex`/`total_debt`) — so any DCF attempt fails loud on
   missing revenue (`dcf_compute.py:175`) instead of silently zeroing net_debt. The
   explicit "DCF refuses financial sector" guard is analysis-layer (Out of Scope /
   BACKLOG), but this arc must not ENABLE the silent trap.
3. **`-fh` (FHC) canonical + bank asset-quality notes** — three statements (deposits
   vs interest-bearing borrowings distinct) + NPL/coverage/NPL-amount/gross-receivables
   (`tifrs-notes:`), Total-Loans aggregate row, tagged with banking-subsidiary name
   (`NameOfTheCompany`) so the memo states scope honestly (bank-sub only, NOT group).
4. **`-basi` (standalone bank + bills-finance) canonical + asset-quality notes** —
   three statements (its own renamed debt-financing concepts + 20+ deposit sub-lines
   rolled to a comparable shape) + NPL/coverage read from `context_ref` suffix (NO
   tuple parser needed). Must degrade gracefully for bills-finance `-basi` filers
   (華票-type) that carry NO NPL / NO deposits — absence is normal, not an error.
5. **Parser tuple support** — capture `tupleRef`/`tupleID` in the fact dict, keyed by
   subsidiary (post-merger FHCs carry duplicated NPL1+NPL2 trees). Needed for `-fh`
   NPL only; additive to the fact dict (existing fact-count guard stays valid).
6. **`-bd` (broker) canonical** — separate taxonomy, no NPL. Stable (88% cross-filer).
7. **`-ins` (insurer) canonical** — covers the 67-concept UNION across life / P&C /
   reinsurance; keys on `tifrs-bsci-ins:` PLUS an `ifrs-full:` fallback for the
   reinsurer sub-shape (which books insurance liabilities under generic concepts).
   Fetch layer routes insurers to `report_id=A` by filer type.
8. Fixtures captured from real filings (Big5, `binary` in `.gitattributes`),
   provenance-cited, cross-checked against the live extractor — ≥1 per taxonomy, plus
   a two-bank-subsidiary FHC and ≥2 insurer sub-shapes.

## Decision

**Build** (scope B, user-ratified): a financial-sector data layer covering FOUR new
taxonomies (`-fh`, `-basi`, `-bd`, `-ins`) behind a 5-way classifier. Lenders
(`-fh` + `-basi`) get the full treatment incl. asset-quality notes (逾放/覆蓋);
`-fh` NPL needs the tuple parser, `-basi` NPL does not. `-bd` and `-ins` get
three-statement canonical (`-ins` covering the life/P&C/reinsurance concept union).
A `sector_class` marker + DCF-trigger-field omission keep downstream valuation
fail-loud, not silent-wrong. Wired into memo-fetch with iXBRL-failure degradation to
yfinance stub (as `-ci`).

**Do NOT build** (this arc): any valuation method (DDM / residual-income / P-B-vs-ROE
scorecard), memo presentation/formatting of bank metrics, group-consolidated NPL (does
not exist in the source), annual-filing note reconstruction.

**Scope-adjustment (kickoff, user decision A 2026-07-22)**: the DCF sector-refusal
guard — originally deferred to BACKLOG — is PULLED INTO this arc. Reason: the intended
"DCF fails loud via missing revenue" would trip the memo's Phase-3 artifact gate
(`report-equity-memo/SKILL.md:344`) and hard-crash a TW financial memo, a regression
from today's runs-on-yfinance-stub behavior. Realized minimally as a structured
`not_applicable` result in `dcf_compute.py` (testable), NOT a prose skip in the memo
orchestrator (untestable). Still no valuation method, no memo rendering — those remain
domain-teams / BACKLOG.

**Why**: (a) the bank metrics the user actually needs are MEASURED-present, and the
one reason the current parser can't reach them (dropped tuple attributes) is
diagnosed, so the risk is known engineering, not unknown; (b) valuation shape is
genuinely a different jurisdiction (domain-teams / analysis layer) and the research
shows it's an unsettled design choice (EN/JA discount-model vs TW scorecard practice)
that should not be rushed inside a data arc; (c) omitting DCF-trigger fields + a
sector marker is the minimum that prevents this arc from regressing FHC memos into
silently-wrong valuations.

## Alternatives Considered

- **Scope A (lenders only: `-fh` + `-basi`)** — agent-recommended after round 2:
  both carry the asset-quality signal the user wanted; brokers/insurers do not.
  Rejected by user in favor of B (full coverage) — user has specific broker/insurer
  targets to research, so `-bd`/`-ins` earn their place now rather than a later arc.
- **Scope C (`-fh` only)** — the original arc name; rejected: leaves standalone banks
  (a whole lending sector with cheap-to-extract NPL) and the securities/insurance
  sectors unserved.
- **Three-statements-only (no bank ratios)** — rejected by user after briefing: a
  bank memo without asset-quality signal is half-blind. Would avoid the parser change
  (lowest risk) but delivers the least bank-relevant surface.
- **Emit a bank `total_debt` and let DCF run** — rejected: deposits-vs-borrowings is
  separable so a "technically correct" interest-bearing total_debt IS computable, but
  DCF on a bank is meaningless (FCFE undefined); producing a number would be confident-
  wrong. Chose sector-marker + fail-loud instead.
- **Full note-table reconstruction engine** — rejected (same as `-ci` arc): the value
  concentrates in a handful of machine-readable concepts; tuple support reaches them
  without a general table engine. CONDITIONAL REVERSAL trip-wire: if during
  implementation the NPL tuple rows cannot be stably mapped to loan categories without
  reconstructing the whole note table, drop bank-ratio scope back to BACKLOG and ship
  three-statements + sector marker only (do not build the table engine).
