## Problem

**JTBD**: When I (the human) am mid-conversation with an AI agent (typically Claude Code or similar) and lose track of *what we were doing / why the agent just asked what it asked / which option I was about to choose* — usually after a context-switch break, a long tool-output scroll, or an agent question whose premise I lost — I want the agent to **stop, restructure the last N minutes into a fixed 7-block schema, and end with a Synthesis-check** so I can read myself back into state in under 60 seconds **without** the agent silently guessing what I want and continuing.

The failure mode this skill exists to prevent: the agent's *own* free-form "let me summarize where we are" responses, which (a) drop user-stated intent (paraphrases instead of quoting), (b) skip the WHY behind the agent's last question, and (c) immediately continue without a Synthesis-check, so misalignment doesn't surface until 3 turns later when the user realizes the agent has been answering the wrong question.

## Users

- **Primary (v0.1 dogfood)**: kouko himself, mid-session in Claude Code, typically after one of:
  - Coming back from a 15-30 min break or context-switch to another window
  - Reading a long tool output (50+ lines) and losing thread of the question that triggered it
  - An agent question that loads on a premise the user can no longer reconstruct (*"why are you asking this?"*)
  - Working in Traditional Chinese / Japanese / English — triggers must be multilingual
- **Secondary (v0.2+)**: other monkey-skills users who recognize the same failure mode. Skill should work zero-config; no per-user customization at v0.1.
- **User non-goal**: NOT *"new session continuation"* (that's HANDOFF L2 — separate sister skill, deferred). NOT *"compress context to save tokens"* (that's Claude Code's `/compact` L4 — built-in, parallel layer). NOT *"return-after-away summary"* (that's Claude Code's built-in `/recap` v2.1.108 — different use case, see §Alternatives).

## Smallest End State

**One-paragraph summary**: User invokes the skill (typed slash `/recap-here` OR natural-language triggers: *"where were we"* / *"我們剛剛在幹嘛"* / *「ちょっと振り返って」* / *"recap"* / *"I'm lost"* / etc.) → skill produces a **single chat-inline message** following the V1 7-block schema (Situation / Background / Assessment / All-user-messages-verbatim / Why-this-question / Pending / Synthesis-check) → ends with the Synthesis-check explicitly asking user to confirm direction in one sentence before the agent continues → agent **does not proceed** until user confirms.

**Success criteria (4 共通核心原則 from synthesis research + 1 skill-specific 5th)**:
1. **Structured 7-block schema** every time — no free-form summary; consistency is the user-visible value
2. **Quote-not-paraphrase** on blocks 2 (Background) and 4 (User messages) — original strings preserved, especially file paths / error messages / user's exact wording
3. **All user messages retained** — block 4 lists every user turn in current session, no LLM filtering of *"which user messages were important"*
4. **Synthesis-check blocks continuation** — block 7 asks user to confirm next-step direction in one sentence; agent must wait
5. **Plain language by default** *(skill-specific, 2026-05-26)* — recap output uses plain everyday language so the human can read themselves back into state under 60 seconds without parsing jargon. Only use a technical term if the user introduced it first in the session (mirrors quote-not-paraphrase on the agent's own vocabulary). Acronyms expanded on first mention. Short sentences over long. One fact per bullet. Aligns with kouko's standing global preference ("Plain language over jargon unless I introduce the term first").

**Non-criteria (explicitly NOT measured at v0.1)**:
- ❌ Token efficiency — the skill is *human-optimized*, not token-optimized; verbose if needed
- ❌ Auto-trigger detection — user explicitly invokes; no "agent infers user is confused"
- ❌ Per-task variants (debug / design / research) — single schema, deferred to v0.2 per research V2 todo
- ❌ Cross-session continuation — that's HANDOFF (L2), separate skill
- ❌ File output — chat-inline only

## Alternatives Considered

WebSearch findings (EN + JA, run 2026-05-26):

1. **Claude Code built-in `/recap` (v2.1.108+, "away-summary")** — fires when user returns to session after being away; configurable in `/config` or `CLAUDE_CODE_ENABLE_AWAY_SUMMARY`. **Different use case**: cross-context return, not in-session re-orientation. Opaque schema (no public spec for the 9-block discipline we want enforced). **Conclusion**: coexist; our skill targets the gap (mid-session, explicit user invoke, opinionated schema visible to user). Source: [Claude Code Docs](https://code.claude.com/docs/en/how-claude-code-works) + [GitHub issue #48084](https://github.com/anthropics/claude-code/issues/48084).
2. **`hatawong/claude-recap` + `tzu-hsun/recap` (OSS plugins)** — both are *cross-session topic memory* (auto-capture across sessions, restore in new sessions). Closer to L2 (HANDOFF) than L3 (in-session recap). **Conclusion**: orthogonal to our scope; possible HANDOFF-skill prior art for next round. Sources: [hatawong/claude-recap](https://github.com/hatawong/claude-recap), [tzu-hsun/recap](https://github.com/tzu-hsun/recap).
3. **Custom slash command (plain `.claude/commands/recap.md`)** — Anthropic-recommended lightweight path: drop a markdown file with the prompt, invoke as `/<name>`. **Conclusion**: too thin for our 7-block discipline + multilingual triggers + Synthesis-check enforcement; a skill (with frontmatter triggers + description disambiguation + bundled READMEs) is the right ceremony level. Slash commands lack the trigger-description routing layer that skills give us. Source: [Claude Code Docs — Slash Commands](https://code.claude.com/docs/en/skills).
4. **DIY: user retypes the V1 7-block template each time** — the status quo before this skill. Inconsistent (user fatigue → drops blocks 4 + 7 first), error-prone (paraphrase creeps in), and the failure mode the skill exists to prevent. **Conclusion**: this IS what becomes obsolete (see §What Becomes Obsolete).

**JA search note**: EN search returned the built-in `/recap` finding directly; JA search (Qiita / Zenn / `振り返り` queries) returned general prompt-engineering guides + Claude Code skill-creation tutorials with **0 in-session-recap-specific** prior art. The Japanese ecosystem has not yet productized this L3 layer — finding itself is a signal that the gap is real cross-language.

**My take**: build it as a skill in `dev-workflow` (not new plugin), single SKILL.md scope at v0.1, name disambiguates from built-in `/recap`.

## Decision

**Build `dev-workflow:recap` v0.1** as a single-skill addition to the `dev-workflow` plugin. Skill is a structured-prompt template implementing the Recap-research V1 7-block schema, with multilingual triggers (zh-TW / ja / en), Synthesis-check enforcement, and an explicit description that disambiguates from built-in `/recap` (away-summary). One SKILL.md + tri-language READMEs + bundled rubric for the 4 共通核心原則. NO subagent dispatch (self-execute in main conversation, since the whole point is the human reads the agent's reply). NO Python scripts. NO file output. v1.0 deferred items: per-task-type variants (debug / design / research), auto-trigger detection, test-prompts/trigger-eval scaffolding.

The plugin placement (`dev-workflow:` over a new `continuation-stack:` plugin) mirrors the `distill-sessions` decision: a single skill does not justify a new plugin. If HANDOFF (L2) ships next and a third L1/L4 skill follows, graduate the family to `continuation-stack:` at that point.

- **`test-prompts.json` + `trigger-eval.json`** — defer to v0.2 once dogfood reveals real trigger patterns; v0.1 ships description-only routing
- **Slash command registration** (`.claude/commands/recap-here.md` or similar) — let the skill's description handle natural-language triggers at v0.1; add explicit slash command in v0.2 if needed

## What Becomes Obsolete

- **Ad-hoc "where were we" / "我們剛剛在幹嘛" prompts** — they survived because nothing better existed; once the skill ships and the user gets in the habit of triggering it, the ad-hoc versions retire naturally. **Not same-PR removable** (they're in user's muscle memory, not in code); the skill's description acts as the redirect.
- **Manual re-typing of the V1 7-block template** — the Recap research note's §"我的 Recap Prompt 模板（V1）" was a static reference for kouko to copy-paste; once the skill exists, the template lives in `SKILL.md` and the research note becomes a *design source* not a *runtime crib sheet*. The note itself stays in vault (it's the spec); the copy-paste workflow retires.
