## Time-box rationale (per reviewer feedback 2026-05-19 round 1)

Tasks T2 / T4 / T5 are coarser than the strict ≤5-min ideal documented in writing-plans SKILL.md §Splitting framework #1:

- **T2** (version bump 0.1.6 → 0.2.0 lock-step across plugin.json + 3 READMEs + possibly 5 SKILL.md frontmatters): a focused implementer with `grep -rln '0\.1\.6' collab-toolkit/` + batched Edit calls completes in 4-6 min; tight but realistic.
- **T4** (rewrite 204-line `commands/collab-setup.md`): genuine ~8-10 min for full body rewrite + live `brew search gws` verification.
- **T5** (3-locale README rewrite, mostly mirrored content): genuine ~8-12 min.

**Decision**: keep as single tasks. Rationale:
1. Splitting T4 into "write body" + "verify install path" introduces handoff cost ≥ the original task's overrun.
2. Splitting T5 into EN + JA + zh-TW would triple SDD reviewer overhead for mostly-mirrored content.
3. SDD implementer is empowered to invoke **Beck (2002) Child-Test fallback** (return BLOCKED with `unblock_step: "this task needs to be split smaller"`) if the task genuinely blocks mid-execution — the recursive fallback mechanism documented in writing-plans SKILL.md §BLOCKED fallback is the safety net.

Per Part 2 same rationale applies (T1-T5 each rewrite 6 files within one skill folder).
