## Time-box rationale (per reviewer feedback 2026-05-19 round 1)

Each task T1-T5 rewrites 6 files (SKILL.md + 4 protocols + failure-modes.md) within one skill folder + runs live MCP tool discovery via `/mcp add <service>` to confirm protocol coverage. Realistic estimate: ~8-12 min per skill.

**Decision**: keep as single tasks per skill. Rationale:
1. Splitting "discover MCP tools" + "rewrite SKILL/protocols" introduces handoff cost ≥ the original task's overrun; both ops need the same loaded context (tool list + spec mapping).
2. The 6-file scope is bounded by **one skill module** (criterion 2 — module scope — strictly satisfied). The criterion-1 ≤5-min ideal is the looser constraint.
3. SDD implementer is empowered to invoke **Beck (2002) Child-Test fallback** (return BLOCKED with `unblock_step: "this task needs to be split smaller"`) if mid-execution mapping discovers many missing tools — recursive fallback documented in writing-plans SKILL.md §BLOCKED fallback.
4. Coverage gap protocol (in §Common per-task contract): if any v0.1.6 protocol has no MCP/CLI mapping, implementer drops it + documents + flags `DONE_WITH_CONCERNS`. This absorbs the most common "task ballooned" failure mode without a full Child-Test re-split.

## Common per-task contract

Each of T1-T5 rewrites one skill folder under `collab-toolkit/skills/<service>-automate/`:
- `SKILL.md` — replace `allowed-tools` line with new tool channel; rewrite Prerequisites + Hero protocols + Workflow pattern; drop `## Supported UI languages` and `## Localized labels` sections (locale-independence is free with API)
- `protocols/*.md` — rewrite each as MCP/CLI invocation recipe (no UI-snapshot patterns); drop `## Localized labels` sections
- `references/failure-modes.md` — rewrite for new failure modes per brief §JA-search migration risk findings: OAuth scope, token refresh, rate limit, browser-flow timeout, confused-deputy

If a v0.1.6 hero protocol has **no MCP/CLI mapping** in the new channel (Open Q3-Q5 surfaces here):
- Remove the protocol file
- Document the gap + reason in `references/failure-modes.md` "Coverage gaps from v0.1.6 → v0.2.0" subsection
- Note removed protocol in `SKILL.md` Hero protocols list (omitted with brief note pointing to failure-modes)
- Implementer flags task `DONE_WITH_CONCERNS` if any protocol is dropped
