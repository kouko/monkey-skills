# Kaizen Skill

**English** | [日本語](README.ja.md) | [繁體中文](README.zh-TW.md)

Identify and implement small, continuous improvements in existing processes
through a structured 6-step cycle.

## The Six Steps

| Step | Japanese | Meaning | Key Action |
|------|----------|---------|-----------|
| 1 | 現状把握 | Grasp the current state | Observe without judgment |
| 2 | 問題発見 | Find the problem | Identify friction and waste |
| 3 | 根本原因 | Root cause | Ask "why?" until you reach the core |
| 4 | 改善案 | Improvement proposal | Propose the smallest change |
| 5 | 実行と検証 | Execute and verify | Try it, measure the result |
| 6 | 標準化 | Standardize | Make it the new default |

## Method Type

Process-driven (not dialogue-driven like Socratic method).
Agent guides the user through each step sequentially, ensuring
improvements stay small, reversible, and measurable.

Key principle: Kaizen is about continuous small improvements,
not big-bang transformations. If the proposed change feels like
a revolution, it's too big.

## Seven Wastes (Muda)

Adapted from Toyota Production System for knowledge work:

| Waste Type | Japanese | Knowledge Work Example |
|-----------|----------|----------------------|
| Overproduction | 作りすぎ | Building features nobody asked for |
| Waiting | 待ち | Blocked on code review, approval queues |
| Transportation | 運搬 | Copying data between tools, manual transfers |
| Over-processing | 加工 | Unnecessary steps, gold-plating |
| Inventory | 在庫 | Unprocessed backlog, unread notifications |
| Motion | 動作 | Context switching, tool-hopping |
| Defects | 不良 | Bugs, rework, miscommunication |

## Improvement Principles

| Principle | Description | Anti-pattern |
|-----------|-------------|-------------|
| Smallest change | Change one thing at a time | "Let's rewrite the whole pipeline" |
| Reversible | Can undo if it doesn't work | Irreversible infrastructure changes |
| Measurable | Can quantify before/after | "It feels better" without data |
| Immediate | Start this week, not "someday" | "We'll do it in Q3" |
| Cyclical | One improvement leads to the next | "We're done improving" |

## Cultural Origin

Kaizen originates from post-WWII Japanese manufacturing, most notably
the Toyota Production System developed by Taiichi Ohno. The philosophy
emphasizes that every employee, from CEO to factory floor worker, should
continuously look for ways to improve their work. W. Edwards Deming's
quality management principles heavily influenced the development of
Kaizen methodology.
