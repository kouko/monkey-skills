# Four Causes — Additional Business Cases

Extended examples for reference. Use these as inspiration when guiding
users through four-cause analysis in different domains.

## Patagonia (Apparel / Environmental)

| Cause | Analysis |
|-------|----------|
| Material | Recycled materials, organic cotton, fair-trade certified factories |
| Formal | Outdoor apparel + repair service model (Worn Wear program) |
| Efficient | Yvon Chouinard's environmental activism + climbing culture origins |
| Final | "We're in business to save our home planet" |

**Key insight:** The final cause (environmental mission) reshaped the
material cause — choosing recycled/organic materials costs more but
serves the purpose. Patagonia's "Don't Buy This Jacket" campaign is
the final cause directly contradicting typical retail efficient cause
(drive sales).

## Notion (Software / Knowledge Management)

| Cause | Analysis |
|-------|----------|
| Material | Block-based database, React, collaborative editing engine |
| Formal | "Everything is a block" — unified data model (pages, databases, embeds all blocks) |
| Efficient | Ivan Zhao's belief that tools should be beautiful + frustration with fragmented tooling |
| Final | Consolidate team knowledge into one flexible workspace |

**Key insight:** The formal cause (everything-is-a-block) is the core
innovation. It enables the final cause (one workspace for everything)
by making all content types interchangeable.

## GitHub (Software / Developer Collaboration)

| Cause | Analysis |
|-------|----------|
| Material | Git (distributed VCS), Ruby on Rails, servers, user repositories |
| Formal | Repository → Issue → Pull Request → Review workflow |
| Efficient | Tom Preston-Werner's vision to make Git social + OSS community |
| Final | Democratize software development collaboration |

**Key insight:** Git existed before GitHub (material), but GitHub's
formal cause (the PR workflow) transformed it from a version control
tool into a collaboration platform. The form changed the purpose.

## BEAR.Sunday Order Cancellation (Software Architecture)

| Cause | Analysis |
|-------|----------|
| Material (Hyle) | JSON schemas, database tables, payment/inventory APIs |
| Formal (Eidos) | Resource relationships defining when orders are cancellable |
| Efficient (Kinesis) | Service calls: refund payment, restock inventory, notify stakeholders |
| Final (Telos) | Maintain system-wide consistency across the entire cancellation process |

**Key insight:** REST is not CRUD over HTTP — it embodies four causes
by ensuring resources maintain coherence through proper state transitions
that respect business rules.

Source: [四原因説とリソース設計 — Qiita](https://qiita.com/koriym/items/b5e23e23d17b18bc1f38)

## Compost Business (Resource Redefinition)

| Cause | Analysis |
|-------|----------|
| Material | Kitchen waste (reframed from "garbage" to "resource") |
| Formal | Composting process converting waste into premium fertilizer |
| Efficient | Environmental regulations + organic farming demand |
| Final | Solve waste disposal problems by creating value from refuse |

**Key insight:** The entire business model is a material cause
redefinition — the same physical substance (kitchen waste) shifts
from "disposal problem" to "raw material" by changing the formal
cause (composting process).

Source: [四原因説から生まれるビジネスアイデア — Financial Note](https://financial-note.com/aristoteles/)
