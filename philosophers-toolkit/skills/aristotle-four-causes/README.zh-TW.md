# Aristotle's Four Causes Skill

[English](README.md) | [日本語](README.ja.md) | **繁體中文**

透過理解的四個基本維度分析任何主題。

## 四因

| Cause | 提問 | Software 範例 |
|-------|------|--------------|
| Material | 它由什麼構成？ | 程式語言、framework、資料結構 |
| Formal | 是什麼讓它成為它自己？ | 架構、介面、design patterns |
| Efficient | 是什麼造就了它？ | 開發者、市場驅動力、使用者需求 |
| Final | 它的目的是什麼？ | 解決的使用者問題、商業價值 |

## Method Type

Framework-driven 分析（不像 Socratic method 是 dialogue-driven）。
agent 依序帶領使用者走過 4 個維度，每個維度提出 1-2 個問題，
最後綜合各 cause 之間的關聯。

## SKILL.md 中的範例

| 範例 | Domain | 關鍵洞見 |
|------|--------|----------|
| IKEA | 零售 | 每個 cause 都對齊同一個 telos（可負擔性） |
| Tesla | 汽車 / 能源 | Material cause（電池供應）限制了 final cause（大眾普及） |
| Slack | Software / SaaS | Efficient cause（意外轉型）在發布前就驗證了 final cause |

## 適用領域

| Domain | Material | Formal | Efficient | Final |
|--------|----------|--------|-----------|-------|
| Software | 技術堆疊、資料 | 架構、API | 開發流程、觸發點 | 使用者問題、價值 |
| Product | 元件、輸入 | 形式、UX | 設計者、市場 | JTBD、價值主張 |
| Organization | 人、資本 | 結構、文化 | 創辦人、決策 | Mission、價值 |
| Concept | 假設、前提 | 定義、邊界 | 思想家、脈絡 | 解釋的問題 |
