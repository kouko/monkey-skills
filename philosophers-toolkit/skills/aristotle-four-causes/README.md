# Aristotle's Four Causes Skill

Analyze any subject through four fundamental dimensions of understanding.

## The Four Causes

| Cause | Question | Software Example |
|-------|----------|-----------------|
| Material | What is it made of? | Languages, frameworks, data structures |
| Formal | What makes it what it is? | Architecture, interfaces, design patterns |
| Efficient | What brought it about? | Developers, market forces, user needs |
| Final | What is it for? | User problem solved, business value |

## Method Type

Framework-driven analysis (not dialogue-driven like Socratic method).
Agent guides user through 4 dimensions in order, asking 1-2 questions
per dimension, then synthesizes connections between causes.

## Examples in SKILL.md

| Example | Domain | Key Insight |
|---------|--------|-------------|
| IKEA | Retail | Every cause aligns toward the same telos (affordability) |
| Tesla | Automotive / Energy | Material cause (battery supply) constrains final cause (mass adoption) |
| Slack | Software / SaaS | Efficient cause (accidental pivot) validated final cause before launch |

## Applicable Domains

| Domain | Material | Formal | Efficient | Final |
|--------|----------|--------|-----------|-------|
| Software | Tech stack, data | Architecture, APIs | Dev process, triggers | User problem, value |
| Product | Components, inputs | Form factor, UX | Designer, market | JTBD, value prop |
| Organization | People, capital | Structure, culture | Founders, decisions | Mission, value |
| Concept | Assumptions, priors | Definition, boundaries | Thinkers, context | Problem explained |
