# Hegelian Dialectics Skill

[English](README.md) | [日本語](README.ja.md) | **繁體中文**

透過 thesis、antithesis 與 synthesis 來檢視任何立場，
找出超越二元選擇的更高階解法。

## 三個 phase

| Phase | 提問 | 目的 |
|-------|------|------|
| Thesis | 立場是什麼？ | 表述並 steel-man 起始主張 |
| Antithesis | 它在何處自我矛盾？ | 浮現內部矛盾，而非外部攻擊 |
| Synthesis | 什麼能超越二者？ | 重新框架問題，使張力消解 |

## Method Type

Phase-driven 流程（不像 Socratic 是 dialogue-driven，也不像 Four Causes 是 framework-driven）。
三個依序的 phase，每個都基於前一個。可選擇迭代：synthesis 變成新的 thesis。

## 關鍵區別

| | 妥協 | Synthesis |
|--|------|-----------|
| 做法 | 取兩端的中間 | 重新框架問題 |
| 結果 | 任一邊都未完全滿意 | 張力在更高層次消解 |
| 範例 | 「一半 monolith、一半 microservices」 | 「modular monolith：有邊界但不分散」 |

## SKILL.md 中的範例

| 範例 | Domain | Thesis | Antithesis | Synthesis |
|------|--------|--------|------------|-----------|
| Monolith vs Microservices | Software architecture | Monolith 簡單 | 規模化時統一性反成瓶頸 | Modular monolith |
| Speed vs Quality | 開發流程 | 為市場快速出貨 | 速度製造會自我拖慢的債 | 適切的 scope 同時兼顧兩者 |
| Centralization vs Autonomy | 組織 | 透過標準達成一致性 | 標準形成瓶頸 | Golden path with opt-out |
