# Hegelian Dialectics Skill

Examine any position through thesis, antithesis, and synthesis
to find higher-order solutions that transcend binary choices.

## The Three Phases

| Phase | Question | Purpose |
|-------|----------|---------|
| Thesis | What is the position? | Articulate and steel-man the starting claim |
| Antithesis | Where does it contradict itself? | Surface internal contradictions, not external attacks |
| Synthesis | What transcends both? | Reframe the problem to dissolve the tension |

## Method Type

Phase-driven process (not dialogue-driven like Socratic, not framework-driven
like Four Causes). Three sequential phases, each building on the previous.
Optional iteration: synthesis becomes new thesis.

## Key Distinction

| | Compromise | Synthesis |
|--|-----------|-----------|
| Approach | Split the difference | Reframe the problem |
| Result | Neither side fully satisfied | Tension dissolves at higher level |
| Example | "Do half monolith, half microservices" | "Modular monolith: boundaries without distribution" |

## Examples in SKILL.md

| Example | Domain | Thesis | Antithesis | Synthesis |
|---------|--------|--------|------------|-----------|
| Monolith vs Microservices | Software architecture | Monolith simplicity | Unity becomes bottleneck at scale | Modular monolith |
| Speed vs Quality | Development process | Ship fast for market | Speed creates debt that slows itself | Right scope enables both |
| Centralization vs Autonomy | Organization | Consistency via standards | Standards create bottlenecks | Golden path with opt-out |
