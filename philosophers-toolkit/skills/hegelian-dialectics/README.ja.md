# Hegelian Dialectics Skill

[English](README.md) | **日本語** | [繁體中文](README.zh-TW.md)

任意の立場を thesis、antithesis、synthesis を通して検証し、
二者択一を超える高次の解を見出す。

## 3 つの phase

| Phase | 問い | 目的 |
|-------|------|------|
| Thesis | 立場は何か？ | 出発点となる主張を明確にし steel-man する |
| Antithesis | どこで自己矛盾するか？ | 外部からの攻撃ではなく内部矛盾を浮かび上がらせる |
| Synthesis | 両者を超えるものは？ | 問題の枠組みを変え緊張を解消する |

## Method Type

Phase-driven プロセス（Socratic のような dialogue-driven でも、Four Causes のような framework-driven でもない）。
3 つの順次 phase が積み重なる。任意の反復：synthesis が新しい thesis になる。

## 主な違い

| | 妥協 | Synthesis |
|--|------|-----------|
| アプローチ | 中間を取る | 問題の枠組みを変える |
| 結果 | どちらも完全には満たされない | 緊張がより高い層で解消する |
| 例 | 「半分 monolith、半分 microservices」 | 「modular monolith：境界はあるが分散しない」 |

## SKILL.md 中の例

| 例 | Domain | Thesis | Antithesis | Synthesis |
|------|--------|--------|------------|-----------|
| Monolith vs Microservices | Software architecture | Monolith の単純さ | 規模拡大で統一性が bottleneck になる | Modular monolith |
| Speed vs Quality | 開発プロセス | 市場へ素早く出す | 速度が自らを遅らせる負債を生む | 適切な scope で両立 |
| Centralization vs Autonomy | 組織 | 標準による一貫性 | 標準が bottleneck になる | Golden path with opt-out |
