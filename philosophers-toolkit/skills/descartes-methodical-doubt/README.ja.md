# Descartes' Methodical Doubt Skill

[English](README.md) | **日本語** | [繁體中文](README.zh-TW.md)

不確実な前提をすべて体系的に剥がし、疑い得ないものだけが残るまで進め、
その基盤から再構築する。

## 4 つの layer

| Layer | 問い | 目的 |
|-------|------|------|
| State the Belief | 何を「知っている」と主張しているのか？ | 信念を精密かつ具体的にする |
| Apply Doubt Layers | これは疑えるか？どのような条件で？ | 信念を sensory、reasoning、systemic の疑いに晒す |
| Identify What Survives | 最大限の懐疑のあとに何が残るか？ | 疑い得ない基盤を見つける |
| Rebuild from the Indubitable | 残ったものから何を正当化できるか？ | 岩盤から誠実な結論を組み立てる |

## 3 つの疑いの layer（Layer 2 内）

| Doubt Type | 対象 | 例 |
|------------|------|-----|
| Sensory | 知覚、証拠、計測 | 「monitoring 自体は信頼できるか？」 |
| Reasoning | 論理、推論、因果 | 「authentication は security と等しいか？」 |
| Systemic | framework、パラダイム、問題の枠組み | 「そもそも正しい問題を解いているか？」 |

## Method Type

Process-driven（体系的な排除と再構築）。
First Principles（真理まで分解して再構築）や
Socratic Method（問いかけによる dialogue-driven な挑戦）とは異なる。

## 信念の判定

| Verdict | 意味 |
|---------|------|
| Survives | 疑う妥当な理由が見つからなかった |
| Weakened | 疑う理由は存在するが決定的ではない |
| Defeated | 疑う妥当な理由が見つかった — 不確実として扱う |

## SKILL.md 中の例

| 例 | Domain | 元の信念 | 疑いを生き残ったもの |
|------|--------|----------|---------------------|
| Security Audit | API security | 「JWT + HTTPS なので API は安全」 | 「これらのツールは使っているが security は検証していない」 |
| Data Decision | A/B testing | 「ユーザーは新しい dashboard を好む」 | 「power user の操作は多いが理由は不明」 |

## インスピレーション

4 layer の疑いプロセスは Descartes『Meditations on First
Philosophy』（1641）に基づき、技術的な意思決定の文脈に翻案。
