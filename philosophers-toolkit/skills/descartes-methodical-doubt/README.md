# Descartes' Methodical Doubt Skill

Systematically strip away every uncertain premise until only the
indubitable remains, then rebuild from that foundation.

## The Four Layers

| Layer | Question | Purpose |
|-------|----------|---------|
| State the Belief | What exactly are you claiming to know? | Make the belief precise and concrete |
| Apply Doubt Layers | Can this be doubted? Under what conditions? | Subject belief to sensory, reasoning, and systemic doubt |
| Identify What Survives | What remains after maximum skepticism? | Find the indubitable foundations |
| Rebuild from the Indubitable | What can you justify from what survived? | Construct honest conclusions from bedrock |

## Three Doubt Layers (within Layer 2)

| Doubt Type | Target | Example |
|------------|--------|---------|
| Sensory | Perception, evidence, measurement | "Is the monitoring itself reliable?" |
| Reasoning | Logic, inference, causation | "Does authentication equal security?" |
| Systemic | Framework, paradigm, problem framing | "Are we even solving the right problem?" |

## Method Type

Process-driven (systematic elimination and reconstruction).
Unlike First Principles (decompose and rebuild from truths) or
Socratic Method (dialogue-driven challenge through questions).

## Belief Verdicts

| Verdict | Meaning |
|---------|---------|
| Survives | No credible reason to doubt was found |
| Weakened | Plausible reasons to doubt exist but are not conclusive |
| Defeated | A credible reason to doubt was found — treat as uncertain |

## Examples in SKILL.md

| Example | Domain | Original Belief | What Survived Doubt |
|---------|--------|----------------|---------------------|
| Security Audit | API security | "Our API is secure because JWT + HTTPS" | "We use these tools but haven't verified security" |
| Data Decision | A/B testing | "Users prefer the new dashboard" | "Power users interact more, but we don't know why" |

## Inspiration

Four-layer doubt process inspired by Descartes' *Meditations on First
Philosophy* (1641), adapted for technical decision-making contexts.
