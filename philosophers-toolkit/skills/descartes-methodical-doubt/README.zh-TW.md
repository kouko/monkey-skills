# Descartes' Methodical Doubt Skill

[English](README.md) | [日本語](README.ja.md) | **繁體中文**

系統性地剝除每一個不確定的前提，直到只剩下不可懷疑的部分，然後從那個基礎重建。

## 四個層次

| Layer | 提問 | 目的 |
|-------|------|------|
| State the Belief | 你究竟主張自己知道什麼？ | 讓信念精確而具體 |
| Apply Doubt Layers | 這能被懷疑嗎？在什麼條件下？ | 將信念置於 sensory、reasoning、systemic 的懷疑之下 |
| Identify What Survives | 經過最大限度的懷疑後，剩下什麼？ | 找出不可懷疑的基礎 |
| Rebuild from the Indubitable | 從倖存的部分能正當推出什麼？ | 從基石建立誠實的結論 |

## 三種懷疑層次（Layer 2 內）

| Doubt Type | 對象 | 範例 |
|------------|------|------|
| Sensory | 感知、證據、量測 | 「monitoring 本身是否可靠？」 |
| Reasoning | 邏輯、推論、因果 | 「authentication 等於 security 嗎？」 |
| Systemic | 框架、典範、問題框架 | 「我們解的是對的問題嗎？」 |

## Method Type

Process-driven（系統性消除與重建）。
與 First Principles（拆解到真理再重建）或
Socratic Method（透過提問挑戰思考的 dialogue-driven）不同。

## 信念裁決

| Verdict | 含義 |
|---------|------|
| Survives | 找不到可信的懷疑理由 |
| Weakened | 存在合理的懷疑理由，但尚未具決定性 |
| Defeated | 找到可信的懷疑理由 — 視為不確定 |

## SKILL.md 中的範例

| 範例 | Domain | 原始信念 | 通過懷疑的結果 |
|------|--------|----------|---------------|
| Security Audit | API security | 「我們的 API 安全，因為有 JWT + HTTPS」 | 「我們使用了這些工具，但尚未驗證安全性」 |
| Data Decision | A/B testing | 「使用者偏好新的 dashboard」 | 「power user 互動較多，但我們不知道為什麼」 |

## 靈感來源

四層懷疑流程靈感來自 Descartes《Meditations on First
Philosophy》（1641），改編以適用於技術決策情境。
