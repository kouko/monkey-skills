# 輸出版型 — daily-brief 雙產物

每天產出三個檔(扁平日期前綴,append-only 台帳):一份 **curated 晨報**(`<date>_晨報.md`)、一份 **零省略完整事項表**(`<date>_完整事項.md`)、一份 **核心機讀 CSV**(`<date>_完整事項.csv`)。選配 `README.md`。

**這份檔的三條 load-bearing 知識(不要稀釋成一般模板)**:

1. **雙產物化解 PDB 張力** — 晨報「狠下心省略」(只決定誰升上焦點),完整表「零省略」(fan-out 撈到的每一筆都在)。過濾只決定升級,**不決定丟棄**。兩者缺一,要嘛漏事、要嘛回一面牆。
2. **無出處不靜默省略** — 鏡像 source skill「無出處不寫」的**反向約束**:無單一來源的項目**仍列入**,連結欄標「⚠️ 無直連」。靜默丟掉「找不到連結的事」=「假完整」,這是本 skill 要防的失敗模式。
3. **CSV 唯一識別碼欄 = 隔天延續性的 JOIN 主鍵** — 不是裝飾欄。明天 continuity-load 讀這份 CSV、用唯一識別碼去 live 平台重驗、算出 🆕/⏳/✅/🔄。**唯一識別碼欄缺失 = 延續性鏈斷裂**。

---

## 1. 檔案累積 + 命名規則

輸出資料夾 = **append-only 每日台帳**,扁平日期前綴(不開每日子資料夾):

```
<使用者在 0-B Intake 指定的根資料夾>/
├── 2026-06-01_晨報.md   2026-06-01_完整事項.md   2026-06-01_完整事項.csv
├── 2026-06-02_晨報.md   2026-06-02_完整事項.md   2026-06-02_完整事項.csv
└── 2026-06-03_晨報.md   2026-06-03_完整事項.md   2026-06-03_完整事項.csv   ← 今天
```

- **檔名格式**:`<YYYY-MM-DD>_晨報.md`、`<YYYY-MM-DD>_完整事項.md`、`<YYYY-MM-DD>_完整事項.csv`。日期 = 產生當天(`currentDate`),不是涵蓋窗起點。
- **為何扁平日期前綴**(不開子資料夾):好 `ls` 排序、好 `grep`、continuity-load 用字串比較就能定位「日期<今天的最近一份」。
- **產生前必做 continuity-load**:定位「日期<今天的最近一份」那組檔 → 讀其 `.csv`(JOIN key)+ 晨報的焦點/未來段做延續性對照。判定與 4 狀態(🆕/⏳/✅/🔄)、live 重驗、首份/跳天降級 → 見 `prioritization-framework.md` §延續性。
- **首份**(資料夾內無更早日期):跳過晨報「自上次以來」段,標「首份簡報,無對照」。**跳天**(最近一份不是昨天):讀最近一份並標「距上次 N 天」。

## 2. 每份開頭強制區塊

**三個檔的 `.md` 開頭都要有**(CSV 不放,CSV 是純機讀);順序固定:

1. **期間 / 今日日期 / 身份** — 收件人、產生時刻、涵蓋窗(今日 + 未來 N 天 + 近 48h)。
2. **資料源涵蓋聲明(0-A Gate 結果)** — 直接搬 source skill 的三態,**盲區必明標**:
   ```
   > 資料源涵蓋(0-A Gate 結果):
   > ✅ 已涵蓋:{就緒平台清單}
   > ⚠️ 受阻:{平台 — 原因,如被 AI-eligibility 標籤封鎖 / 權限不足 / 讀取逾時}
   > ❌ 未連:{未連 MCP 的平台}
   > 以上 ⚠️/❌ 平台為今日盲區,相關項目可能不完整。
   ```
   - ⚠️ 與 ❌ 的差別:**⚠️ = 連上了但讀不到**(要寫原因)、**❌ = 根本沒連 MCP**。兩者都進盲區聲明,但原因不同 → 使用者判斷「是否值得去修連線」。
3. **範圍 / 排除聲明** — 涵蓋窗 N 天、Intake 指定的排除主題(原始資料未動)。

## 3. `<date>_晨報.md` 版型(curated,PDB 風格)

**6 段結構**(brief §Output Artifact Specs 定版,不增刪段):涵蓋聲明 → 📈 自上次以來 → 🎯 今日焦點 → 📅 當日行程 → ✉️ 要回覆·要處理 → 🚧 進行中專案現況 → 🔭 未來 N 天 → 底部下鑽指標 + 信心圖例。

### 字數上限規則(硬規則 — JP 實務圈反覆點名的最大坑)

| 範圍 | 建議上限 | 超量處置 |
|---|---|---|
| 每段 | ≤ 約 300 字 | **收緊納入門檻**,不省略已選中項 |
| 全文晨報 | ≤ 約 1,500 字(中文)| 同上 |

- **超量時收緊門檻,不是砍內容**:晨報太長 → 提高「今日焦點」納入門檻(只留會卡別人/今天有後果的)、把次要項下推到完整表。**完整事項表永遠零省略,不受字數天花板約束。**
- 理由:無上限 = AI 回一面牆,使用者讀不完 → 簡報失去「60 秒讀回全貌」的價值。

### emoji 開關

- **預設**:emoji 分區(📈🎯📅✉️🚧🔭),掃讀快。
- **Intake 可關**:純文字標題版(`## 自上次以來` / `## 今日焦點` …),段落結構與內容完全相同,只去 emoji 與信心圖示(信心改文字 `[交叉印證]` / `[單一來源]` / `[推論]`)。

## 4. `<date>_完整事項.md` 版型(零省略,可點處理)

依**重要度降序**的 markdown 表,**每一列都帶可點深連結**。欄位固定(brief 定版,不增刪欄):

```
# | 重要度 | 分類 | 平台 | 標題 | 🔗連結 | 日期/期限 | 動作 | 一句話 | 信心
```

- **分類**:`行程` / `要回覆` / `專案` / `未來`(對應晨報的 4 個內容段)。
- **動作**:`需回覆` / `需準備` / `僅知會`。
- **信心**:`✅`(2+ 平台交叉印證)/ `⚠️`(單一來源)/ `推論`(由情境推得)。
- **硬規則(鏡像「無出處不寫」的反向)**:無單一來源 canonical 連結的項目**仍列入**,🔗連結欄填 `⚠️ 無直連`,**不靜默省略**。

## 5. `<date>_完整事項.csv` schema(核心機讀 — 隔天 delta 的 JOIN key)

鏡像 source skill 的索引 CSV。**欄位順序固定,逗號分隔**:

```
編號,重要度,分類,平台,標題,唯一識別碼,連結,日期,動作,一句話,信心
```

- **唯一識別碼欄 = 隔天延續性分析的 JOIN 主鍵 → 不可省略、不可留空。** 明天 continuity-load 讀這欄、拿 ID 去 live 平台重驗、算 🆕/⏳/✅/🔄。缺這欄 = 延續性鏈斷。
- 唯一識別碼形式依平台而定(Gmail 16 碼 thread ID / Slack `頻道ID-p時間戳` / Notion 32 碼 page ID / Asana 數字 GID / GitHub `repo#號` / Calendar `event id` / Drive `file ID`)— 完整對照見 `platform-search-playbook.md` §canonical 連結與識別碼。
- **無單一來源者唯一識別碼欄**:填可重建的 fallback key(如 `<平台>-<標題雜湊>`),**不留空** — 否則明天無法 JOIN。連結欄填 `⚠️無直連`。
- **避免值內逗號**:一句話/標題若含逗號,改頓號「、」或用全形逗號「,」;不引號跳脫(下游可能裸讀)。
- **`.md` 與 `.csv` 同列數**:CSV 是完整事項表的機讀鏡像,逐列對應,不增不減。

### CSV 範本(示意資料用佔位)

```csv

2,高,行程,<平台>,<Q3 路線圖定案>,<event-id>,https://...,<YYYY-MM-DD HH:MM>,需準備,<要你拍板>,✅
3,高,專案,<平台>,<payments-api#482 待 review>,<repo#482>,https://...,,需準備,<CI 綠、阻擋部署>,✅
4,中,未來,<平台>,<結帳改版上線>,<asana-gid>,https://...,<YYYY-MM-DD>,需準備,<你是 release owner>,✅
5,中,要回覆,<平台>,<法務 NDA 用印>,<平台-NDA雜湊>,⚠️無直連,<到期日>,需確認,<推測等你>,推論
6,低,行程,<平台>,<Daily standup>,<event-id>,https://...,<HH:MM>,僅知會,<例行>,✅
```

## 6.(選配)`README.md`

**僅 Intake 要求時產**。導覽資料夾、重申範圍聲明,不重複晨報內容。

```markdown
# 每日簡報台帳 — <根資料夾名>

每天三檔(扁平日期前綴,append-only):
- `<date>_晨報.md` — curated 6 段晨報(PDB 風格,有字數上限)
- `<date>_完整事項.md` — 零省略可點處理表(每列帶深連結)
- `<date>_完整事項.csv` — 機讀;**唯一識別碼欄 = 隔天延續性 JOIN key**

## 範圍與盲區
- 涵蓋窗:今日 + 未來 <N> 天 + 近 48h。
- 資料源涵蓋 / 盲區:見每份檔開頭的「資料源涵蓋(0-A Gate 結果)」。
- 排除主題:<排除主題>(原始資料未動)。
- read-only / draft-only:本台帳只讀各平台、只寫本機;不代送、不回寫官方系統。
```
