# briefing-toolkit

接続済みツールを横断する毎朝の**デイリーブリーフ**。MCP 接続済みの各サービスを read-only で並列に巡回し、大統領デイリーブリーフ（PDB）風の要約に蒸留——さらに、**一件も省略しない**リンク付きのアクション表を、日々**累積**して継続性分析に使います。

> [`README.md`](README.md) ・ [`README.zh-TW.md`](README.zh-TW.md)

## できること

`/daily-brief` は最大 **7 プラットフォーム**（Gmail / Slack / Notion / Asana / Google Drive / Google Calendar / **GitHub**）を並列・read-only で巡回し、**あなたに関係する**事項を集め、指定フォルダに 2 つの成果物を書き出します。

| 成果物 | 役割 |
|---|---|
| `<日付>_晨報.md` | **キュレーション済みブリーフ**（PDB 風）：今日のフォーカス（動的・閾値駆動）・予定・返信待ち・進行中の案件・今後 N 日——短く、厳選、全行クリック可。 |
| `<日付>_完整事項.md` | **一件も省略しないアクション表**——全項目を重要度順に、各行が処理用のディープリンク。 |
| `<日付>_完整事項.csv` | 機械可読インデックス。一意識別子の列が翌日の継続性 diff の **JOIN キー**。 |

### 継続性（目玉機能）

成果物は日付付きの台帳として累積します。各実行は**直近の前回**ブリーフの CSV を読み、差分を取り——**✅ 解決済み**・**⏳ まだあなた待ち（N 日経過）**・**🆕 新規**を提示。継続性は各項目の ID を**ライブのプラットフォームで再検証**して根拠づけ、昨日のテキストを鵜呑みにしません（ハルシネーションの持ち越し防止）。

## 前提

このツールキットは**既に接続済みの MCP サーバーを利用**し、優雅に縮退します（独自登録はしません）。必要なサービスを接続してください。例：

- [`collab-toolkit`](../collab-toolkit/) — Slack / Asana / Notion
- [`gws-toolkit`](../gws-toolkit/) — Gmail / Google Calendar / Drive
- GitHub MCP サーバー（または `gh` CLI）

未接続のサービスはブリーフのカバレッジ宣言で「盲点」として明示され、残りは通常通り実行されます。

> ⚠️ Claude Code CLI 専用——基盤の MCP 取得は Cowork サンドボックスでは動作しません。

## 安全性

- 全工程 **read-only**（search / fetch / read）。何も変更・削除・送信しません。
- **draft-only**——指定したローカルフォルダにのみ書き込み。Notion / Slack / Gmail / Asana 等の公式システムへ**書き戻さず**、自動返信もしません。

## スケジューリング

スキルはオンデマンドです。**毎朝**動かすにはハーネス側のトリガーを重ねます——`/schedule`・cron・Cowork のスケジュールタスクで `/daily-brief` を起動。スケジューリングと配信は意図的にスキルの外です。

## 対象外

- 人事評価 / 自己評価 / 期間の振り返り → `performance-evidence-audit`（同じ横断機構だが過去向き・証拠志向）。
- 各プラットフォームへの書き戻し / 送信 / 自動操作 → 設計上の対象外。

## スキル

| スキル | コマンド | 目的 |
|---|---|---|
| `daily-brief` | `/daily-brief` | 今日の横断モーニングブリーフ + アクション表を生成。 |
