# 4DX D4 — Member Account Debrief（Session に綺麗に入る）

[English](README.md) | **日本語** | [繁體中文](README.zh-TW.md)

> あなた（チームメンバー）を coach し、WIG Session 前に先週の commitment を正直に自己覆盤し、Account segment で話す ~30 秒の陳述を準備する。

## このスキルが起動するとき

- 「先週のコミットメント果たせなかった、どう振り返る？」
- 「セッション明日で、何て言えばいい？」
- 「コミットメント部分達成、どう報告すればいい？」
- 「金曜遅れ + regression あり — kept か miss か？」
- session が憂鬱で、no-mention か excuse に走りたくなっている

## 何をするか（プロトコル概要）

Personal-coach 声であなたへ、locker-room ブリーフ、~5 分。成果: 30 秒 Account script — 恥じない、言い訳しない、直球で:

1. **開幕 + frame（≤30 sec）** — session script、therapy ではない
2. **Q1 — commitment を逐語で復元** — 言えなければそれ自体がデータ（上流 P-22 violation）
3. **Q2 — 一語で判定** — kept / partial / missed。P-22 二段テスト: 具体的成果物 + 約束した時間窓
4. **Q3 — miss / partial なら何が阻んだ？** — 事実ベース、責めない。excuse-mode から agency を回収。他人の urgent を拾ったなら Marcus pattern を名指し
5. **Q4 — 何を学んだか？** — commitment 設計 / 一週の capacity / lead-measure 機構（Crisafulli の規律: 勝った時も「なぜ勝ったか」を理解）
6. **Q5 — 再 commit か redesign か？** — 1 miss → 同 scope で再 / 2 misses → scope を縮 / 3+ → redesign（commitment-prep へ）
7. **30 秒 script を組む** — commitment → 結果 → 原因 → 学習 → 次。5 つの failure mode（excuse / shame / vague / no-mention / over-commit）で点検
8. **closing** — 言って座る。再 commit は Plan segment、Account ではない

## 使わない場面

| 状況 | 代わりの行き先 |
|---|---|
| solo user、team session 不在 | `4dx-sustain-personal-momentum-rescue` |
| 3 連続 miss（commitment-design 問題で debrief ではない） | `4dx-d4-member-commitment-prep` |
| 次の commitment を選んでいる | `4dx-d4-member-commitment-prep` |
| leader 側 facilitation / 受け取る側 | `4dx-d4-team-wig-session-lead` |
| therapy / shame 処理（「私はダメだ」） | 4DX の上流、対象外 |
