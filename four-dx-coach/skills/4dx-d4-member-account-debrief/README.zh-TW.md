# 4DX D4 — Member Account Debrief（乾乾淨淨進房間）

[English](README.md) | [日本語](README.ja.md) | **繁體中文**

> 帶你（團隊成員）在 WIG Session 前誠實自我覆盤上週 commitment，並備好 Account segment 要講的 ~30 秒陳述。

## 何時觸發這個 skill

- 「上週 commitment 沒達成，這次 session 怎麼面對？」
- 「半完成的承諾要怎麼講？」
- 「WIG Session 明天，我還沒做完，我要說什麼？」
- 「我週五才出貨而且還有 regression — 算 kept 還是 miss？」
- 你害怕 session 而且想跳過或找藉口

## 它做什麼（protocol 摘要）

Personal-coach 對你發聲，更衣室式短促，~5 分鐘。產出：30 秒 Account script — 不羞辱、不找藉口、直球：

1. **開場 + 對齊 frame（≤30 sec）** — 是 session script、不是 therapy
2. **Q1 — 一字不差重述 commitment** — 講不出 → 那本身就是資料（上游 P-22 violation）
3. **Q2 — 一個字判定** — kept / partial / missed。跑 P-22 兩段測試：是否具體交付物 + 是否在承諾的時間窗
4. **Q3 — miss / partial 的話，什麼擋住了？** — 事實，不是責怪；excuse-mode 把 agency 拉回；如果是替別人做 urgent → 點出 Marcus pattern
5. **Q4 — 我學到什麼？** — commitment design / 一週 capacity / lead-measure 機制（Crisafulli 紀律：贏的時候也要懂為什麼）
6. **Q5 — 再 commit 還是重設計？** — 1 miss → 同 scope 再來 / 2 misses → 縮小 / 3+ → 重設計（去 commitment-prep）
7. **組裝 30 秒 script** — commitment → 結果 → 原因 → 學習 → 下一步；過 5 個 failure mode（excuse / shame / vague / no-mention / over-commit）
8. **收尾** — 講完坐下。再 commit 是 Plan segment 的事，不是 Account 的事

## 不要在這些情境使用

| 情境 | 改去 |
|---|---|
| solo 用戶、沒 team session | `4dx-sustain-personal-momentum-rescue` |
| 連續 3 次 miss（commitment-design 問題，不是 debrief） | `4dx-d4-member-commitment-prep` |
| 在挑下次的 commitment | `4dx-d4-member-commitment-prep` |
| leader 主持 / 接收 account | `4dx-d4-team-wig-session-lead` |
| therapy / shame 處理（「我就是廢」） | 在 4DX 上游，不在範圍 |

## 出處

V1 ⚠️ **partial** — *The 4 Disciplines of Execution*（2nd ed., 2021）寫的是 *leader 側* accountability dialogue（Susan 對 Marcus）。書中最乾淨的 member POV 自我覆盤是 Marcus 自己那句（「I wasn't even able to keep my commitments…」）。Member 側 debrief protocol 是 leader 側 accountability-with-respect 的對稱反推，錨於 P-22（specific-deliverable test）、P-39（≥90% fulfillment）、CE-22/23/24（vague recognition / Marcus pattern / Goodhart 自我欺騙）自我套用。三個 anchor case：Marcus 自述（Ch 10）、Yolanda 倉儲區乾淨回報（Ch 11）、Mike Crisafulli 拒收沒搞懂機制的 win。
