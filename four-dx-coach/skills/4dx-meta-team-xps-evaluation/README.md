# 4DX Meta — Team XPS Evaluation

**English** | [日本語](README.ja.md) | [繁體中文](README.zh-TW.md)

> Audit a team's 4DX execution quality on the canonical XPS — four components × max 1.0 = max 4.0 — diagnose band, name the weakest component, set a single fix priority.

## When this skill activates

- "Audit our team's 4DX execution quality" / "Score our 4DX implementation"
- "Is our 4DX actually working?" / "Are we really doing 4DX or just pretending?"
- Quarter-end / mid-cycle health check after 6-12 weeks of operation
- Pre-mortem on a stalled WIG (audit before declaring the WIG wrong)
- External coach inheriting a team that "is doing 4DX" wants a baseline

## What it does (the protocol in brief)

Auditor / consultant voice — facts, scores, fix priority. No coaching, no encouragement:

1. **Establish the audit subject** — one team, WIG stated, weeks-since-launch known (XPS does not aggregate)
2. **Component 1 audit — Establishing a Cadence (0.0-1.0)** — sessions held / expected × meeting-discipline multiplier; book standard 100%
3. **Component 2 audit — Fulfilling High-Impact Commitments (0.0-1.0)** — fulfillment rate × specificity multiplier; book standard 90%+
4. **Component 3 audit — Optimizing Lead-Measures Performance (0.0-1.0)** — five optimizing questions + scoreboard health; book standard 90%+ weekly target
5. **Component 4 audit — Achieving Lag-Measure Results (0.0-1.0)** — actual progress / required progress, capped at 1.0
6. **Compute total + assign band** — 3.6-4.0 excellence / 3.2-3.59 good / 2.5-3.19 fair / 0-2.49 significant concern
7. **Identify fix priority (1-3 actions)** — lowest-scoring component first; tie-break C1 > C2 > C3 > C4
8. **Deliver the XPS report** — `Components → Total → Band → Weakest → Fix priority`

## When NOT to use

| Situation | Where to go instead |
|---|---|
| Solo / personal-coach mode | `4dx-sustain-personal-momentum-rescue` |
| Running this week's WIG session (not auditing past) | `4dx-d4-personal-wig-session` |
| "Should we use 4DX at all?" | `4dx-meta-team-strategy-triage` |
| Cross-team head-to-head ranking | XPS is intra-team self-diagnostic, not a leaderboard |
| User has no authority / intent to act on the result | Don't run audit-without-action |
