# 4DX D4 — WIG Session

[English](README.md) | [日本語](README.ja.md) | **繁體中文**

> D4 的核心 — 為單一使用者跑一場 20-30 分鐘的 weekly WIG Session。三段 Socratic agenda：Account → Review → Plan。agent 扮演 peer-witness。

## 何時觸發這個 skill

- 「想要每週固定 review 維持目標進度」「每週檢查 WIG 進度」
- 「目標每週都被推開，怎麼維持？」
- 「lead measure 有了但沒在執行」
- 「跟我跑一次 WIG Session」「帶我做這週的 check-in」

## 它做什麼（protocol 摘要）

每週一場、20-30 分鐘。agent 是 peer-witness（不是 boss、不是 coach、不是 status-checker）：

1. **frame & open（≤2 分）** — 把 WIG 一字一句講出來，宣告三段結構和 time-box
2. **Segment 1 — Account（~5-7 分）** — 上週各 commitment 分類成 kept / partially / not kept。沒做到只問一次原因，session 內不做 problem-solve
3. **Segment 2 — Review scoreboard（~5-7 分）** — 先讓使用者自己解讀 lead trend / lag trend。如果 lead 綠 + lag 平 ≥3 週 → flag P-50 並 route 到 sustain-rescue
4. **Segment 3 — Plan（~8-10 分）** — 問題要逐字精準：「**這一週**為了推動你的 **lead measure**，最重要的 **一到兩件事** 是？」。兩個 filter：具體可驗證 deliverable + 直接動到 lead。靠別人才能成立的 commitment 退回
5. **confirm & log（~1-2 分）** — 使用者一字不漏唸出 1-2 個 commitment，agent 複誦並記下供下次 Account 用
6. **close & schedule（~1 分）** — 下週同一時間。可以 cancel-and-reschedule，但不能 skip
7. **whirlwind 侵入時的 redirect（任何 segment）** — 禮貌但堅定打斷，守住議程

## 不要在這些情境使用

| 情境 | 改去 |
|---|---|
| 團隊 facilitation（manager 主持團隊會） | 這個 skill 是 solo 個人 scope；直接讀書中第 5 + 15 章 |
| daily standup / status meeting / 1:1 / project review | 別跟 WIG Session 的特定架構混為一談 |
| 還沒有 WIG | 先去 `4dx-d1-personal-wig-defining` |
| cadence 已經斷掉 2 週以上 | 先去 `4dx-sustain-personal-momentum-rescue`，再回來這裡 |
| 年度 / 季度 retrospective | cadence scope 不對 |
| reactive / on-call / 緊急應變的工作 | `4dx-meta-personal-strategy-triage`（CE-26） |
