# 4DX D4 — WIG Session

[English](README.md) | **日本語** | [繁體中文](README.zh-TW.md)

> D4 の中核 — 単独 user 向けに 20-30 分の weekly WIG Session を運用する。3 segment の Socratic agenda: Account → Review → Plan。agent は peer-witness。

## このスキルが起動するとき

- 「毎週の振り返りで進捗を保ちたい」「ゴールへの進捗を毎週チェックしたい」
- 「目標が毎週ずるずる遅れていく、どう維持する？」
- 「lead measure はあるけど行動に繋がっていない」
- 「WIG Session を一緒に回してほしい」「今週の check-in を案内して」

## 何をするか（プロトコル概要）

週 1 回 20-30 分の単発 session。agent は peer-witness（boss / coach / status-checker ではない）:

1. **frame & open（≤2 min）** — WIG を一字一句声に出し、3 segment と time-box を宣言
2. **Segment 1 — Account（~5-7 分）** — 前回 commitment を kept / partially / not kept に分類。理由は 1 度だけ聞く。session 内で problem-solve はしない
3. **Segment 2 — Review scoreboard（~5-7 分）** — lead trend / lag trend → user 自身に先に解釈させる。lead green + lag flat が 3 週以上なら P-50 を flag し sustain-rescue に route
4. **Segment 3 — Plan（~8-10 分）** — 質問は厳密に: 「来週、lead measure を動かすために最も重要な **1 つか 2 つ** は？」。2 つの filter: specific deliverable + 直接 lead を動かす。他人依存の commitment は却下
5. **confirm & log（~1-2 分）** — user に 1-2 commitment を一字一句述べてもらい、agent が echo して記録（次回 Account 用）
6. **close & schedule（~1 min）** — 来週同時刻。cancel-and-reschedule は OK、skip は不可
7. **whirlwind 侵入時の redirect（任意の segment）** — 礼儀正しく遮り agenda を守る

## 使わない場面

| 状況 | 代わりの行き先 |
|---|---|
| team facilitation（manager が team の会を回す） | このスキルは solo 個人 scope。書籍 5 + 15 章を直接参照 |
| daily standup / status meeting / 1:1 / project review | WIG Session の固有形式と混ぜない |
| まだ WIG がない | 先に `4dx-d1-personal-wig-defining` |
| cadence がすでに 2 週以上途切れている | 先に `4dx-sustain-personal-momentum-rescue`、その後ここに戻る |
| 年次 / 四半期の retrospective | cadence scope が違う |
| reactive / on-call / 緊急対応業務 | `4dx-meta-personal-strategy-triage`（CE-26） |
