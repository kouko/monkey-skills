# 4DX Meta — Team Strategy Triage

[English](README.md) | **日本語** | [繁體中文](README.zh-TW.md)

> 直属 3-12 人を持つ team leader のための pre-D1 gate — インストール前に「このチームに 4DX は本当に合うのか？」を判定する。

## このスキルが起動するとき

- 「私たちのチームに 4DX 合うか？」「うちのチームで 4DX 効くかな」
- 「チームに 4DX 導入すべき？」「うちの規模で 4DX は合う？」
- Leader が本を読み終わった / workshop を受けた直後で運用 cadence を選んでいる
- チームが OKR（または quarterly goals）で失速し、次に 4DX を検討している
- Leader の上司 / peer が 4DX を推している。yes / no / not-yet を根拠ベースで返したい

## 何をするか（プロトコル概要）

Agent は leader の consultant として、leader 個人ではなく「チーム」について質問する:

1. **チームと候補 goal を具体化** — 人数 + 1 文の headline outcome
2. **stroke-of-the-pen check** — leader 単独の意思決定で達成できるか？該当 → halt
3. **whirlwind-handleable check** — 既存業務をやり込めば届くか？該当 → halt
4. **team-shape screen** — 規模 3-12 / firefighting role でない / 時間権限あり / single-shot project でない
5. **WIG-shape check** — *From X to Y by When* で書けるか + mission/vision 整合（P-45 veto）
6. **adoption-curve readiness** — Models / Not Yets / Nevers 分布（F-28）
7. **matched-set commitment** — 四 discipline 全部 or 0（P-23）
8. **verdict** — TEAM-APPLICABLE → D1 へ、または 5 種類の redirect verdict のいずれか

## 使わない場面

| 状況 | 代わりの行き先 |
|---|---|
| solo / individual contributor の goal（チームなし） | `4dx-meta-personal-strategy-triage` |
| チームが導入済で D1 の開始方法を聞いている | `4dx-d1-team-primary-wig-selection` |
| Leader-of-Leaders による複数チーム rollout | 本 Ch 6-10 + `4dx-d1-team-wig-cascade` |
| 個別 discipline の質問（lead measure / scoreboard） | 該当する D-skill |
| 4DX 文脈なしの一般的なチーム戦略質問 | 他の意思決定 skill |
