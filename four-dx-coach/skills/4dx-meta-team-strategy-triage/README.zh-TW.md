# 4DX Meta — Team Strategy Triage

[English](README.md) | [日本語](README.ja.md) | **繁體中文**

> Team leader（直屬 3-12 人）的 pre-D1 gate — 在動手裝 4DX 之前先答「我們這個團隊到底適不適合 4DX？」。

## 何時觸發這個 skill

- 「我們團隊適合導入 4DX 嗎？」「團隊要不要用 4DX？」
- 「我這個團隊用 4DX 有用嗎？」「我們這種規模用 4DX 合適嗎？」
- Leader 看完書 / 上完 workshop，正在挑運作節奏
- 團隊試過 OKR（或季度目標）後熄火，下一個想試 4DX
- Leader 的老闆 / 同儕在推 4DX，leader 想要一個有依據的 yes / no / not yet

## 它做什麼（protocol 摘要）

Agent 扮演 leader 的 consultant，問題都針對「團隊」、不是 leader 個人：

1. **問清團隊與候選 goal** — 團隊人數 + 一句話 headline outcome
2. **stroke-of-the-pen check** — leader 一個決策就能達成？若是 → halt
3. **whirlwind-handleable check** — 把現有工作做好一點就能達成？若是 → halt
4. **team-shape screen** — 規模 3-12 / 不是 firefighting role / leader 有時間權限 / 不是 single-shot project
5. **WIG-shape check** — 能寫成 *From X to Y by When* + mission/vision 不違背（P-45 veto）
6. **adoption-curve readiness** — Models / Not Yets / Nevers 分佈（F-28）
7. **matched-set commitment** — 四個 discipline 全做或不做（P-23）
8. **verdict** — TEAM-APPLICABLE → 接 D1，或 5 種 redirect verdict 之一

## 不要在這些情境使用

| 情境 | 改去 |
|---|---|
| solo / individual-contributor 目標（沒有團隊） | `4dx-meta-personal-strategy-triage` |
| 團隊已決定用 4DX，問怎麼開始 D1 | `4dx-d1-team-primary-wig-selection` |
| Leader-of-Leaders 多團隊 rollout | 看書 Ch 6-10 + `4dx-d1-team-wig-cascade` |
| 單一 discipline 問題（lead measure / scoreboard） | 對應的 D-skill |
| 跟 4DX 無關的一般團隊策略問題 | 其他決策 skill |
