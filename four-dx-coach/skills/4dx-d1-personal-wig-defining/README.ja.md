# 4DX D1 — WIG Defining

[English](README.md) | **日本語** | [繁體中文](README.zh-TW.md)

> D1 の中核 — 漠然とした願望から、*From X to Y by When* 形式の WIG を 1 つ、書面で取り出す。

## このスキルが起動するとき

- 「目標がぼんやりしている」「優先順位が多すぎて決められない」
- 「やりたいことが多すぎる」
- 結果ではなく *activity* で語られている目標（「もっと運動」「毎日日本語を勉強」）
- 何ヶ月も「% 完了」で追跡されて停滞している project
- 4DX を使いたいが、まだ *From X to Y by When* 形式で goal を言葉にできていない

## 何をするか（プロトコル概要）

Socratic な coaching dialogue — agent が WIG を代筆せず、user から引き出す:

1. **goal を生のまま取り込む** — user 本人の言葉そのままを記録
2. **lag measure を表に出す** — 1 年後に測れる下流の数値（kg / revenue / chapters / clients）を 1 つ
3. **X（現在地）を確定** — 現時点の数値
4. **Y（目標地）を確定** — 同じ単位での目標数値
5. **When を確定** — 「もうすぐ」「今年中」ではなく、固定の暦日
6. **wildly-important テスト** — transformation か incremental か。incremental なら停止（それは whirlwind work で WIG ではない）
7. **count テスト** — 1 つに絞る（独立 & 非競合なら最大 2）
8. **battle-vs-war テスト** — war なら最小の sub-WIG 集合に分解し、各 sub-WIG で step 2-7 を再走
9. **project-as-WIG check** — project なら deliverables-list + date に書き直す（% complete は却下）
10. **WIG を出力** — 声に出して読み、第三者が締切日に勝ち負けを判定できるか user に確認させる

## 使わない場面

| 状況 | 代わりの行き先 |
|---|---|
| stroke-of-pen な意思決定（机を買う、CRM を切替） | 通常の意思決定 protocol |
| 純粋な習慣設計（毎日 meditate）— 固定の lag 終点がない | Atomic Habits / habit-stacking |
| すでに X / Y / When が明示された数値 goal | `4dx-d2-personal-lead-measure-discovery` に直接 |
| reactive / on-call で whirlwind 自体が戦略 | `4dx-meta-personal-strategy-triage` |
| そもそも 4DX が合うか不明 | まず `4dx-meta-personal-strategy-triage` |

## 出典

*The 4 Disciplines of Execution* 第 2 版（2021）第 2 章 Discipline 1: Focus on the Wildly Important + 第 6 章 + 第 12 章より蒸留。3 つの anchor case: NASA moon shot、Tim Cook の Apple（"say no to good ideas"）、Mountain Land Rehab（12 年停滞した training project が deliverables-list-with-deadline で復活）。
