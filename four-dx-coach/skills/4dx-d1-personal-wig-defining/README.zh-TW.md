# 4DX D1 — WIG Defining

[English](README.md) | [日本語](README.ja.md) | **繁體中文**

> D1 的核心 — 把使用者從模糊願望帶到一條寫好的 WIG，形式為 *From X to Y by When*。

## 何時觸發這個 skill

- 「目標太模糊」「想做的事太多」「不知道該專注哪個」
- 「我有太多優先事項」「幫我訂一個目標」
- 用 *activity* 描述而不是用結果（「多運動」「每天讀日文」）
- 一直用「完成度 %」追蹤、卡了好幾個月的 project
- 想用 4DX 但還沒把 goal 講成 *From X to Y by When*

## 它做什麼（protocol 摘要）

Socratic coaching dialogue — agent 不替使用者寫 WIG，要從他口中問出來：

1. **抓原始 goal** — 把使用者自己的話原文記下來
2. **逼出 lag measure** — 一年後能量化的下游數字（kg / revenue / 章節數 / 客戶數）
3. **釘住 X（現況）** — 今天的數值
4. **釘住 Y（目標）** — 同單位的目標值
5. **釘住 When** — 固定行事曆日期，不是「快了」「今年內」
6. **wildly-important 測試** — 是 transformation 還是 incremental？incremental → 停（那是 whirlwind work，不是 WIG）
7. **count 測試** — 收斂到 1 個（獨立且不互衝才能最多 2 個）
8. **battle-vs-war 測試** — 是 war 就拆成最小 sub-WIG 集合，每個 sub-WIG 再跑 step 2-7
9. **project-as-WIG check** — project 就改寫成 deliverables-list + date（拒絕 % complete）
10. **輸出 WIG** — 唸出聲，讓使用者確認陌生人能在截止日判定勝負

## 不要在這些情境使用

| 情境 | 改去 |
|---|---|
| stroke-of-pen 決策（買桌子、換 CRM） | 一般決策 protocol |
| 純習慣設計（每天 meditate）— 沒有固定的 lag 終點 | Atomic Habits / habit-stacking |
| X / Y / When 都已寫清楚的數值 goal | 直接去 `4dx-d2-personal-lead-measure-discovery` |
| reactive / on-call 領域，whirlwind 本身就是戰略 | `4dx-meta-personal-strategy-triage` |
| 連 4DX 適不適合都還沒確認 | 先去 `4dx-meta-personal-strategy-triage` |

蒸餾自 *The 4 Disciplines of Execution*（2nd ed., 2021）第 2、6、12 章。三個 anchor case：NASA moon shot、Tim Cook 的 Apple（"say no to good ideas"）、Mountain Land Rehab（卡了 12 年的 training project 改成 deliverables-list-with-deadline 後出貨）。
