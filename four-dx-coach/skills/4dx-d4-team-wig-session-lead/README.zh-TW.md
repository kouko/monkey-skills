# 4DX D4 — Team WIG Session（Leader 主持）

[English](README.md) | [日本語](README.ja.md) | **繁體中文**

> 帶你（leader）主持 20-30 分鐘的 weekly WIG Session — Account → Review → Plan — 成員自己提 commitment、你 veto 但不 dictate。

## 何時觸發這個 skill

- 「帶我們團隊的 weekly WIG Session」「協助我主持團隊 WIG 會議」
- D1-D3 已裝（Team WIG + lead measure + scoreboard），第一次 session 在即
- WIG Session 已退化成 status meeting — 沒 specific commitment、沒進度
- 有成員沒守住 commitment，你不知道怎麼回應才不羞辱人又不放掉標準
- 你出差中，想跳過這次 session

## 它做什麼（protocol 摘要）

Consultant 對 leader。Agent 在你耳邊耳語；房間是你在帶：

1. **會前 check（≤2 min）** — scoreboard 更新、上週 commitment 可視、時間 box 設好、WIG 一字不差
2. **開場（≤2 min）** — WIG 一字不差念出來，說明三個 segment
3. **Segment 1 — Account，leader 先報，再輪流（~7-10 min）** — 你先示範；有人沒守住 → 跑 Respect → Accountability → Encourage 三步
4. **Segment 2 — Review the scoreboard（~5-7 min）** — 看趨勢，先問團隊怎麼解讀，點名肯定 win
5. **Segment 3 — Plan，成員提案、leader 把關（~8-10 min）** — 跑四部分 specificity gate；veto 不 dictate；每人 cap 1-2 個
6. **大聲確認、紀錄（~2 min）** — 每人把最終 commitment 一字不差再講一次
7. **收尾、確認下次（~1 min）** — 同時間同地點；出差就 dial in 或委託代主持，絕不跳過

## 不要在這些情境使用

| 情境 | 改去 |
|---|---|
| solo 用戶、沒有 peer 團隊 | `4dx-d4-personal-wig-session` |
| 團隊成員要準備自己這週的 commitment | `4dx-d4-member-commitment-prep` |
| 團隊成員會前要處理上週結果 | `4dx-d4-member-account-debrief` |
| 還在做 goal-discovery / WIG selection | D1 team skill |
| 日 standup / sprint / 1:1 / 季度 review | 不同產物 — WIG Session 有三段固定結構 |
