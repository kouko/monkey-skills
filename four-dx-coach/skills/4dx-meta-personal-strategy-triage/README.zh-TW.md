# 4DX Meta-Strategy Triage

[English](README.md) | [日本語](README.ja.md) | **繁體中文**

> 在安裝四項 discipline *之前*，先判定 4DX 是不是真的適合這個目標。

## 何時觸發這個 skill

- 「4DX 適合我這個目標嗎？」「我這個目標可以用 4DX 嗎？」
- 剛讀完 / 聽過 4DX，想先確認套不套得上自己的情境
- OKR / GTD / atomic habits / 純意志力都試過卡住了，想把 4DX 當成下一個方法
- 懷疑自己把 4DX 過度套用到習慣、創作型 project 或 on-call role 上
- D1 之前的 gate — personal-coach pipeline 在進入 WIG-defining 前先過這一關

## 它做什麼（protocol 摘要）

Socratic 七步 triage，回傳 6 類 verdict 之一：

1. **把 goal 具體化** — 講到夠明確、能歸類為止
2. **stroke-of-the-pen check** — 單一決策 / 購買 / 雇用就能達成 → 停止
3. **whirlwind-handleable check** — 現有 routine 做得更精準就行 → 停止
4. **個人尺度 anti-pattern screen** — habit / portfolio-bet / emergency role / creative output
5. **time-sovereignty check** — 確認至少 ~20% 的週時間是自己能掌控的
6. **matched-set capacity check** — 四項 discipline 全做或全不做（不能挑食）
7. **verdict + handoff** — `APPLICABLE` 才進 D1，其餘出 redirect verdict

可能的 verdict：`APPLICABLE` / `stroke-of-the-pen` / `whirlwind-handleable` / `habit-formation` / `portfolio-bet` / `emergency-role` / `creative-output` / `no-time-sovereignty`。

## 不要在這些情境使用

| 情境 | 改去 |
|---|---|
| 已經決定要用 4DX、在問「怎麼開始？」 | `4dx-d1-personal-whirlwind-triage` → `4dx-d1-personal-wig-defining` |
| 在問某個特定 discipline | 對應的 D-skill |
| Enterprise rollout / 跨部門 cascading WIGs | 直接讀原書第 6-10 章（Leader-of-Leaders） |
| 沒有 4DX 字彙的一般決策問題 | 其他決策方法類 skill |
