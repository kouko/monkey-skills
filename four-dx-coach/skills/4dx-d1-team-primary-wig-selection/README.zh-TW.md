# 4DX D1 — Team Primary WIG Selection

[English](README.md) | [日本語](README.ja.md) | **繁體中文**

> 帶你（leader）用「Importance × Feasibility」2x2 從候選 Battles 中收斂出團隊的 Primary WIG（上限 ≤2 個 Primary、≤3 個 team WIG）。

## 何時觸發這個 skill

- 「幫我們團隊選 Primary WIG」「團隊今年該專注哪一個」
- 「我們有太多戰略目標想做」— 5+ 個候選收不下來
- 目前的 Primary WIG 太廣（「總收入」），跑不出 traction
- 在 Approach A（單一 WIG）vs Approach B（跨 segment 雙 WIG）之間猶豫
- 領導團隊可以排 importance，但漏掉 feasibility 軸

## 它做什麼（protocol 摘要）

Agent 跟 leader 的 consultant dialogue — facilitator，不替你選 WIG：

1. **釐清團隊 mission 與策略 context** — 並確認是 selection 還是 cascade（cascade 就 route 出去）
2. **brainstorm 5-10 個候選 Battle** — 廣度優先，先不過濾
3. **每個候選打 Importance（1-5）** — 突破會「轉化」團隊嗎？
4. **每個候選打 Feasibility（1-5）** — 1-2 年內團隊靠自己可以動嗎？
5. **畫 2x2 文字 grid** — 視覺化位置
6. **篩到右上角** — 選 ripple effect 最大的（Covenant Transport pattern）
7. **Approach A vs B 決策** — 預設 A；兩個真正獨立的營運段才用 B
8. **四陷阱檢查** — 太多 / 太廣 / 不可量化 / 偏離 mission
9. **寫成 *From X to Y by When*** — 通過 stranger test
10. **輸出 WIG + parked Battles 清單** — 明確宣告 Approach

## 不要在這些情境使用

| 情境 | 改去 |
|---|---|
| 個人 / 單獨目標 | `4dx-d1-personal-wig-defining` |
| 組織 WIG 已定，要切自家份 | `4dx-d1-team-wig-cascade` |
| 團隊成員想理解 leader 為何挑這個 WIG | `4dx-d1-member-team-wig-comprehension` |
| 還沒定義 mission 的 pre-strategy 階段 | 先定 mission |
| reactive / on-call 領域（whirlwind 就是戰略） | `4dx-meta-team-strategy-triage` |
