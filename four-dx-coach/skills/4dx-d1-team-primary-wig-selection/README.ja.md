# 4DX D1 — Team Primary WIG Selection

[English](README.md) | **日本語** | [繁體中文](README.zh-TW.md)

> あなた（leader）を coach し、Importance × Feasibility の 2x2 で候補 Battle を篩い、チームの Primary WIG を 1 つに収斂させる（上限 ≤2 Primary / ≤3 team WIG）。

## このスキルが起動するとき

- 「チームの Primary WIG をどう選ぶ？」「今年チームはどこに集中すべき？」
- 「戦略候補が多すぎてチームの焦点が決まらない」 — 5+ 候補が収束しない
- 現在の Primary WIG が広すぎて（「総売上」）traction が出ていない
- Approach A（単一 WIG）と Approach B（segment 別の 2 WIG）で迷っている
- 領導チームが importance のランキングはできるが feasibility 軸を飛ばす

## 何をするか（プロトコル概要）

Agent は leader の consultant — facilitator として、WIG を代わりに選ばない:

1. **チーム mission と戦略 context を明示** — selection か cascade か確認（cascade なら route）
2. **5-10 個の候補 Battle を brainstorm** — 広く、先に絞らない
3. **各候補に Importance（1-5）** — 突破でチームが「変容」するか？
4. **各候補に Feasibility（1-5）** — 1-2 年でチーム自身の行動変容で動かせるか？
5. **2x2 を text grid で描画** — 視覚配置
6. **右上に絞る** — ripple effect が最大のものを選ぶ（Covenant Transport pattern）
7. **Approach A vs B 決定** — 既定 A、真に独立した 2 segment のみ B
8. **4 つの罠 checklist** — too many / too broad / unmeasurable / mission-disconnected
9. **From X to Y by When で表現** — stranger test を通す
10. **WIG + parked Battles list を出力** — Approach を明示宣言

## 使わない場面

| 状況 | 代わりの行き先 |
|---|---|
| solo / 個人目標 | `4dx-d1-personal-wig-defining` |
| 組織 WIG 既定、自チーム分の切り出し | `4dx-d1-team-wig-cascade` |
| Member が leader の選択理由を理解したい | `4dx-d1-member-team-wig-comprehension` |
| mission 未定義の pre-strategy 段階 | 先に mission を定義 |
| reactive / on-call 領域（whirlwind が戦略） | `4dx-meta-team-strategy-triage` |
