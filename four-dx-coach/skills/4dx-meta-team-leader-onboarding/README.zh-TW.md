# 4DX Meta — Team Leader Onboarding

[English](README.md) | [日本語](README.ja.md) | **繁體中文**

> 帶你（leader-of-leaders）從下屬 team-leader 那裡贏得真心 commitment（不是 compliance）。**兩個 mode**：**coach-mode**（啟動前準備 buy-in 對話）與 **audit-mode**（過去 onboarding 產生 compliance / 反彈時，從 rollout 信件 + 1:1 紀錄 + 團隊回饋診斷）。

## 何時觸發這個 skill

**coach-mode**（準備階段 — 還沒 rollout）：
- 「我要怎麼讓我的下屬 leader 們真心接受 4DX？」
- 「我的直屬主管會抵抗，要怎麼導入」
- 「不想被當成又一個流行 KPI 工具」
- 「要全面導入還是先做 pilot？」「〇〇 一定會反對，我要怎麼跟他談」
- 之前 OKR / scorecards 在 leader 層熄火，現在想避免重蹈覆轍

**audit-mode**（診斷階段 — 過去 onboarding 的 artifact 已在手）：
- 「導入了 4DX 但團隊不買單，幫我診斷」
- 「下屬 leader 表面配合心裡抗拒，問題在哪？」
- 「rollout 信件加上幾週的 leader 回饋，幫我看哪裡走樣」

## 它做什麼（protocol 摘要）

**coach-mode** — 對單一 leader-of-leaders 的 consultant。產出是 per-leader plan + 一段排演過的對話，不是 rollout 文件：

1. **確認前置條件** — 已決定要用 4DX + 已起草 Primary WIG / Key Battles
2. **點名 leaders 並標 skepticism level** — 3-5 位（enthusiast / neutral / skeptic / hard-no）
3. **每位 leader：他們想要什麼？** — 用「他們的」聲音，不是你的
4. **steel-man 每位 leader 最強的反對理由** — 完整句子，不是標籤；先不要反駁
5. **Pilot vs hold-back 分類** — voluntary pilot / hold back / engage but don't push
6. **針對 ONE leader 排演對話** — 槓桿最大的那位，用 user 自己的話講出聲
7. **輸出 per-leader plan + 排演腳本** — 這週就能用

**audit-mode** — consultant-from-artifacts。讀 rollout 信件 / kickoff slides / 1:1 紀錄 / 團隊回饋；跑 4 lens（commitment-vs-compliance · Kotter Stage-1 urgency · per-leader steel-manning · pilot-vs-hold-back）；把 critique mapping 到違反的 rule；輸出 per-leader recovery plan + 最高槓桿 missed leader 的對話 revision + onward route。

## 不要在這些情境使用

| 情境 | 改去 |
|---|---|
| 還沒決定 4DX 適合組織 | 先 `4dx-meta-strategy-triage` |
| Buy-in 已搞定，要做技術性 cascade | `4dx-d1-wig-cascade` |
| frontline 成員的 buy-in | 走 D1 Team WIG 選擇流程，不是第 8 章 |
| 跟 4DX 無關的一般 change management | Kotter / ADKAR / Bridges |
| Onboarding 後 cadence 連續多週中斷 | `4dx-sustain-momentum-rescue` |
| 跨 layer audit（WIG + leads + scoreboard + cadence） | `4dx-audit` |
| 你其實只想要 compliance，不是 commitment | 停 — 這個 skill 沒辦法幫你 |
