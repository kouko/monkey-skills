# 4DX Meta — Team Leader Onboarding

[English](README.md) | **日本語** | [繁體中文](README.zh-TW.md)

> あなた（leader-of-leaders）を coach し、4DX 立ち上げ前に直属 team-leader から compliance ではなく genuine commitment を引き出す buy-in 対話を準備する。

## このスキルが起動するとき

- 「部下リーダーに 4DX を腹落ちさせたい」
- 「直属マネージャーが反発しそう、どう導入する」
- 「また流行りの施策と思われたくない」
- 「全社展開かパイロットか」「〇〇さんが反対しそう、どう話す」
- 以前 OKR / scorecards が leader 層で失速した経験があり、繰り返したくない

## 何をするか（プロトコル概要）

単一の leader-of-leaders に対する consultant。成果は per-leader plan + リハーサル対話 1 本（rollout 文書ではない）:

1. **前提確認** — 4DX 適合済 + Primary WIG / Key Battles のドラフト保有
2. **leaders を名指し + skepticism レベル付け** — 3-5 名（enthusiast / neutral / skeptic / hard-no）
3. **各 leader: 彼らは何を望んでいる？** — あなたの声ではなく「彼らの」声で
4. **各 leader の最強反対意見を steel-man** — ラベルではなく完全文。反論はまだしない
5. **Pilot vs hold-back 分類** — voluntary pilot / hold back / engage but don't push
6. **ONE leader 用に対話をリハーサル** — 最高 leverage の相手、user 自身の言葉で口に出す
7. **per-leader plan + リハーサル script を出力** — 今週から動ける

## 使わない場面

| 状況 | 代わりの行き先 |
|---|---|
| 組織への 4DX 適合判断が未了 | まず `4dx-meta-team-strategy-triage` |
| Buy-in 確保済、技術 cascade が次 | `4dx-d1-team-wig-cascade` |
| frontline メンバーの buy-in | D1 Team WIG 選択プロセス、Ch 8 ではない |
| 4DX 非依存の一般 change management | Kotter / ADKAR / Bridges |
| 実は commitment ではなく compliance が欲しい | 停止 — この skill では助けにならない |
