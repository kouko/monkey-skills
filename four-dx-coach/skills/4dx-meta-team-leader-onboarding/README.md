# 4DX Meta — Team Leader Onboarding

**English** | [日本語](README.ja.md) | [繁體中文](README.zh-TW.md)

> Coach you (the leader-of-leaders) through preparing the buy-in conversation that earns genuine commitment — not compliance — from your direct-report team-leaders before launching 4DX.

## When this skill activates

- "How do I get my team-leaders on board with 4DX?"
- "My direct reports will resist this — how do I roll it out?"
- "How do I avoid this looking like another corporate flavor-of-the-month?"
- "Should I pilot or mandate?" / "Leader X will hate this — what do I say?"
- Failed-prior-rollout rebound (OKRs / scorecards fizzled at the leader level)

## What it does (the protocol in brief)

Consultant-to-single-leader-of-leaders. Output is a per-leader plan + one rehearsed conversation, not a rollout doc:

1. **Confirm precondition** — 4DX fit decided + draft Primary WIG / Key Battles in hand
2. **Name the leaders, with skepticism level** — 3-5 named (enthusiast / neutral / skeptic / hard-no)
3. **For each leader: what do THEY want?** — outcome in their voice, not yours
4. **Steel-man each leader's strongest objection** — full sentences, not labels; no counter yet
5. **Pilot vs hold-back classification** — voluntary pilot / hold back / engage but don't push
6. **Rehearse the conversation for ONE leader** — highest-leverage case, said aloud in user's own words
7. **Output a per-leader plan + rehearsed script** — actionable this week

## When NOT to use

| Situation | Where to go instead |
|---|---|
| Haven't decided 4DX fits the org | `4dx-meta-strategy-triage` first |
| Buy-in already secured; need the technical cascade | `4dx-d1-team-wig-cascade` |
| Frontline-team-member buy-in | Use D1 Team WIG-choosing process, not Ch 8 |
| Generic change management not 4DX-specific | Kotter / ADKAR / Bridges |
| You actually want compliance, not commitment | Stop — this skill won't help |
