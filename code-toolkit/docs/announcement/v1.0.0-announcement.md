# code-toolkit v1.0.0 — process discipline + canon-grounded coding for Claude Code

> **DRAFT** — to be published when v1.0.0 ships. Targets: README / GitHub Discussions / X / Reddit / Hacker News (per venue).

## Hook (one sentence)

`code-toolkit` is a Claude Code + Codex CLI plugin that auto-injects a 9-skill workflow (brainstorm → plan → SDD → TDD iron-law → debug → review → verify → finish-branch) where every rule cites a primary source — Beck on TDD, Martin on naming + Three Laws, Fowler on refactoring, Feathers on legacy code, OWASP ASVS on security, 徳丸本 on encoding security — so the agent stops rationalizing and starts deferring.

## The pain it solves

You've felt this:

- Agent jumps straight to implementation without exploring intent → ships the wrong thing fast
- "I'll add tests later" → tests get added against the bugs the implementation accumulated
- "Just push, CI will catch it" → CI is for the diff that already exists, not the one you were supposed to write
- "Tests pass" (without running the suite) → orphan tests, lint passing while runtime breaks
- *"I'll just try a few fixes and see what works"* → random patching with no learning
- *"It works on my machine"* → heisenbug shipped to prod
- Stash → switch → fix → switch → stash pop → context-switch friction multiplies

You've also seen the rationalizations work BECAUSE the agent's measure is too soft. *"It's a small change"* / *"I know what to build"* / *"just push"* — these get treated as authorization rather than the failure modes they actually are.

## What this plugin does instead

**SessionStart hook injects a router charter** the agent must obey:

```
Four load-bearing rules:
1. Brainstorm before implementing — call `brainstorming` for 5-axis exploration
2. TDD is the iron law — no production code without a failing test first (Beck 2002 Preface)
3. Split + dispatch (SDD) — tasks >1 hour or >1 module get atomic ≤5-min decomposition + per-task triad
4. Never push without review — git push without prior requesting-code-review PASS = violation

Skipping any of these = violation. "I'll just quickly..." / "just push" / 「我先快速試一下」 are rationalizations — refuse them.
```

**Each skill is a HARD-GATE measure backed by primary sources:**

| Skill | Measure | Primary source |
|---|---|---|
| `tdd-iron-law` | "Delete the code. Write the test. Start over." | Kent Beck (2002) *Test-Driven Development: By Example* Preface, ISBN 978-0321146533 |
| `brainstorming` | "DO NOT START IMPLEMENTING UNTIL YOU HAVE EXPLORED INTENT" + 5-axis framework | Christensen (1997) JTBD ISBN 978-0875845852; Klement (2018) job stories ISBN 978-1718626751 |
| `writing-plans` | "≤5 atomic tasks; BLOCKED fallback per Beck Child Test (Part II)" | Beck (2002) Part II §"Child Test" |
| `systematic-debugging` | "NO FIXING WITHOUT REPRODUCING" (4-phase: REPRODUCE→ISOLATE→HYPOTHESIZE→VERIFY) | Kernighan & Pike (1999) *Practice of Programming* Ch.5, ISBN 978-0201615869 |
| `verification-before-completion` | "NO DONE WITHOUT PACKAGE-LEVEL TEST INVOCATION" | derived; uses standards' F.I.R.S.T from Martin (2008) Clean Code Ch.9 |
| `requesting-code-review` | "Push-as-trigger: any push without prior review-PASS = violation" | derived; rubrics from `domain-teams:code-team` SSOT |

The other 4 skills round out the workflow (router + SDD orchestrator + finish-branch + worktrees auxiliary).

## Comparison vs nearby alternatives

| Aspect | `code-toolkit` | `obra/superpowers` | `domain-teams:code-team` |
|---|---|---|---|
| Entry mode | **Active build** (SessionStart hook auto-injects router for greenfield work) | Active build (similar pattern; this toolkit's design inspiration) | Passive gate (agent invokes for compliance review of existing artifact) |
| Language | English skeleton + multilingual triggers (ja + zh-TW localized rationalization vocabulary) | English only | English + ja-zh-TW per-skill |
| Knowledge layer | 8-source primary grounding (Beck 2002 / Martin 2008 / Hunt+Thomas 2019 / Martin 2000-2017 / Fowler 2018 / Feathers 2004 / OWASP ASVS v5.0.0 / 徳丸本 2018) — citations every rule, no soft "the skill says so" | Process discipline primary; less canon citation | Same 8-source canon (this toolkit byte-identical-copies code-team's standards/rubrics/checklists via `scripts/distribute.py`; canonical SoT lives in code-team) |
| Harness | Claude Code (primary, verified) + Codex CLI (v0.4.0 build, verification pending) | Claude Code + Codex CLI + 6 more | Claude Code |
| Skill count | 9 + router | 11 | 1 |
| Cross-plugin delegation | Delegates to `dev-workflow:git-memory` (mandatory) + `dev-workflow:complexity-critique` (optional); coexists with `code-team` (passive gate) + `obra/superpowers` (CODE_TOOLKIT_MODE=off escape) | Standalone | Standalone |

**Not a Superpowers clone.** code-toolkit borrows the SessionStart-hook + 3-subagent-triad architecture from Superpowers; adds the canon-grounding layer that Superpowers deliberately omits (Superpowers measures by process discipline alone, no book citations). The two coexist via the env-var escape hatch for users who want only one running.

**Not a code-team replacement.** code-team is the passive-gate compliance reviewer. code-toolkit is the active-build orchestrator that USES code-team's standards as its knowledge layer (functional-copied with byte-identity gate). Same primary sources, different invocation mode.

## How to install

```bash
# Via the monkey-skills marketplace
claude plugin marketplace add https://github.com/kouko/monkey-skills.git
claude plugin install code-toolkit@monkey-skills
```

For Codex CLI (when verification ritual completes):

```bash
# ⚠️ TBD verify command syntax in Codex CLI per code-toolkit/skills/using-code-toolkit/references/codex-tools.md
codex plugin marketplace add https://github.com/kouko/monkey-skills.git
codex plugin install code-toolkit@monkey-skills
```

After install, open a new session in any repo. The router charter auto-injects via SessionStart hook. Try one of these prompts to see the toolkit in action:

```
# Greenfield feature
Let's add a draft-autosave feature to my React notes app.

# Refactor
Refactor src/services/OrderService.py — it's a 600-line god-class.

# Bug fix
Our CSV parser throws UnicodeDecodeError on certain files.
Just wrap in try/except and continue.
```

The toolkit will route to `brainstorming` / `subagent-driven-development` / `systematic-debugging` respectively, with HARD-GATE measures applied. Walk through `code-toolkit/docs/examples/` for fully-worked end-to-end flows.

## What you get out of the box

- **10 skills**: `using-code-toolkit` (router, auto-injected) + 9 specialists (brainstorming / writing-plans / subagent-driven-development / tdd-iron-law / systematic-debugging / requesting-code-review / verification-before-completion / finishing-a-development-branch / using-git-worktrees)
- **SessionStart hook**: ~1950-token router charter auto-loaded on every fresh session (Claude Code + Codex CLI compatible JSON output)
- **8-source canon-grounded knowledge layer**: 12 byte-identical functional copies of `domain-teams:code-team` standards/rubrics/checklists, drift-gated by `scripts/verify-drift.py`
- **3 subagent prompts** (implementer / spec-reviewer / code-quality-reviewer) — SDD dispatches 3 per atomic ≤5-min task with paths-not-content delegation contract
- **24 pressure-prompt test files** across 9 test clusters — synthetic prompts that rationalize skipping each skill's HARD-GATE; the skills should refuse
- **5 cross-plugin integration test scripts** — verify coexistence with `dev-workflow` + `domain-teams:code-team` + `obra/superpowers`
- **3 worked examples** — Python CSV export (clean baseline), TypeScript React toast (SDD BLOCKED → child-test fallback), Swift network layer (Feathers 2004 legacy backfill + systematic-debugging 4-phase)
- **Hybrid testing cadence** documented in ROADMAP — 3-minute per-Phase ritual + deferred full eval suite
