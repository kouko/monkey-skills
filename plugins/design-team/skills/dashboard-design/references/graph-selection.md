# グラフの種類と選び方

ダッシュボードの目的と照らし合わせ、伝える情報に対して適切なグラフを選ぶ。

## 選択の判断基準

| 伝えたい情報 | 推奨グラフ | 値の表現 |
|----------|---------|--------|
| 時間変化、傾向 | 折れ線グラフ | 位置 |
| 数量比較 | 棒グラフ | 長さ |
| 時間変化 + 構成比 | 面グラフ | 位置、面積 |
| 構成比（総量が明らか） | 円グラフ | 角度、面積 |

## 折れ線グラフ（ラインチャート）

縦軸に数量、横軸に時間をとり、データ点の間を線で結ぶ（補間する）ことで、**数量の時間変化とその傾向**を伝える。

### 構成要素
- **数量軸**（縦軸）：数量を表す
- **時間軸**（横軸）：時間である必要がある。時間以外の変数なら棒グラフを検討
- **データ点**：丸や四角などの図形を配置し、補間の有無が分かるように
- **系列名**：複数あるときは系列名を表示し、できる限り折れ線の近傍に配置

### 使用例
- **二つの系列**：系列間の比較
- **参照値との比較**：基準線や前年比との対比
- **目標値との比較**：実績値と目標値を線種（実線 vs 点線）で区別

### 判断ポイント
- 横軸が時間でない場合 → 棒グラフを検討
- 系列が多すぎる場合 → 色数が増え判別困難になるため、系列を絞るか別グラフに分割

## 棒グラフ（バーチャート）

一つの軸に数量、もう一つの軸に分類をとり、配置する棒の長さで**数量を比較する**。

### 構成要素
- **数量軸**：数量を表す
- **分類軸**：カテゴリを表す
- **棒の間隔**：隙間を空ける（棒同士が密着しない）

### バリエーション
- **単独棒グラフ**：一つの系列の数量比較
- **集合棒グラフ**：分類毎に複数の棒を並べて比較
- **積み上げ棒グラフ**：棒を積み上げて構成比を表現

### 判断ポイント
- 分類毎の数量比較 → まず棒グラフを検討
- 時刻毎の数量比較 → 折れ線グラフや面グラフが適している

## 面グラフ（エリアチャート）

折れ線グラフに塗りを加えることで、**数量と構成比の時間変化を同時に伝える**。

### バリエーション
- **積み上げ面グラフ**：複数の系列を面として積み上げ、総量の変化と各系列の内訳の推移を同時に表示
- **100%積み上げ面グラフ**：値自体を構成比とし、構成比の時間変化のみに着目

### 判断ポイント
- 構成比を伝える必要がないとき → 折れ線グラフを使用
- 系列の数が多い場合や、時間変化を伝えない場合 → 棒グラフが適している

## 円グラフ（パイチャート）

円を全体の総量とし、分類に対応する数量と弧の長さ（または角度および面積）を対応づけることで、**数量の構成比**を伝える。

### バリエーション
- **円グラフ**：基本形。できる限り数値を併記する
- **ドーナツグラフ**（ドーナツチャート）：円の中心をくり抜き情報を書き込む
  - **数量のドーナツグラフ**：中心に全体の総量を表示
  - **比率のドーナツグラフ**：中心に全体の総量、外周に比率を表示

### 判断ポイント
- 円グラフが適するのは、全体の総量が明らかであり、構成比をコンパクトに伝える場合
- そうでない多くの場合 → 棒グラフの方がデータを正確に伝えられる
- 全体の総量が時間変化をともなう場合 → 面グラフが適している

## 指標（数値テキスト）

グラフではないが、ダッシュボードの重要な構成要素。人々の判断や行動を左右する情報を端的に表す値。

- 単一の尺度を表す
- 全体傾向を表す
- 複数の尺度を合成した値を表す

指標は一般に、データを正確に反映しつつ、見る人にとって有益な情報を得られるような指標化が重要。

## 表

値に関連する情報を行と列へ対応づけ、値や値の集合を格子状に並べることで、データを視覚的に整理して伝える。表の中の値は、数値や文字に限らず、色や図形によって表現することもある。
