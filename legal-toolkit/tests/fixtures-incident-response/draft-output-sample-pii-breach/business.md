# PII Breach 事件 — 業務摘要

## §1 事件 1-句話

2026-05-10 偵測異常存取，2026-05-11 確認個資外洩，需即時通報 PDPC + 通知當事人。

## §2 Top 3 即時動作

1. 即時封堵異常存取（IT 主導）+ 證據保存（log / DB snapshot）
2. PDPC 通報文 + 當事人通知文 並行起草、走核稿流程
3. 法務評估通報範圍 + 確認當事人數精確盤點

## §3 deadline 警示

🔴 高度警示：個資法施行細則 §22 要求即時通報，且通知當事人不得延遲。

- PDPC 通報：施行細則 §22「即時」（具體小時數見子法 TBD_PDPC_timeframe）
- 當事人通知：個資法 §12 無具體時限但應即時

## §4 後續流程

- 通報後 24-48h 內準備 incident response 完整報告（§27 安全維護義務）
- 客戶服務窗口準備 FAQ + 應對話術
- 內部檢討會議排程 + Lesson learned 文件化
