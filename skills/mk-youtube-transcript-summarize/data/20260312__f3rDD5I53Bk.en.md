## Content Summary

#### Agency Agents 是什麼

- 開源 GitHub repo，含約 120 個 Markdown 格式的 AI 人格檔案
  - 12 天內累積 30,000 stars
  - 涵蓋行銷、工程、銷售、設計、QA、遊戲開發、空間運算等領域
- 重要澄清：這些不是自主 AI 代理人，而是結構化的系統提示（system prompts）
  - 不會自行使用工具、沒有持久記憶、不會做自主決策
  - 本質上是「給 AI 的詳細職位描述」

#### 內部結構解析

- 以 marketing growth hacker 為例，每個檔案包含：
  - YAML frontmatter（名稱、描述、工具、顏色、emoji）
  - 身份層（名字、個性）
  - 核心能力、專業技能、決策框架、成功指標

#### 實測比較：通用提示 vs. 人格化提示

- 測試任務：為 B2B SaaS 專案管理工具制定內容行銷策略
- 通用版：生成執行摘要、買家人格、內容支柱、季度路線圖等標準內容
- 人格化版：多層 SEO 關鍵字策略、12 個月內容行事曆、Lead Generation 優化、30/60/90 天行動計畫
- Claude 自評結果：人格化版以 68% vs 32% 獲勝，加權分數 8.65 vs 6.05（提升約 60-70%）

#### 安裝設置

- 前往 GitHub repo，可直接下載 ZIP 或 clone
- 建議 fork 後再 clone（方便推回自訂修改）
- 在 VS Code + Claude Code extension 的終端執行 git clone
- 執行設置指令，Agency Agents 檔案會被複製到 ~/.claude/agents/ 路徑

#### 與 Claude Code 整合：轉換為 Subagents

- 使用方式：在提示中指定人格名稱，或直接 @ 對應的 .md 檔案
- 平行 subagent 示範：以「建立 Landing Page」為例
  - 同時啟動 UX Architect、Content Creator、SEO Specialist 三個 subagent
  - 三者並行執行，各自產出詳細策略文件
- Claude 擔任 orchestrator，自動將任務路由給正確的專家，無需手動切換人格

#### 自訂與持久記憶

- 可修改人格的 vibe、顏色、可用工具
- 透過 `model: <model_name>` 為 subagent 指定特定模型
- 透過 `memory: project` 賦予持久記憶目錄，跨對話保留學習
  - Agent 會記住內容創作模式、用語、架構偏好
  - 每次對話結束後寫入記憶檔，下次對話開始時讀取

#### 誠實評估：限制與不完美之處

- 尚無嚴格的量化測試，不能保證這些是最佳系統提示
- 品質參差不齊：120 個人格來自 50+ 貢獻者，部分詳盡，部分較薄弱
- 新增的分類（如空間運算）較不成熟，未經充分驗證
- 建議視為起點而非魔法解決方案，應先挑選、測試、fork 後再客製化

## Key Takeaways

- Agency Agents 是結構化系統提示集合，而非真正自主的 AI 代理，但透過 Claude Code 可轉化為實際運作的 subagents
- 同一任務、同一模型下，人格化提示比通用提示品質提升約 60-70%（加權分數 8.65 vs 6.05）
- 安裝後放置於 ~/.claude/agents/，Claude Code 即可自動識別並在需要時調用對應專家
- 平行 subagent 架構能同時處理多個面向（UX、內容、SEO），大幅提升效率
- 持久記憶功能（memory: project）讓 subagent 跨對話積累專案知識，越用越聰明
- 建議視為起點而非魔法解決方案：fork、客製化、加入記憶，才能建立真正長期有效的 AI 工作流
- 品質因貢獻者不同而有差異，使用前應先測試並評估特定人格的適用性
