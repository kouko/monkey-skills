# Canon: Interaction / UX design systems & theories

Agent-facing recall insurance, consulted at the propose-candidates step as a
completeness audit — never shown raw to the user.

Including but not limited to the entries below; doctrine content stays in
the model (this list carries name + fits-when hint + stability note +
source only).

Grounded in `docs/loom/research/2026-07-10-principles-canon-base-lists.md` §2.

| Name (originator/steward, year) | Fits when… | Stability (as of 2026) | Source |
|---|---|---|---|
| Nielsen's 10 Usability Heuristics (1994) | Quick heuristic evaluation, any interface/stage | Unchanged since 1994; wording refined 2020 | [nngroup.com](https://www.nngroup.com/articles/ten-usability-heuristics/) |
| Shneiderman's Eight Golden Rules (1986) | Interaction/control-flow design (shortcuts, error prevention) | Stable textbook canon | [IxDF](https://ixdf.org/literature/article/shneiderman-s-eight-golden-rules-will-help-you-design-better-interfaces) |
| Norman's Design Principles (DOET, 1988/2013) | Affordance/mapping/signifier problems | 2013 rev. current; stable | [wikipedia](https://en.wikipedia.org/wiki/The_Design_of_Everyday_Things) |
| Laws of UX (Yablonski, 2020) | Cognitive-load/motor-control decisions (Fitts, Hick, Miller) | Living site, ~30 laws, maintained | [lawsofux.com](https://lawsofux.com/) |
| Apple Human Interface Guidelines | Native Apple-platform product, platform-idiomatic feel | Current: "Liquid Glass" era (OS 26, 2025-26) | [developer.apple.com](https://developer.apple.com/design/human-interface-guidelines/) |
| Material Design 3 (Google) | Android/cross-platform with theming + dynamic color | Current: "Material 3 Expressive" (2025-26) | [m3.material.io](https://m3.material.io/) |
| Fluent 2 (Microsoft) | Enterprise/productivity suite spanning web+native | Current major version, active | [fluent2.microsoft.design](https://fluent2.microsoft.design/) |
| IBM Carbon | Enterprise/data-dense B2B, multi-framework | v11 stable (v11.108, 2026-05); v12 refresh in progress | [carbondesignsystem.com](https://carbondesignsystem.com/) |
| Atlassian Design System | Collaboration/SaaS with shared component + content language | Evolving (2026 typography + motion) | [atlassian.design](https://atlassian.design/design-system) |
| GOV.UK Design System | Government/public service, plain-language + a11y mandate | Active; brand refresh since 2025 | [design-system.service.gov.uk](https://design-system.service.gov.uk/) |
| Ant Design (Ant Group) | Data-heavy admin/dashboard React product | Active OSS | [ant.design](https://ant.design/) |
| OOUX / ORCA (Sophia Prater, 2015) | Many entities/relationships — model objects before screens | Active practice | [ooux.com](https://ooux.com/what-is-ooux) |
| Atomic Design (Brad Frost, 2013) | Building/organizing a component-library hierarchy | Stable, foundational | [atomicdesign.bradfrost.com](https://atomicdesign.bradfrost.com/) |
| ISO 9241-210 Human-Centred Design (2019) | Formal auditable process (regulated/enterprise) | Current edition 2019 | [iso.org](https://www.iso.org/standard/77520.html) |
| JTBD for UX (Christensen/Ulwick lineage) | Prioritization anchored to user goal, not persona demographics | Stable | [nngroup.com](https://www.nngroup.com/articles/personas-jobs-be-done/) |
| WCAG 2.2 (W3C) | Legal/regulatory accessibility conformance today | Current standard; WCAG 3.0 still Working Draft (~2028-29) — do not substitute | [w3.org](https://www.w3.org/TR/WCAG22/) |
| Emotional Design (Norman, 2004) | Attachment/trust, not just usability (visceral/behavioral/reflective) | Stable | [jnd.org](https://jnd.org/emotional-design-people-and-things/) |
| Hooked Model (Nir Eyal, 2014) | Habit-forming consumer product | Stable; flag ethical/dark-pattern caveat when proposing | [nirandfar.com](https://www.nirandfar.com/how-to-manufacture-desire/) |
| Calm Technology (Weiser/Brown 1995; Case 2015) | Ambient/IoT/notification-heavy; attention is the scarce resource | Stable; Calm Tech Institute active | [calmtech.institute](https://www.calmtech.institute/calm-tech-principles) |
| Gestalt Principles (1920s) | Visual hierarchy/grouping decisions | Perceptual-science canon, permanent | [nngroup.com](https://www.nngroup.com/articles/gestalt-proximity/) |
| Rams' Ten Principles (1976-80s) | Industrial-adjacent, "less-but-better" reduction argument | Canonical, unrevised (steward: Vitsœ) | [vitsoe.com](https://www.vitsoe.com/us/about/good-design) |
| Microsoft Inclusive Design / Persona Spectrum (~2016) | A11y as permanent/temporary/situational spectrum, not a checkbox | Active toolkit | [inclusive.microsoft.design](https://inclusive.microsoft.design/) |

**Popularity head**: Nielsen's 10, Apple HIG, Material 3, WCAG — crowd out
e.g. OOUX (object-heavy domains), Carbon/Fluent (enterprise), Calm Technology
(ambient/notification-heavy). If a draft's candidate set is only these four,
re-check the full list above before proposing.
