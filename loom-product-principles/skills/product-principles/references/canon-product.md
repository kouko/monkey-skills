# Canon: Product strategy & discovery frameworks

Agent-facing recall insurance, consulted at the propose-candidates step as a
completeness audit — never shown raw to the user.

Including but not limited to the entries below; doctrine content stays in
the model (this list carries name + fits-when hint + stability note +
source only).

Grounded in `docs/loom/research/2026-07-10-principles-canon-base-lists.md` §1.

| Name (originator, year) | Fits when… | Stability | Source |
|---|---|---|---|
| Jobs-to-be-Done (Christensen, 2003; Ulwick ODI 1990s) | Team builds features nobody "hires"; reframe around the task, not the persona | Stable canon (two schools: Ulwick ODI vs Christensen narrative) | [christenseninstitute.org](https://www.christenseninstitute.org/theory/jobs-to-be-done/) |
| Opportunity Solution Tree (Teresa Torres, 2016) | Discovery is solution-jumping; trace ideas to outcome + opportunity | Evolving (Continuous Discovery Habits) | [producttalk.org](https://www.producttalk.org/opportunity-solution-trees/) |
| Lean Startup (Eric Ries, 2011) | High uncertainty whether the product should exist; build-measure-learn | Stable canon | [theleanstartup.com](http://theleanstartup.com/) |
| Design Sprint (Jake Knapp / GV, 2016) | Fast time-boxed concept test before committing engineering | Stable canon | [gv.com/sprint](https://www.gv.com/sprint/) |
| Kano Model (Kano et al., 1984) | Separate must-be table-stakes from delighters | Stable canon | [wikipedia](https://en.wikipedia.org/wiki/Kano_model) |
| RICE (McBride / Intercom, 2016) | Many candidate bets need one comparable score | Evolving (ICE/PIE/WSJF variants) | [intercom.com](https://www.intercom.com/blog/rice-simple-prioritization-for-product-managers/) |
| MoSCoW (Dai Clegg, 1994; DSDM 2002) | Scope negotiation on fixed-time/budget release | Stable canon | [agilebusiness.org](https://www.agilebusiness.org/dsdm-project-framework/moscow-prioritisation.html) |
| Blue Ocean Strategy (Kim & Mauborgne, 2004/05) | Saturated market; reframe the value curve | Stable canon | [blueoceanstrategy.com](https://www.blueoceanstrategy.com/) |
| Crossing the Chasm (Geoffrey Moore, 1991) | Works with early adopters, stalls at mainstream | Stable canon | [geoffreymoore.com](https://www.geoffreymoore.com/crossing-the-chasm) |
| Two-Sided Market / Platform Theory (Rochet & Tirole, 2003) | Two user groups; pricing/chicken-egg dynamics | Stable academic canon (2014 Nobel) | [JEEA paper](https://onlinelibrary.wiley.com/doi/abs/10.1162/154247603322493212) |
| Aggregation Theory (Ben Thompson, 2015) | Demand vs distribution control in internet-native value chain | Evolving (extended to AI) | [stratechery.com](https://stratechery.com/2015/aggregation-theory/) |
| Disruptive Innovation (Christensen, 1997) | Incumbent threatened by cheaper/simpler entrant | Stable canon (academically contested) | [christenseninstitute.org](https://www.christenseninstitute.org/theory/disruptive-innovation/) |
| Shape Up (Ryan Singer / Basecamp, 2019) | Estimate-driven over-commitment; fixed-time variable-scope appetites | Evolving (book still revised) | [basecamp.com/shapeup](https://basecamp.com/shapeup) |
| Working Backwards / PR-FAQ (Amazon, ~2004) | Idea needs a customer-first forcing function before spec/code | Stable canon | [workingbackwards.com](https://workingbackwards.com/concepts/working-backwards-pr-faq-process/) |
| AARRR / Pirate Metrics (Dave McClure, 2007) | Simple funnel vocabulary to find growth leaks | Stable canon | [original talk](https://mcgaw.io/wp-content/uploads/2016/04/PirateMetrics_Final.pdf) |
| North Star Framework (Ellis 2010; Amplitude 2017) | Drowning in vanity metrics; one metric tying value to outcome | Evolving | [amplitude.com/north-star](https://amplitude.com/north-star) |
| HEART (Rodden et al. / Google, 2010) | Operationalize UX quality into measurable metrics at scale | Stable canon | [Google Research](https://research.google/pubs/measuring-the-user-experience-on-a-large-scale-user-centered-metrics-for-web-applications/) |
| Wardley Mapping (Simon Wardley, 2005) | Strategy lacks shared situational awareness of component maturity | Evolving (community-driven) | [wardleymaps.com](https://www.wardleymaps.com/) |
| Cynefin (Snowden, 1999/2003) | "Best practice" process applied to a complex/chaotic problem | Evolving | [thecynefin.co](https://thecynefin.co/) |

**Popularity head**: JTBD, Lean Startup, Kano, RICE — crowd out e.g. Working
Backwards (narrative-driven org) or Platform Theory (marketplace). If a
draft's candidate set is only these four, re-check the full list above
before proposing.
