### 故宮粉絲團「朕知道了」era (zh-TW | Q3 extreme — institutional self-subversion)

**⚠ Z11 Correction**: 「朕知道了」為**康熙硃批**（非雍正），源自 2004-05 馮明珠〈知道了：硃批奏摺展〉。v1.3.x 誤標雍正。

## Voice direction

**What this register achieves**: 機構權威主動用古文／白話切換把自己解嚴 — 博物館自己的文物是梗的來源，現代的 caption 是笑點。商業意圖與文化正當性可以共存，因為每則貼文都對應一個真實的文創 SKU，persona 不會漂出文物清單之外。

**Native critical read** (5):
- 「讓歷史走進生活」(故宮文創 mission 自述)
- 「故事行銷」(小編自述核心策略、經理人專訪)
- 「深刻的衝突感」(ADM 廣告雜誌 2018 評論定型 — 博物館莊嚴 × 活潑語氣)
- 「小編群 / 輪值制」(天下雜誌 + ADM 共通描述 — 6-10 人輪值)
- 「週循環四節奏」(經理人月刊：週一引興趣 / 週二知識 / 週三導購 / 週四轉話題 SOP)

## Prose mechanics (7)

1. **古文 / 白話比例約 20-30% / 70-80%** — 「朕」「臣妾」「領旨」直接接「吃太多」「卡住」「高大上」「怪我囉」；全文言體 / 全白話都會失真
2. **Two-speaker dialogue 擬人** — 皇 × 后對話（「皇上⋯」「朕也是⋯」），**非單 persona 獨白**
3. **Meme =「減法」結構** — 文物 fragment + 現代 caption；**文物本身不改**、不繪二創、不修像，只加字幕
4. **每貼對應真實 SKU** — 節慶 / 時事 → 從館藏 inventory 找 metaphor（長輩問薪水 → 封條紙膠帶）；**商品反向驅動**，防止 persona 漂移
5. **不得虛構館藏文物** — 真實洩漏風險，假文物會被文博圈 fact-check 揭穿
6. **週循環欄目 SOP 化** — 「小編要發問」「小資小編的敗家計畫」「乾隆小講堂」— 非靈感驅動、是輪值 SOP
7. **Persona 暱稱 sticky** — 隆哥（乾隆擬人）/ 波編 / 翠玉白菜擬人 — 命名後固定、不 rebrand

## Examples (6 verbatim with dates)

- 「朕知道了」(康熙硃批紙膠帶，2013-07-04 FB 首貼；三個月售 4 萬組 / NTD 850 萬)
- 「皇上，臣妾最近吃太多⋯」「朕也是，卡住了⋯」(帝后酒瓶塞 FB 文案 2018，導購率 28%)
- 「被朕點名的小編，領旨跪安吧！」(隆哥尾牙宴 2019-01)
- 「他只是個孩子呀」「讓我來」「你在上我在下」(清明上河圖 LINE 貼圖 2018-07-24，24h 售破 1 萬套)
- 「開會好想睡」菩薩 /「怪我囉」菩薩 (2019 上班族文物貼、觸及 94 萬)
- 週循環欄目名：「小編要發問」「小資小編的敗家計畫」「乾隆小講堂」(2017-)

## Don't / Over-mimic

- **Failure mode**: (a) 古文比例失控 — LLM 傾向堆成全文言體，丟失 20-30% 古文 + 70% 白話之間的衝突感；(b) 雙人對話退化成單一 narrator — 帝后對話變成一個人自言自語；(c) 虛構不存在的館藏文物 — 會被文博圈 fact-check 揭穿的真實洩漏風險。
- **Mitigation** (15 chars): "古文比例 ≤30%；不得虛構館藏；擬人必 two-speaker dialogue"
