# v1.6.1 Structural E2E Verification

## Scope

Post-migration (v1.6.0 → v1.6.1) verification that:
1. All 12 router indexes resolve to existing anchor-*.md files
2. All anchor-*.md files have `schema_version: 2.0` frontmatter
3. Pass 3d two-step read protocol (router → anchor body) is structurally sound

Not in scope: actual Pass 3 agent run (requires full pipeline orchestration).

## Method

`/tmp/verify_routers.py` + `/tmp/find_unlinked.py` scan `standards/` directory for link integrity and schema compliance.

## Results

### Router link integrity: PASS

- **12/12 routers** scanned (jp/zh/en × q1/q2/q3/q4)
- **60 markdown links** to `anchor-*.md` files found in routers
- **0 broken links** — every link resolves to an existing file

### Schema compliance: PASS

- **67/67** `anchor-*.md` files carry `schema_version: 2.0` in frontmatter
- Pass 3d schema auto-detect will correctly identify v2 entries on load

### Minor finding: 7 craft-gate anchors unlinked from quadrant routers

7 anchor-*.md files exist but are not linked from any `{lang}-q{N}-anchors.md` router:

- `anchor-jp-itoi-shigesato-state-proposal.md`
- `anchor-jp-iwasaki-shunichi-yonin.md`
- `anchor-jp-maki-jun-craft-aphorism.md`
- `anchor-jp-taniyama-masakazu-discipline.md`
- `anchor-zh-tw-xu-shunying-ideological-definitional.md`
- `anchor-zh-tw-lee-hsin-ping-literary-consumption.md`
- `anchor-zh-tw-ye-mingui-strategic-aphorism.md`

**Not a bug** — these are **craft-gate** masters (糸井重里 / 岩崎俊一 / 眞木準 / 谷山雅計 / 許舜英 / 李欣頻 / 葉明桂). Pass 3 routes craft-gate briefs through `Pass 3a` (JP) / `Pass 3b` (ZH) branches which load `jp-copy-craft-lineage.md` / `zh-copy-craft-lineage.md` directly, NOT through the register-signal quadrant router. The flat anchor files are **additive v2-schema translations** of craft-gate content — accessible via direct slug lookup or `Glob` but not on Pass 3's hot path.

**Recommendation for v1.7.0+**: either (a) explicitly link these from quadrant router §"Craft-gate pointer" sections for register-signal-path reachability, OR (b) let them remain as slug-indexed alternates to the craft-lineage canonical sections, with craft-lineage files still the authoritative craft-gate source.
