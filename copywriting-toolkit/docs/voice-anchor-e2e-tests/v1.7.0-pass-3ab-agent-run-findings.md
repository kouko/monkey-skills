## Architectural findings (4 total; 1 actionable immediately)

### 🔴 Finding 1 (actionable, v1.7.1): duplicate mitigation registry entries

**Scope**: `voice-anchor-meta-core.md §Over-mimic mitigation registry` table contains entries for craft-gate masters (許舜英 at line 179 confirmed; likely similar for 糸井 / 岩崎 / 眞木 / 谷山 / 李欣頻 / 葉明桂) that **duplicate** each anchor's own `Don't / Over-mimic` block.

**Why it matters**: v2 schema declared "anchor is SSOT for mitigation" (meta-core §v2 schema). Registry table entries for v2-migrated masters violate the invariant. Real risk: when anchor updates its mitigation clause, registry drifts unnoticed.

**Action for v1.7.1**: either delete the 7 registry rows for v2-migrated masters, or replace each with a stub pointer (`see anchor-{slug}.md §Don't`). Registry table remains for v1-schema entries (非-craft-gate anchors whose v2 migration hasn't happened).

### 🟡 Finding 2 (flag, deferred): `Cross-reference-valid-for` tier inconsistency

Anchor `anchor-zh-tw-xu-shunying-ideological-definitional.md` has `Cross-reference-valid-for: 張愛玲 aphoristic parallel (stylistic, not documented)` — a free-text note. Meta-core §v2 schema specifies the field should carry `STRONG / MEDIUM / WEAK` tier with lang code (e.g. `ja: STRONG`). This anchor uses it for "other-master parallel" which is a different semantic (stylistic-sibling, not cross-language transferability).

**Action**: either extend v2 schema to recognize "stylistic-sibling" field, or normalize this anchor's entry to `zh-cross-master-sibling: 張愛玲` as separate field.

**Scope**: low priority — Pass 3 doesn't consume this field for rewrite; it's informational.

### 🟡 Finding 3 (flag, deferred): Form enum granularity gap

JP anchor 糸井 declared `pairs with form: [mid-form-brand-tagline, short-form-catchcopy, light-action-lifestyle]`. These sub-form labels are not in the current Phase 4 form enum. Pass 3 agent had to self-resolve `mid-form` → `mid-form-brand-tagline`.

**Action**: Phase 4 form taxonomy refinement — decide whether to enumerate sub-forms or standardize labels to top-level forms. Scope belongs to Phase 4, not Pass 3.

### 🟢 Finding 4 (minor, cleanup): anchor §Coverage self-check meta-commentary

Some anchor files (糸井 flagged) include a §Coverage self-check section that is meta-commentary about the anchor's own schema completeness, not Pass 3 rewrite content. Adds ~80 tokens per affected anchor × 7 craft-gate = ~560 tokens non-operational load.

**Action**: either remove these sections or mark as frontmatter-only metadata. Low priority.
