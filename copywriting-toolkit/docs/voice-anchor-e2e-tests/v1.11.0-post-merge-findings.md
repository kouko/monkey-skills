## Test 3 — 張愛玲 → 白先勇 substitute (PASS_WITH_NOTES — **REAL BUG**)

**Brief**: 400-字 上海精品酒店, `voice_reference: 張愛玲《傾城之戀》式的老上海 register`, `tone_cue: 華麗頹廢、今昔對照、懷舊`.

**BUG found**: Meta-core §Safe-substitute lookup (line 126) restricts trigger to *"when brief.voice_reference names a master with **HIGH over-mimic risk** in the registry"*. But:
- 張愛玲 anchor frontmatter: `Over-mimic risk: MEDIUM`
- 張愛玲 NOT listed in §Over-mimic mitigation registry table

→ Trigger predicate **does not fire** → `substitute_suggested: null` → 白先勇 never surfaces, even though v1.11.0 wired the frontmatter pairing.

**v1.11.0 data change was not matched by rule-text broadening.** This affects any pairing where target is MEDIUM-risk or missing from the registry table.

**Audit of 6 new pairings**:
| Pair | Target risk | In registry? | Fires? |
|---|---|---|---|
| Carver → [Hemingway] | HIGH | ✓ | ✓ |
| Ephron → [Didion] | HIGH | ✓ | ✓ |
| Hammett → [Chandler] | HIGH | ✓ | ✓ |
| 谷川 → [糸井] | HIGH+ (craft-gate) | ✓ | ✓ |
| **白先勇 → [張愛玲]** | **MEDIUM** | **✗** | **✗ BUG** |
| 錢鍾書 → [DFW] | HIGH | ✓ | ✓ |
| 吉本ばなな → [村上] (v1.10.0) | HIGH | ✓ | ✓ |
| 養老 → [原研哉] | (need audit) | ? | ? |
| 外山 → [伊丹] | (need audit) | ? | ? |

**Also discovered**: `tone_cue` contains "華麗頹廢" which exactly matches 白先勇's Don't-block warning about 張愛玲 contamination. This is a case where substitute isn't just "safer" — it's *strictly necessary* to avoid the failure mode.

**Proposed fix** (v1.11.1):

Edit `voice-anchor-meta-core.md §Safe-substitute lookup` line 126:
- **Current**: "When `brief.voice_reference` names a master with **HIGH over-mimic risk** in the registry..."
- **Fix**: "When `brief.voice_reference` names a master AND at least one anchor in `standards/anchor-*.md` lists that master in `frontmatter.safe_substitute_for[]`, Pass 3 MAY auto-suggest. The substitute qualifies iff its over-mimic risk is **strictly lower** than the named master's (LOW < MEDIUM < HIGH < HIGH+)."

Same fix mirror-applied to `SKILL.md` Pass 3d step 6.

**Secondary enhancement** (optional): `tone_cue_contamination_check` — when `brief.tone_cue` contains tokens matching substitute-target's failure-mode warnings, upgrade `substitute_suggested` to `contamination_warning` (stronger signal than default suggestion).
