## Breakage report

v1.13.0 cross-tradition check fires correctly — no breakage.

**Trace**: The cross-tradition guard is defined in SKILL.md §When Pass 3 applies, immediately after the two activation conditions, as a hard pre-check before Step 1 (Load). The guard text reads: *"If lineage-language conflict detected (e.g., voice_reference: 糸井重里 but output_language: zh-TW) → emit violation envelope per ../../CLAUDE.md §Envelope Violation; router routes to copywriting-intake."* The input envelope is the verbatim example cited in the spec. v1.13.0's simplification (collapsing Pass 3a/3b/3c/3d into a single 8-step flow) did not remove or relocate this guard — it remains at the §When Pass 3 applies gate, which runs **before** the unified 8-step flow is entered. The violation path therefore fires independently of the Step 1–8 simplification. No regression.
