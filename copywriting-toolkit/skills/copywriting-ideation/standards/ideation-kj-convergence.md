# KJ法 収束発想法

Tier 3 standard: self-contained. Uses 川喜田二郎's 1967 book
『発想法』(中公新書 136) as canonical source, with 1970
『続・発想法』(中公新書 210) as extension and 1986 『KJ法』as the
complete systematization. This file provides the standard convergence
tool for copywriting-team's 選ぶ (convergence) stage: receiving
64–81 candidates from the Mandal-Art / Verbalized Sampling divergence
stage, then converging to 3–5 winners through a 6-stage process.
Core principle:「渾沌をして語らしめる」(let chaos speak for itself)
— do not classify top-down, but cluster bottom-up by affective affinity,
letting data reveal its own emergent structure.

- 川喜田二郎 (1967) 『発想法 —— 創造性開発のために』中公新書 136 — KJ法 canonical first publication. Presents the system: テーマ設定 → ラベル作成 → グループ編成 → 表札づくり → 図解化 → 文章化.

- Wikipedia 日本語版「KJ法」 https://ja.wikipedia.org/wiki/KJ法 — secondary dual-verify source: 川喜田 1967 first publication, narrow 4-stage / broad 6-stage, ネパール fieldwork origin.
- Swing Root「KJ法とは」 https://swingroot.com/kj-method — secondary dual-verify source: 6-stage practical procedure and 表札づくり affective-affinity explanation.
- UX Times「KJ法の基礎」 — secondary dual-verify source: 6-stage operation in UX research context and グループ編成 failure cases.
- Christopher Roosen (EN) "KJ-Method: A Primer" — secondary dual-verify source：英語圏への紹介と affinity diagramming との差分解説。

## Framing

KJ法 is a **convergence-only** ideation method, distilled from
川喜田二郎's data-organization experience during 1960s Nepal fieldwork.
Its complementary relationship with brainstorming is clear: brainstorming
(divergence) generates raw material, while KJ法 (convergence) synthesizes
that material into actionable insights.

The core principle「**渾沌をして語らしめる**」(chaos speaks for itself)
mandates: do not pre-define a classification framework from the
researcher's perspective, but cluster **bottom-up** by affective affinity
of the data itself. The fundamental difference from top-down classification
(decision trees / taxonomies) is that KJ法 preserves **the data's own
space for logic emergence**, preventing the researcher's prior cognition
from flattening undiscovered patterns.

Within the copywriting pipeline, this tool is used for:
- Organizing consumer-interview transcripts (extracting insights from
  large volumes of raw responses)
- Converging copy angles (from Mandal-Art 81 cells or Verbalized
  Sampling's 64 candidates down to 3–5 winners)
- Competitor analysis (pattern extraction from multi-firm ad taglines)

## 6-stage detailed definitions

### Stage 1: テーマ設定 / ブレインストーミング

- **テーマ設定**: establish the focal question for this KJ法 session
  (e.g.「ミドル世代向けサプリの訴求角度は何か」). An ambiguous theme
  causes all subsequent stages to collapse.
- **ブレインストーミング** (optional): if material is insufficient,
  generate more via BS first. If material is adequate (e.g. interview
  transcripts, Mandal-Art 81 cells), proceed directly to stage 2.

### Stage 2: ラベル作成 (card creation)

- **Rule**: **one card, one concept**. Each card contains only one idea
  or one verbatim quote. Mixing multiple concepts on one card destroys
  subsequent clustering granularity.
- **Expression quality**: write concretely. Abstract labels like
  「品質向上」are inadequate; concrete labels like「アミノ酸配合量が
  競合より 2 倍」are what qualify.
- **Output scale**: 50–200 cards is typical. Too few (<30) means
  insufficient data for emergence; too many (>300) requires batch
  splitting for a single-person session.

### Stage 3: グループ編成 (clustering)

- **Rule**: cluster by **affective affinity** ("似たもの同士") —
  bringing together items that "feel related". **Do not classify by
  semantic category** (this is the root difference from top-down methods).
- **Procedure**:
  1. Spread all cards on a desk / whiteboard
  2. Bring 2–3 cards together based on "vague closeness" (**intuition,
     not logic**)
  3. Form 10–30 small groups (2–5 cards each)
  4. Merge small groups into medium groups, then medium into large —
     creating a hierarchy
- **Prohibition**: classifying with pre-existing frameworks (persona /
  4P / SWOT, etc.) before grouping. This violates「渾沌をして語らしめる」.

### Stage 4: 表札づくり

- **Rule**: assign a「表札」(header label) to each group. The header
  verbalizes the **common essence** of all cards in the group in a
  single line.
- **Quality criteria**:
  - **Bad example**:「品質関連」(category name, abstract)
  - **Good example**:「科学的根拠で選びたい合理派」(the group's story)
- **Hierarchy**: assign headers at each level — small → medium → large.
  The large-group headers form the backbone of KJ法's final insights.

### Stage 5: 図解化 (A-type)

- **Rule**: arrange large groups in **spatial layout**. Not just listing
  them, but showing **causal / opposing / parallel / containment**
  relationships between groups with arrows and symbols.
- **Meaning of A-type**: Analysis の A. This diagram is KJ法's
  "visualization" deliverable.
- **Layout principles**:
  - Vertical → abstract / concrete or cause / effect
  - Horizontal → chronological or opposition axis
  - Center → core theme, periphery → supporting elements
- **LLM must not do**: spatial language depends on human spatial
  cognition. Embedding-based 2D layouts show semantic proximity but
  cannot produce the "conscious placement of causal / opposing
  relationships" that KJ法 A-type requires.

### Stage 6: 文章化 (B-type)

- **Rule**: based on the A-type diagram, write a **narrative text**
  for the conclusion. The text should be comprehensible to readers who
  have not seen the diagram — a continuous text.
- **Meaning of B-type**: Breakdown / Building の B.
- **Typical length**: A4 1–2 pages, 400–800 characters.
- **Structure**: core insight → narrative of major groups → inter-group
  relationships → implications / next actions.

## Narrow vs broad KJ法

- **Narrow KJ法**: stages 2–5 (labels → group formation → headers →
  A-type diagram). KJ法 as a data-organization method.
- **Broad KJ法**: stages 1–6 (theme setting through text composition).
  KJ法 as a comprehensive problem-solving / decision-making method.

What copywriting-team uses is fundamentally **broad KJ法**. However,
for purposes like "first-pass structure visualization of interview
transcripts", stopping at narrow KJ法 is acceptable.

## 「渾沌をして語らしめる」principle

The root philosophy of KJ法, formalized in 川喜田 1986:

> データを既存の枠組みに押し込むのではなく、データ自身が持つ秩序が
> 湧き上がるのを待つ。研究者は触媒であり、分類者ではない。

Operational implications:
- **Do not fix pre-existing hypotheses** (hypothesis testing should use
  a different method, not KJ法)
- **Do not import existing taxonomies** (persona / 4P / SWOT, etc.)
- **If grouping feels wrong, disassemble and re-arrange** (structural
  changes mid-process are normal; no changes is suspicious)
- **When using LLM, embedding similarity is an "approximation of
  affective affinity", not a substitute**. Embeddings over-weight
  categorical proximity and miss the "essential similarity amid
  heterogeneity" that KJ法 values.

## Automation boundary table

| Stage | LLM can do | Semi-auto (human checkpoint) | LLM must not do |
|---|---|---|---|
| Stage 1: テーマ設定 | — | ✓ LLM proposes, human decides | — |
| Stage 2: ラベル作成 (transcript → cards) | ✓ concept extraction + one-card-one-concept decomposition | — | — |
| Stage 3: グループ編成 | — | ✓ embedding initial clusters + human rearrangement | ✗ final judgment of affective affinity |
| Stage 4: 表札づくり | — | ✓ LLM proposes 3–5 candidates, human selects | — |
| Stage 5: A-type 図解 | — | — | ✗ spatial layout requires human cognition |
| Stage 6: B-type 文章化 | ✓ narrative composition from A-type + headers | — | — |

**Rules**:
- When using LLM at the embedding stage 3, treat it as **a draft for
  initial clusters only**. A human checkpoint must be set where humans
  re-arrange ("this doesn't belong together", "these should be split").
- Do not have the LLM output stage 5 A-type diagrams. If automation
  is absolutely required, limit to "hierarchical list in text format"
  (no spatial layout).
- Do not adopt LLM-generated headers verbatim at stage 4. LLMs tend
  to produce category-name-style headers (「品質関連」「価格関連」)
  that fail to capture the group's story.

## Application to copywriting

### Primary use cases

- **Consumer interview organization** — organize 10–30 interview
  transcripts via KJ法 → ペルソナ / insight discovery → feedback
  to planning-team
- **Copy angle convergence** — Mandal-Art 81 cells + Verbalized
  Sampling 5–10 candidates = ~100–150 candidates, KJ法 narrows to
  3–5 winners
- **Competitor tagline analysis** — KJ法 clustering of 20–50
  competitor copies → differentiation whitespace discovery
- **キャッチコピー 64-candidate convergence** — from 谷山 2007's
  散らかす stage output of 64 lines, selecting the 3–5 winners
  (→ integrated with `ideation-taniyama-discipline.md`「選ぶ」stage)

### Representative workflow (catch copy 64 candidates → 3–5 winners)

```
【Step 1】テーマ設定
  └── "Select 3–5 winning catch copies for product 〇〇"

【Step 2】ラベル作成
  └── Expand 64 lines to 64 cards
  └── Each card contains copy text + the ideation angle (Mandal-Art direction)

【Step 3】グループ編成
  └── Embedding produces initial 10–15 clusters
  └── Human rearranges 3–5 items that "don't belong"
  └── Hierarchy: small → medium → large

【Step 4】表札づくり
  └── One-line header for each large group: "the shared essence
      this copy cluster appeals to"
  └── Example:「合理派を安心させる数字訴求群」「感性派を物語で引き込む群」

【Step 5】A-type 図解
  └── Arrange large groups: X-axis = rational / emotional,
      Y-axis = short / long
  └── Visualize each cluster's strength and identify whitespace

【Step 6】B-type 文章化 + winner selection
  └── Narrate: "For this round's target (received from
       planning-team), which cluster is optimal?"
  └── Extract 3–5 lines from the selected cluster as winners
  └── → ideation-taniyama-discipline.md「磨く」stage
```

## Anti-Patterns

- **Single-step LLM full-auto completion**: prompting "narrow 64 items
  to 5 via KJ法" and letting the LLM complete it independently. This
  is top-down compression that violates「渾沌をして語らしめる」—
  it's majority-vote summarization borrowing KJ法's name.
- **Replacing グループ編成 with embedding alone**: calling fully
  automated embedding cosine clusters "KJ法". Embedding measures
  categorical similarity, not affective affinity ("essential closeness
  amid heterogeneity"). Human checkpoints are mandatory.
- **Grouping with existing frameworks**: using persona / 4P / AIDMA
  stages as pre-set categories. 川喜田 explicitly prohibits this
  "forcing data into existing frameworks". It eliminates KJ法's value.
- **Skipping human checkpoints**: running embedding initial clusters →
  straight to headers → direct text composition through the pipeline.
  At minimum, human (or evaluator agent) confirmation of "does anything
  feel wrong?" at stages 3 and 4 is required — otherwise LLM's
  pre-training bias crushes unexpected essences.
- **Having the LLM draw A-type diagrams**: even ASCII or graphviz
  layouts do not achieve "spatial placement where humans can read
  causal / opposing relationships". A-type is human work.
- **Using category names as headers**: "品質関連", "価格関連", etc.
  These fail to verbalize the group's story. Headers must narrate
  "what this group is saying."
- **Confusing narrow with broad KJ法**: stopping at stages 2–3 and
  reporting "KJ法 done." That is narrow KJ法, incomplete as
  decision material. Copywriting usually requires through stage 6.
