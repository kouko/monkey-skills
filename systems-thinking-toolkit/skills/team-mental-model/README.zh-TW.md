# team-mental-model

浮現每位團隊成員的心智模型，協商根本價值的收斂，並透過可衡量的領導力能量指標（節奏 / 積極傾聽 / 價值重訪）持續維繫。不是一次性的 offsite — 持續投入是必要條件。

## 使用時機

- 團隊心智模型漂移: 「以前大家會自動同步」/ 決策耗時拉長 3 倍 / 個人卓越度持續上升但 code review、design review 卻越來越針鋒相對。
- 併購或整合後的對齊: `cld-overlay` 已找到群組間的跨騎點 (straddle)，整合後團隊需要持續維繫該收斂的階段。
- 「個人優秀但團隊緩慢」訊號: 從 FAANG / 競爭對手挖到明星人才，速度反而下降，「需要更多團隊建設」的說法不斷出現但沒有可衡量的改善。

## 啟動方式

`/systems-thinking-toolkit:team` 或透過路由器 `/systems-thinking-toolkit:stt`。

## 產出

- 每位成員一張因果迴路圖 (CLD)，主題為「什麼導致這個團隊成功」；並排比較以定位根本價值節點上分歧的 S/O 標籤。
- 共同撰寫的價值收斂聲明，每位參與者都能認可為「自己的」（不是備忘錄式的單向通告）。
- 追蹤至少 1 個季度的 4 個可觀測領導力能量指標: 節奏次數（每週至少 1 次、團隊自發）、積極傾聽時數（每小時至少 15 分鐘）、價值重訪次數（每週至少 1 次）、符號與敘事重述（選用）。

## 邊界

- 心理安全 (Edmondson) 是約束條件 — 若成員不敢發言，先處理安全問題。沒有安全的浮現只會產生表演式同意，不是真正的心智模型。
- 不適用於遠端 / 分散式 / 非同步優先文化 — 能量泵浦的隱喻假設共處一地。Sherwood「多年共享晚餐」模式早於 2020 年混合辦公轉變，書中未提供非同步環境的可操作定義。
- 不是一次性 offsite 或年度 all-hands — Sherwood 明確對比 Nelson 多年持續的共享晚餐與 Coopers & Lybrand 100 位合夥人反例（一次性訓話）。
- 不適用於短期任務小組、以認知多樣性為設計目的的團隊（red team、devil's advocate、稽核）、也不能取代招募或解雇決策。
- 期間標註的例證: Nelson 在特拉法加海戰的 Band of Brothers (c18，黃金標準) 與 Coopers & Lybrand 100 位合夥人案例 (c19，訓話反模式) 是 Sherwood 的正典示例。
