# team-mental-model

各メンバーのメンタルモデルを可視化し、根本価値での収束を交渉し、観測可能なリーダーシップ・エネルギー指標（カデンス / 能動的傾聴 / 価値再訪）で持続させる。一回限りのオフサイトではなく、継続的投入が要件。

## 使うとき

- チームのメンタルモデルがドリフトしている兆候:「以前は阿吽の呼吸だった」/ 意思決定が 3 倍遅くなった / 個人の卓越性は上がっているのに code / design review が紛糾。
- M&A / 統合後のアライメント: `cld-overlay` でグループ間のストラドルが既に発見されており、統合後チームで収束を持続させる必要がある段階。
- 「個々は優秀だがチームが遅い」シグナル: FAANG / 競合からスター人材を採用 → ベロシティ低下 →「もっとチームビルディングを」というフレーミングが繰り返されるが計測可能な改善が出ない。

## 起動方法

`/systems-thinking-toolkit:team` またはルーター `/systems-thinking-toolkit:stt` 経由。

## 得られる成果物

- メンバーごとの「このチームが成功する原因」因果ループ図 (CLD)。並列比較で根本価値ノードの S/O ラベルの分岐を特定。
- 全参加者が「自分のもの」と認識できる、共著された価値収束ステートメント（メモ通達ではない）。
- 1 四半期以上トラックされる 4 つの観測可能なリーダーシップ・エネルギー指標: カデンス数（週 1 回以上、チーム自発）、能動的傾聴時間（1 時間あたり 15 分以上）、価値再訪回数（週 1 回以上）、シンボル & ナラティブ更新（任意）。

## 境界条件

- 心理的安全性 (Edmondson) が拘束条件 — メンバーが発言を恐れている場合は、まず安全性を確保。安全性のない可視化は genuine なメンタルモデルではなくパフォーマティブな同意を生む。
- リモート / 分散 / 非同期ファースト文化には不適 — エネルギー注入のメタファーは共在を前提とする。Sherwood の「年単位の共有ディナー」は 2020 年のハイブリッドワーク転換以前のモデルであり、非同期環境の operational definition は提供されていない。
- 一回限りのオフサイト / 年次 all-hands ではない — Sherwood は Nelson の継続的な年単位の共有ディナーと Coopers & Lybrand 100 パートナー反証事例 (一回限りの檄文) を明示的に対比している。
- 短期タスクフォース、認知多様性が設計目的のチーム（レッドチーム、悪魔の代弁者、監査）、採用 / 退職判断の代替には使用しない。
- 期間限定の事例: Nelson のトラファルガー海戦における Band of Brothers (c18、黄金標準) と Coopers & Lybrand 100 パートナー事例 (c19、檄文アンチパターン) は Sherwood の正典的例示。
