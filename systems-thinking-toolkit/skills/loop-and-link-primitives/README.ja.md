# Loop-and-Link Primitives — R/B 判別 + S/O リンク符号付け

[English](README.md) | [日本語](README.ja.md) | [繁體中文](README.zh-TW.md)

すべての因果矢印に符号 (S/O) を付与し、可逆性テストと Sterman の究極テストをフォールバックとして使い、閉じたループで O を数えて R か B に分類する。

## 使うとき

- 因果ループ図をゼロから描いていて、各矢印で「これは S か O か」と止まってしまうとき
- KPI（NPS、リテンション、株価）が緩やかに下降または非線形に加速しており、フィードバック ループを疑うとき
- 「悪循環」「死のスパイラル」「バブル」という言葉が出ており、回転方向と反転トリガーを診断したいとき
- 変動 KPI（目標 vs 実績）の定義が変更され、既存図のラベルが暗黙に誤っている可能性があるとき

## 呼び出し方

`/systems-thinking-toolkit:link-primitives`

ルーター経由：`/systems-thinking-toolkit:stt`

## 得られるもの

- リンクごとの S または O ラベル（可逆性 / Sterman テスト記録付き、ユニフロー該当は明示）
- ループごとの R（O 数が偶数、ゼロを含む）または B（O 数が奇数）分類、R は現在の回転方向 + 反転トリガー候補、B は目標 + 遅延量
- 一文の動的予測（例：「現在は好循環の R ループ。トリガー X が起きない限り複利的に成長」）

## 境界

- 閉ループのない一回限りの意思決定には使わない（偶数 O / 奇数 O は定義不能）
- 相関関係のラベル付け（因果方向の主張なし）や DAG の因果推論エッジ（構造上、符号なし）には使わない
- 体制依存リンク（閾値で符号反転）は `cld-craft` の split-fuzzy-variable トリックに引き渡す — 単一ラベルを選ばない

完全な手順（R/I/A1/A2/E/B）は [SKILL.md](SKILL.md) を参照。
