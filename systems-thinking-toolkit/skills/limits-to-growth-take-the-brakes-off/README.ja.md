# limits-to-growth-take-the-brakes-off

[English](README.md) | [日本語](README.ja.md) | [繁體中文](README.zh-TW.md)

成長の減速を「強化ループがバランスループにブレーキをかけられている状態」として診断し、エンジンを押し込むのではなく制約を解放する。

## 使うとき

- かつて複利的に伸びていた成長エンジンが、投資・採用・キャンペーンを増やしているのに減速している。
- 会議室の既定反射が「もっと踏み込め」になっている — 予算倍増、人員追加、施策追加。
- 単一の binding な制約（キャパシティ、人材、市場セグメント、規制余地）が、追加投入を吸収していると疑われる。

## 呼び出し方

```
/systems-thinking-toolkit:limits-to-growth
```

## 得られるもの

- 中心となる強化ループを 6 ノード以下で描いたもの。
- 候補となるバランスループ 3〜7 個、それぞれ制約資源の名前付き。
- ONE 個の binding 制約と、それを確定した診断テスト。
- 名前・コスト見積・タイムライン付きの制約解放介入。「これで強化ループは減速前の速度で回り戻る」という明示的予測込み。
- 次に binding する制約 — 今回のブレーキを解放したら次にキューイングされるもの。

## 範囲外

- 真の外生ショック（需要崩壊、規制ステップ変化、競合参入）には使わない — 別診断。
- エンジン自体が壊れたケース（プロダクトマーケットフィット前、品質劣化、悪循環の bust モード）には使わない — そもそも解放すべき強化ループが存在しない。
- 解放手段のない硬い制約には使わない — その場合は「ブレーキを外す」ではなく「小さい定常状態を受け入れ、成長追いの現金燃焼を止める」に切り替える。
