# cld-craft

ワークショップの規律で因果ループ図を描く — 12 の衛生ルール + ファジー変数の昇格。

## 使うべき場面

- CLD ワークショップを開始（または立て直し）しており、図がノード 20〜30 を超えて膨張し止めどころが見えない。
- 明らかに挙動を駆動する変数（モラール、ability to cope、ブランド強度、信頼）について、チームの誰かが「測れないから扱えない」と却下しようとしている。
- 図中のリンクがレジーム依存（飽和・閾値・逆 U 字）で、単一の S / O ラベルがループ全体を静かに誤分類している。

## 起動方法

`/systems-thinking-toolkit:cld-craft` またはルーター `/systems-thinking-toolkit:stt` から。

## 得られるもの

- 明示的なダングルリスト（input / target / rate / output / cloud）で境界づけられ、状態名詞のノードと、すべての矢印にペンで打たれた S / O 符号を備えた衛生的な CLD — `loop-and-link-primitives` による R / B 分類にそのまま渡せる状態。
- 挙動を駆動するファジー変数が第一級ノードに昇格され、手描きの応答曲線が添えられた図。符号がレジームで反転するリンクには split-fuzzy-variable trick が適用される。

## 適用しない場面

- 測定可能なシグナルが一切ない場合は NG — この規律は「未測定だが挙動を駆動する」変数を昇格するためのものであり、因果構造をゼロから捏造する道具ではない。
- ファジー変数を未指定のまま放置する免罪符ではない — 昇格された各ノードは少なくとも曲線形状（線形 / 飽和 / 閾値 / 逆 U 字）を宣言しなければならない。
- 本質が因果構造ではなく価値観の対立である論争には不向きであり、SOX / IFRS / デューデリジェンスが要求する監査可能な数値の代替にはならない。
