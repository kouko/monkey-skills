# cld-craft

Draw a causal loop diagram with workshop discipline — 12 hygiene rules + fuzzy variable elevation.

## When to use

- You are starting (or rescuing) a CLD workshop and the diagram keeps sprawling past 20–30 nodes with no clear stopping rule.
- Someone on the team is rejecting an obviously load-bearing variable (morale, ability to cope, brand strength, trust) on the grounds that "we can't measure it."
- A link in the diagram is regime-dependent (saturation, threshold, inverted-U) and a single S or O label is silently mis-classifying the loop.

## How to invoke

`/systems-thinking-toolkit:cld-craft` or via the router `/systems-thinking-toolkit:stt`.

## What you get

- A hygienic CLD bounded by an explicit dangle list (input / target / rate / output / cloud), state-noun nodes, and S/O signs in pen on every arrow — ready for R/B classification via `loop-and-link-primitives`.
- Load-bearing fuzzy variables elevated to first-class nodes with hand-drawn response curves, plus the split-fuzzy-variable trick applied wherever a link's sign flips with regime.

## Boundaries

- NOT for when you have no measurable signal at all — the discipline elevates unmeasured variables, it does not invent causal structure from thin air.
- NOT a license to leave fuzzy variables unspecified — every elevated node must carry at least a declared curve shape (linear / saturating / threshold / inverted-U).
- NOT for disputes that are fundamentally about values rather than causal structure, and NOT a substitute for audited numbers where SOX / IFRS / due-diligence applies.
