# cld-craft

以工作坊紀律繪製因果迴路圖 — 12 條衛生規則 + 模糊變數提升。

## 使用時機

- 你正在開啟（或挽救）一場 CLD 工作坊，圖卻不斷膨脹超過 20〜30 個節點、找不到停手的標準。
- 團隊裡有人正以「無法量測」為由排除一個顯然承擔關鍵作用的變數（士氣、ability to cope、品牌強度、信任）。
- 圖中某條連結屬於 regime 相依（飽和、閾值、倒 U 字），單一個 S 或 O 標記正悄悄地把整個迴路誤分類。

## 啟動方式

`/systems-thinking-toolkit:cld-craft` 或透過路由器 `/systems-thinking-toolkit:stt`。

## 你會得到

- 一張以明確懸垂端清單（input / target / rate / output / cloud）界定範圍、節點皆為狀態名詞、每支箭頭都用筆當場標上 S / O 的衛生 CLD — 可直接交給 `loop-and-link-primitives` 進行 R / B 分類。
- 承擔關鍵作用的模糊變數被提升為一級節點並附上手繪反應曲線；凡是符號會隨 regime 翻轉的連結都已套用 split-fuzzy-variable trick。

## 邊界條件

- 當你完全沒有任何可量測訊號時，不適用 — 此紀律是提升「未量測但承擔關鍵作用」的變數，並非從空無中虛構因果結構。
- 不是讓模糊變數始終不被界定的免責通行證 — 每個被提升的節點至少要宣告一種曲線形狀（線性 / 飽和 / 閾值 / 倒 U 字）。
- 不適用於本質為價值之爭而非因果結構之爭的場合，也無法取代 SOX / IFRS / 盡職調查所要求的可稽核數字。
