# variance-target-action-template

[English](README.md) | [日本語](README.ja.md) | [繁體中文](README.zh-TW.md)

汎用バランスループ制御テンプレート（ターゲット ↔ 変動 ↔ 介入 ↔ 実績）と、振動下での「何もしない」判断診断。

## 使うべき場面

- KPI がターゲット周りで振動し、振幅が持続あるいは拡大している。
- 介入のたびに振れ幅が広がる「ピンポン調整」状態。
- 管理制御メカニズム（ノルマ、OKR、在庫ルール、金融政策委員会、カスタマーサクセスのプレイブック）をバランスループとしてモデル化したいとき。

## 起動方法

```
/systems-thinking-toolkit:variance-action
```

## 得られるもの

- 4 ノードのバランスループ診断（target / actual / variance / action）。変動方向と S/O リンクラベルを明示。
- 遅延を考慮した推奨案。ループが収束途上または過剰補正が失敗モードのときは「何もしない」選択肢も含む。
- 上位向けの説明根拠。抑制が「不作為」と誤読されないようにする。

## 境界

- 強化ループ（R ループ）や真に発散している系には使わない — `loop-and-link-primitives` を使うこと。
- 振動そのものが望ましい場合（設計された周期的挙動）や、検知遅延が実質ゼロの場合には使わない。
- フィードバックループが存在しない一回限り・非反復の意思決定には使わない。
