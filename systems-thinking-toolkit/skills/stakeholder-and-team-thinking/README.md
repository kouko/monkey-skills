# stakeholder-and-team-thinking

Multi-stakeholder CLD overlay (outward) + mental-model harmony with leadership-energy-pumping (inward). Two paired protocols sharing the same mental-model-surfacing machinery — opposite goals.

## When to use

- **Stakeholder conflict**: 2-4 groups stuck in a recurring fight (Engineering vs Product, Sales vs Operations, vendor vs customer) where compromise has failed.
- **Team mental-model drift**: individually excellent members but collective output is degrading; decisions take longer, code reviews turn contentious.
- **Post-merger alignment**: acquired and acquiring teams talking past each other on roadmap or culture — run outward then inward in sequence.

## How to invoke

```
/systems-thinking-toolkit:stakeholder
```

## What you get

- A **straddle policy** that improves every stakeholder's CLD beneficially (not a compromise midpoint — a third option neither side originally proposed).
- Named **mental-model alignment artifacts**: per-member CLDs with documented divergent S/O labels, missing nodes, and a negotiated convergence on fundamental values.
- A **leadership-energy continuous-input plan**: weekly+ sustained practices (active listening, repeated value reinforcement, shared experience accumulation) — not a one-shot offsite.

## Boundaries

- **NOT a substitute for psychological-safety remediation.** If team members fear speaking up, surfacing mental models produces performative agreement, not genuine surfacing. Fix safety first (see Edmondson's work).
- **NOT for zero-sum games.** Fixed-pool budget splits, layoff target distribution, or M&A redundancy lack the slack a straddle requires; the "wise policy" language can sanitize a power play.
- Also skip for: single-decision-maker problems, key stakeholders absent from the room, short-term task forces, remote/async-first teams (the 2002 energy-pump metaphor assumes co-presence), and cognitive-diversity-by-design teams (red teams, audit) where friction is the point.
