# stakeholder-and-team-thinking

多利害關係人因果迴路圖疊圖（向外）＋ 心智模型和諧搭配領導力能量泵浦（向內）。兩個成對協定共用同一套心智模型 surfacing 機構，目標卻完全相反。

## 適用時機

- **利害關係人衝突**：2 至 4 個群體陷入反覆爭執（Engineering vs Product、Sales vs Operations、供應商 vs 客戶），妥協方案已經失敗。
- **團隊心智模型漂移**：個人能力一流，但集體產出在退化；決策變慢、code review 變得火藥味十足。
- **併購後對齊**：收購方與被收購方在 roadmap 或文化上各說各話；先跑向外、再跑向內。

## 啟動方式

```
/systems-thinking-toolkit:stakeholder
```

## 你會得到

- 一個能讓所有利害關係人因果迴路圖都得到改善的 **straddle 政策**（不是妥協中點，而是雙方原本都沒想到的第三選項）。
- 具名的 **心智模型對齊產出物**：每位成員的因果迴路圖、S/O 標籤分歧與缺漏節點記錄、根本價值上的收斂協商結果。
- **領導力能量持續投入計畫**：每週以上、長期維持的實踐（積極聆聽、價值反覆強化、共同經驗累積），不是一次性的 offsite。

## 邊界

- **不能取代心理安全感的補強**。若成員害怕發言，surface 心智模型只會得到表演性的同意，不是真實的表達。先修復安全感（參考 Edmondson 的研究）。
- **不適用於零和賽局**。固定額度的預算分配、裁員名額分派、併購重疊整併都缺少 straddle 所需的餘裕；「賢明政策」的語言反而會掩蓋權力運作。
- 其他不適用情境：單一決策者問題、關鍵利害關係人不在場、短期任務小組、遠端／非同步優先團隊（2002 年的能量泵浦比喻假設共處一地）、刻意設計認知多樣性的團隊（紅隊、稽核）— 那裡摩擦本來就是目的。
