# innovaction-martian-test

> Warning: V1-weak auxiliary — typically reached from inside
> `strategy-lever-and-cascade` workflow when generating alternative
> futures. Martian-test feature enumeration + one-feature perturbation.

## When to use

- Trivial-variants problem inside sk08-style scenario planning ("same
  world, GDP 5% higher") — the Martian-test density bar is the
  operational completeness threshold that stops perturbations from
  being too thin to be productive.
- Perturbation-method ideation when no existing TRIZ practice — useful
  as a quick on-ramp for teams without an established systematic-
  innovation framework.

## How to invoke

```
/systems-thinking-toolkit:martian-test
```

## What you get

- Martian-test feature list (20-40 bullets at a density a context-free
  outsider could use to distinguish the thing from neighbors).
- Perturbed-feature scenarios (one feature flipped per pass, with
  3-5 downstream consequences and lever-setting implications).

## Warning: Boundaries

- **V1-weak per Stage 1.5** — Sherwood-branded, single-chapter coverage,
  no independent peer-reviewed comparison against alternatives.
- **TRIZ** (Genrich Altshuller, 1946+) and **morphological analysis**
  (Fritz Zwicky, ~1947) are stronger, empirically-grounded alternatives
  with decades of industrial / aerospace / foresight track record.
- If your team already practices TRIZ or morphological analysis, use
  those instead — this skill duplicates them with weaker tooling.
