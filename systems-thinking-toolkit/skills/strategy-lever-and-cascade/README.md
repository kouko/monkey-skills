# strategy-lever-and-cascade

**Languages**: [English](README.md) | [日本語](README.ja.md) | [繁體中文](README.zh-TW.md)

Turn ambition statements into mechanically actionable lever target changes; 3-timescale cascade + 3×N scenario planning.

## When to use

- Strategy off-site where the team needs to convert ambition into concrete input changes, not outcome announcements.
- Ambition-to-action gap — CEO declares a number ("25% margin", "double ARR"), the team cannot name which lever target settings would produce it.
- Multi-year plan refresh where the current 5-year plan is last year's plan with dates shifted (dead strategic loop).

## How to invoke

```
/systems-thinking-toolkit:strategy
```

## What you get

- A strategy artifact expressed as a list of lever target changes (lever name, current actual, current target, proposed target), not an outcome announcement.
- 3-4 plausible-not-probable 2030 worlds — no probability weights, no expected-value math.
- Per-lever classification: **robust** (set now), **scenario-contingent** (build optionality + define triggers), or **bet** (high-upside with named downside protection).

## Boundaries

- NOT for in-year tactical fixes — use `variance-target-action-template` at single-loop variance/target/action level instead.
- NOT a forecasting tool — scenarios exist to test lever settings, not to predict the world. If anyone asks for the probability of a scenario, the exercise has degenerated into forecasting.
