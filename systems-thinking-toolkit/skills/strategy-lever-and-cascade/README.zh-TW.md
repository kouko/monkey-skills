# strategy-lever-and-cascade

**Languages**: [English](README.md) | [日本語](README.ja.md) | [繁體中文](README.zh-TW.md)

把野心宣言（ambition statement）轉換為機械上可執行的槓桿目標值變更；三層串接 + 3×N 情境規劃。

## 何時使用

- 策略 off-site，團隊需要把野心轉成具體的輸入變更，而不是結果指標的宣告。
- 野心與行動的落差 — CEO 喊出一個數字（「毛利 25%」、「ARR 翻倍」），但團隊講不出哪些槓桿的目標設定要怎麼動才會產生這個結果。
- 多年期計畫更新時，目前的 5 年計畫只是去年計畫的日期往後挪（策略迴路已死的徵兆）。

## 如何呼叫

```
/systems-thinking-toolkit:strategy
```

## 你會得到

- 以槓桿目標值變更清單（槓桿名稱、目前實際值、目前目標值、提議的新目標值）表達的策略成果物，而不是一份結果宣告。
- 3〜4 個 plausible-not-probable（合理但不賦予機率）的 2030 世界 — 不允許機率權重，不允許期望值計算。
- 每根槓桿的分類：**穩健型**（現在就設定）、**情境相依型**（建立選擇權 + 定義觸發點）、**下注型**（單一情境上方很大，其餘可接受，必須明示下方保護）。

## 邊界

- 不適用於年度內的戰術調整 — 單一迴路的變異 / 目標 / 行動請改用 `variance-target-action-template`。
- 不是預測工具 — 情境是用來測試槓桿設定，不是用來預測世界。一旦有人問「這個情境的機率是多少」，演練就已經退化為預測。

## 完整規格

完整的 Reading → Interpretation → Application → Execution → Boundary 結構、9 步驟執行流程，以及 Sherwood 列出的失敗模式請見 [SKILL.md](SKILL.md)。

## 出處

Dennis Sherwood, *Seeing the Forest for the Trees* (2002), Chapter 10（槓桿 vs 結果的商業引擎 + 三層串接 + 3×N 情境表）、Chapter 12（以存量重新定義目標）、Chapter 13（汽車經銷商槓桿模擬 + 把野心當作輸入端的目標懸垂端而非預測輸出）。
