# strategy-lever-and-cascade

**Languages**: [English](README.md) | [日本語](README.ja.md) | [繁體中文](README.zh-TW.md)

野心表明（ambition statement）を機械的に実行可能なレバー目標値の変更に変換する。三段カスケード + 3×N シナリオプランニング。

## 使うべきとき

- 戦略オフサイトで、チームが野心をアウトカム宣言ではなく具体的な入力変更に変換する必要があるとき。
- 野心と行動のギャップ — CEO が数値を宣言するが（「マージン 25%」「ARR 倍増」など）、チームはどのレバーのターゲット設定を動かせばそれを生み出せるか言語化できない。
- 複数年計画のリフレッシュで、現行 5 年計画が前年の計画の日付をずらしただけ（戦略ループが死んでいる状態）のとき。

## 起動方法

```
/systems-thinking-toolkit:strategy
```

## 得られるもの

- アウトカム宣言ではなく、レバー目標値の変更リスト（レバー名・現在の実績・現在のターゲット・提案するターゲット）として表現された戦略成果物。
- 3〜4 個の plausible-not-probable（蓋然的ではあるが確率付けされていない）2030 年シナリオ — 確率重み付け禁止、期待値計算禁止。
- レバーごとの分類：**ロバスト**（今すぐ設定）、**シナリオ依存**（オプション性を構築 + トリガーを定義）、**ベット**（上方は大きいがダウンサイド防御を明示）。

## 適用範囲外

- 単年度の戦術的修正には使わない — 単一ループの変動 / ターゲット / アクションには `variance-target-action-template` を使う。
- 予測ツールではない — シナリオはレバー設定の検証のために存在し、世界を予測するためのものではない。「このシナリオの確率は？」という問いが出た時点で、演習は予測に退化している。
