# cld-archetypes

R/B 注釈付き Mermaid 因果ループ図を入力に、どの Sherwood アーキタイプが現れているか（成長極限の R+B カップリング、または遅延付き B ループの V/T/A）を判定し、対応する介入プレイブックを適用する。

## 使うべきタイミング

- 投資や採用を増やしているのに成長が減速し、チームの反射が「同じレバーをさらに強く踏む」になっているとき — 予算を倍にする前にリミッツ・トゥ・グロース診断が必要。
- KPI が四半期ごとに揺れ続け、介入のたびに振幅が広がっているとき — 次の修正を入れる前に V/T/A 診断が必要。
- 因果ループ図がすでに描かれており（`loop-and-link-primitives` で R/B 分類済み）、介入哲学を選ぶ前にアーキタイプを命名したいとき。

## 起動方法

`/systems-thinking-toolkit:archetypes`、またはルーター `/systems-thinking-toolkit:stt` 経由。

## 得られるもの

- 因果ループ図上に命名されたアーキタイプ（`limits-to-growth` または `V/T/A`）、リミッツ・トゥ・グロース側は拘束している制約、V/T/A 側はフィードバック遅延の推定値付き。
- 対応する介入ルール — 拘束している B ループから「ブレーキを外す」か、B ループが収束する間「1 フィードバックサイクルは何もしない」か — コスト / タイムライン / 反証可能な予測を添えて。
- 下流スキルへのポインタ（次に動かすレバーを決めるなら `strategy-lever-and-cascade`、定量的確認が必要なら `simulation-modeling`）を `%% Archetype: ...` Mermaid コメントの下に付加。

## 適用範囲外

- 真に発散しているシステム（無限に走り去る悪循環の R ループ）には使わない — ここで「何もしない」を適用するのは誤適応。`loop-and-link-primitives` の R ループ診断に戻すこと。
- どのブレーキが拘束しているかを確認する前に「ブレーキを外す」許可証として使ってはならない — 違うブレーキを外せば崩壊を加速しうる（イースター島 ce07 の失敗モード）。
- PMF 前で成長エンジンがまだ回っていないケース、繰り返さない一回限りの意思決定、検出遅延がゼロのリアルタイム制御、許容範囲内のランダムノイズには使わない（代わりに SPC / 管理図ツールを使う）。
