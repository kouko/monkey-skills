# cld-overlay

每個利害關係人觀點各畫一張 Mermaid 因果迴路圖，疊放在同一張共用畫布上，找出同時改善所有因果迴路圖的政策（straddle finding）。Stop forcing, start listening.

## 何時使用

- 兩到四個利害關係人群體陷入反覆出現的衝突，調停一輪又一輪，每季、每次回顧會議都重演同一場爭執。
- 你需要一條跨越各方觀點的賢明政策（wise policy） — 一個沒有任何一方提出過的第三選項，因為它原本不存在於任一方自己的因果迴路圖中。
- 場景是調停、合作關係修復、買賣雙方關係惡化、或併購整合，且各方的心智模型在自己的框架內各自內部一致。

## 如何啟動

`/systems-thinking-toolkit:overlay` 或透過路由器 `/systems-thinking-toolkit:stt`。

## 你會得到什麼

- 每個利害關係人觀點各一張、分別獨立繪製的因果迴路圖（遵循 `cld-craft` 紀律 — S/O 符號用筆畫上、模糊變數提升為一級節點、由繪製群體自己認可貼近現實）。
- 一份重疊圖（紙本、白板，或一個 Mermaid `flowchart`，每位利害關係人各佔一個 `subgraph`），標註三項特徵 — 共用節點但 S/O 衝突、僅出現在單側的節點、跨圖同向旋轉的強化迴路。
- 一條已命名的 straddle 政策，逐張穿越所有利害關係人的因果迴路圖追蹤過、並在各自框架內驗證為有益 — 加上一份運作計畫，或交棒給 `team-mental-model`。

## 邊界

- 不用於固定資源上的零和爭議（預算切分、人力配額、固定獎金池分配） — straddle 需要第三維度的餘裕,而那裡並不存在。
- 不用於關鍵利害關係人不在場的情況（顧客、主管機關、工會、下游使用者） — 缺席方未在場時所畫出的內部優雅 straddle,一旦碰到缺席方就會崩潰。
- 不用於單一決策者的問題,或一方可逕行強加的權力不對稱情境 — 先把權力結構攤開來,不要用賢明政策的說法粉飾過去。
