# cld-overlay

ステークホルダーの観点ごとに Mermaid 因果ループ図を 1 枚ずつ描き、共有キャンバス上で重ね合わせ、すべての因果ループ図を改善する政策（straddle finding）を見つけます。Stop forcing, start listening.

## いつ使うか

- 2〜4 つのステークホルダー集団が繰り返される対立に行き詰まり、調停を重ねても四半期ごと・レトロごとに同じ争いが戻ってくるとき。
- 観点をまたいで機能する賢明な政策（wise policy）が必要なとき — どの集団も自分の因果ループ図に含めていなかったがゆえに提案できなかった第三の選択肢。
- 文脈が調停・パートナーシップの修復・ベンダーと顧客の対立悪化・M&A 統合であり、各側のメンタルモデルがそれぞれのフレーム内では内部整合的であるとき。

## 起動方法

`/systems-thinking-toolkit:overlay` またはルーター `/systems-thinking-toolkit:stt` 経由。

## 得られる成果物

- ステークホルダーの観点ごとに 1 枚ずつ、別々に描かれた因果ループ図（`cld-craft` の規律に従う — S/O 符号をペンで、ファジー変数を一級ノードに昇格、描いた集団が「現実を捉えている」と承認）。
- 重ね合わせ図（紙・ホワイトボード、またはステークホルダーごとに `subgraph` を持つ Mermaid `flowchart`）。3 つのシグネチャを注記する — 共有ノードで S/O が衝突する箇所・片側にしか存在しないノード・同期した強化ループ。
- 各ステークホルダーの因果ループ図を貫いてトレースされ、それぞれのフレーム内で有益と検証された straddle 政策（命名済み） — および運用計画、もしくは `team-mental-model` への引き継ぎ。

## 境界

- 固定資源上のゼロサム対立（予算配分・人員枠・固定原資のボーナス）には使わない — straddle は第三の次元の余地を必要とし、そこには存在しない。
- 主要ステークホルダーが部屋にいないとき（顧客・規制当局・労働組合・下流の利用者）は使わない — 不在の当事者抜きで描かれた内部整合的な straddle は、不在側に接触した瞬間に崩れる。
- 単一の意思決定者しかいない問題や、一方が一方的に押し付けられる権力非対称な状況には使わない — 賢明な政策として粉飾せず、まず権力構造を表面化する。
