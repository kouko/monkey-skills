# Protocol: Write Short-Form Copy（キャッチコピー Canon 短文案撰寫）

**When to use**：短文案需求。典型情境為 キャッチコピー（広告見出し）、タグライン、バナー広告、SNS 投稿本文、CM タイトル、雑誌 / 新聞広告の見出し、Web サイトのメインビジュアル上に載る一文。字數帯 7-15 字を default、15 字超は リード文 / 副題 として別扱い。

**Output**：7-15 字黃金範囲内の短文案 artifact。5 種切入点から 1 つを選定し、voice 巨匠のいずれかを参照、「なぜ良いか」3 項を付帯する。3 秒以内で読者が A+I（Attention + Interest）を完結する設計。

**Grounds on**：`../standards/short-form-catchcopy-canon.md`、`../standards/ideation-taniyama-discipline.md`、`../standards/voice-and-tone.md`、`../standards/persuasion-ethics.md`

## Phase 1: 5 種切入点選定

`short-form-catchcopy-canon.md` §5 種切入点決策樹 に従い、product + audience + 訴求目標から 1 つの切入点を選定する。

1. **product / audience / 訴求目標の整理**：
   - `copy-ideation-parallel.md` Phase 3 で handoff された headline seed + 「なぜ良いか」3 項 を起点素材として確認。
   - 商品が解決策寄りか、新カテゴリか、ターゲットが狭いか広いか、読者が広告慣れしているかを判定。

2. **決策樹による選定**（`short-form-catchcopy-canon.md` §5 種切入点決策樹）：

| 切入点 | 選定条件 | 典型構文 |
|---|---|---|
| **利益 / 願望** | 商品が「解決策」寄り + 正の結果追求が響く audience | 「〜になりませんか」「〜を手に入れよう」 |
| **恐怖 / 痛点** | 負の回避訴求が響く audience（保険 / 健康 / セキュリティ） | 「このままで大丈夫？」「〜を失う前に」 |
| **顛覆常識** | 商品が「新カテゴリ」+ 眠った受眾を起こす必要 | 「誰が〇〇は△△と決めた？」 |
| **目標呼喚** | ターゲットが狭い（B2B / ニッチ / 専門サービス） | 「〜したい、あなたへ」「〇〇世代のための」 |
| **提問互動** | 読者が広告慣れ + 好奇心喚起が効く | 「なぜ〇〇は〇〇なのか？」 |

3. **倫理境界確認**（`persuasion-ethics.md` §Dark Pattern 反模式清單）：
   - 恐怖訴求を選んだ場合、`persuasion-ethics.md` §景品表示法要點 + §小霜和也「嘘をつかない」原則 で過度な不安商法に陥っていないかを事前確認。
   - 「誇大しない」「benefit を捏造しない」「煽りを基本戦略としない」の小霜原則を本 Phase で先行 check。

4. **記録**：選定切入点と選定理由を草稿冒頭にメタ記述する。

## Phase 2: AIDMA 短文版作用域の確認

`short-form-catchcopy-canon.md` §AIDMA 短文案作用域 に従い、**短文案は A+I（Attention + Interest）専用**という canonical 分業を再確認する。

1. **A+I 専担の確認**：短文案単独で D（Desire）/ M（Memory）/ A（Action）までカバーしようとしていないか。
   - D は body copy が担う
   - M は反復露出 / ジングル が担う
   - A は CTA / LP が担う
   - 短文案で全 5 段階を詰め込むと「字数オーバー（15 字超）」または「説得力不足」のいずれかに陥る。

2. **3 秒ルール対照**（`short-form-catchcopy-canon.md` §3 秒ルール）：
   - **1 秒目**：キャッチ + メインビジュアルで興味喚起（「これは何だ」と立ち止まらせる）
   - **2 秒目**：一番伝えたい情報（「自分に関係がある」と理解させる）
   - **3 秒目**：次の行動誘発（「もっと見たい / クリックしたい」）
   - SNS 媒体（TikTok / Reels / Shorts）なら 1-3 秒に縮小。
   - **3 秒ルールは formula ではなく heuristic**。段階設計として強制適用しない。

3. **媒体に応じた AIDMA / AISAS / ULSSAS 使い分け**（`short-form-catchcopy-canon.md` §SNS 時代の演化）：
   - B2B / TV CM / 店頭販促 / 雑誌 → AIDMA 主流
   - SNS / e-commerce → AISAS（2004 電通）/ ULSSAS（2015 ホットリンク）主流
   - いずれの framework でも短文案の A+I 担当範囲は不変。

## Phase 3: 字數紀律 — 7-15 字黃金範囲

`short-form-catchcopy-canon.md` §7-15 字 黃金範圍 に従い、字數を厳守する。

1. **7-15 字 canonical 根拠**：
   - 人間の短期記憶スパン（マジカルナンバー 7±2）
   - スマートフォン画面で 1 行に収まる上限
   - 俳句 17 音感覚との近接
   - 4 人の巨匠実測値：糸井 7 / 岩崎 12-14 / 眞木 10 / Yahoo トピックス 13

2. **境界処理**：
   - 15 字を超える場合は「リード文 / 副題 / ボディコピー」として扱う。
   - 超過が必要なのは長文案の headline（PASONA の P 段冒頭など）で、その場合でも 15 字超の理由を明記。
   - 7 字未満（1-6 字）も可能（例：「生茶」2 字）が、リズム / 意味密度が問われる。

3. **字數測定方法**：
   - 全角 1 文字 = 1 字カウント（半角は 2 文字 = 1 字）
   - 句読点（、。）は字数に含む
   - 記号・空白の扱いは媒体規約に従う（Yahoo!トピックス は半角 13 字 canonical）

4. **字數超過時の対応**：
   - 削れる語はないか（谷山「磨く」の第 1 着眼点、`ideation-taniyama-discipline.md` §階段 3）
   - 助詞 1 文字で縮められないか
   - 2 文を 1 文にできないか（句読点削減）

## Phase 4: 「なんかいいよね禁止」審査

`ideation-taniyama-discipline.md` §「なんかいいよね禁止」ルール を候補全件に適用する。

1. **「なぜ良いか」3 項の必須明記**：採用候補 X ごとに以下フォーマットで記述：
   ```
   採用候補：X
   なぜ良いか：
   1. [具体理由 1：何が読者に何を伝えるか]
   2. [具体理由 2：既存の類似コピーに対して何が新しいか]
   3. [具体理由 3：ターゲットの生活 / 文脈で何が共鳴するか]
   ```

2. **描写 vs 解決の判定**（`ideation-taniyama-discipline.md` §描写 vs 解決）：
   - 描写型（NG）：「この化粧水は保湿力が高い」 — 商品特性の言い換え
   - 解決型（OK）：「明日の朝、鏡を見るのが楽しみになる化粧水」 — 生活への意味提示
   - 3 項には「これは描写ではなく、解決 / 新しい意味提示である」ことを証明する理由を含める。

3. **3 項を具体化できない候補は採用しない**。できるまで粘るか、捨てる。

4. **書き手視点の自己審査**：自分の発想ルートを知っているから良く見えるだけではないか（`ideation-taniyama-discipline.md` §階段 2 tip「自分の最初のお気に入りを疑う」）。

## Phase 5: Voice 巨匠対照

`short-form-catchcopy-canon.md` §四位 voice 巨匠 + `voice-and-tone.md` §JP 情緒共鳴傳統 を参照し、候補が**いずれかの voice lineage に属するか、または意図的に外れているか**を明記する。

| 巨匠 | lineage 特徴 | 継承点 |
|---|---|---|
| **糸井重里** | 状態提案型 / 意図的曖昧さ / 生活語彙の詩的再配置 | 命令ではなく状態提示、解釈余地 |
| **岩崎俊一** | 人生観 / 仏教無常観 / 季節感 | 有限性、いのち、「見つける」哲學 |
| **眞木準** | 掛詞 / 音韻 / ダジャレではなくオシャレ | 同音異義、和歌以来の音遊び |
| **谷山雅計** | 入口設計 / 31 訓練 | 体質としての発想力、言語化の徹底 |

1. **参照 lineage の選定**：voice guide（`voice-and-tone.md` §Brand Voice Guide）で既定済みなら従う。未定義ならこの Phase で brand との親和性を検討。

2. **Anglo headline formula の直接適用回避**（`short-form-catchcopy-canon.md` §Anglo vs JP 分界）：
   - Caples の headline types や Ogilvy の headline rule を JP 短文案に直接適用しない。
   - 情緒共鳴 / 掛詞 / 余韻 が欠落した「翻訳調の意味明示型」コピーに陥る。

3. **Ogilvy headline の compression density は参考化**（`voice-and-tone.md` §Ogilvy 長文案 voice 經典原則）：「5× more read the headline」から headline の密度基準は学べるが、voice そのものを Anglo 化しない。

## Rules

- 7-15 字黃金範囲を default とする。超過は明示的理由が必要（`short-form-catchcopy-canon.md` §7-15 字 黃金範囲）。
- 短文案は A+I 専担。D/M/A を 1 行で済ませようとしない（`short-form-catchcopy-canon.md` §AIDMA 短文案作用域）。
- 3 秒ルールは heuristic。formula として段階化しない（`short-form-catchcopy-canon.md` §3 秒ルール canonical 注意）。
- 全候補に「なぜ良いか」3 項を必須付帯。3 項が書けない候補は採用しない。
- 描写型コピーで止めない。解決 / 新しい意味提示 まで届かせる（`ideation-taniyama-discipline.md` §描写 vs 解決）。
- リゲイン「24 時間戦えますか？」を岩崎俊一に帰属しない。canonical は杉山恒太郎 + リゲイン PJ（`short-form-catchcopy-canon.md` §drift #8）。
- TCC 年鑑作品を模倣 / 翻案しない。参照して自分の voice を磨く が canonical。
- 恐怖訴求を選ぶ場合、`persuasion-ethics.md` §景品表示法要點 + §小霜原則 で過度な不安商法に陥っていないかを確認。

## Anti-Patterns

- **15 字超を「キャッチコピー」と呼ぶ**：リード文 / 副題 / ボディコピー として扱うのが canonical 分界（`short-form-catchcopy-canon.md` §Anti-Patterns）。
- **3 秒内に主訊息が届かない設計**：情報密度が低すぎて読者の判断基準を満たさない。
- **Anglo headline formula を JP 短文案に強制適用**：情緒 / 掛詞 / 余韻 が毀損される。
- **リゲイン「24 時間戦えますか？」を岩崎俊一帰属で引用**：drift #8 違反。canonical は杉山恒太郎 + リゲイン PJ。
- **1 つの短文で AIDMA 全段階カバー**：字数オーバーまたは説得力不足のいずれかに陥る。
- **3 秒ルールを formula として段階化**：heuristic であり段階公式ではない。
- **TCC 年鑑作品を模倣 / 翻案する**：voice 固有性を毀損する。
- **AISAS / ULSSAS を AIDMA の完全代替とする**：B2B / TV CM / 店頭 では AIDMA が主流。媒体で使い分け。
- **「なんかいいよね」で候補を採用する**：3 項理由書が書けない候補を選ぶと、再現性のない偶然の一致に終わる（`ideation-taniyama-discipline.md` §Anti-Patterns）。
- **描写コピーで止まる**：「〇〇は△△です」で完結する。商品特性の言い換えで新しい意味提示が欠落。
- **書き手視点で選ぶ**：自分の発想ルートを知っているから良く見えるだけの候補を優勝に選ぶ。
- **最上級表示（「最高」「No.1」「世界初」）を根拠なしに短文で使う**：景品表示法 優良誤認 / 有利誤認 の頻出違反。15 字制約下でも根拠が前提。
