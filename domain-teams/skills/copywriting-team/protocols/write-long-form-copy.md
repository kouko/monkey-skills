# Protocol: Write Long-Form Copy（PASONA Canon 長文案撰寫）

**When to use**：長文案需求。典型情境為 landing page（LP）、sales letter、セミナー / ウェビナー勧誘ページ、メルマガ連載、資料請求トリガー文、記事広告（advertorial）、長尺 CM スクリプト。字數帯 1,500 字以上。

**Output**：依選定 framework 段階式に構成された長文案 artifact。各段に任務 / 字數比例 / 離脱防止 transition を明示。最終的に倫理 self-check を経由。

**Grounds on**：`../standards/long-form-pasona-canon.md`、`../standards/persuasion-psychology-anchor.md`、`../standards/voice-and-tone.md`、`../standards/persuasion-ethics.md`

本 protocol の前工程として `copy-ideation-parallel.md` の Phase 3 handoff が推奨される（特に Affinity / Benefit seed 素材として）。ideation を先行しない場合は、本 Phase 1 で framework 選定と同時に Problem + Affinity の起点素材を用意する必要がある。

## Phase 1: Framework 選擇

`long-form-pasona-canon.md` §三框架の適用表 と `persuasion-psychology-anchor.md` §Schwartz 5 Levels of Awareness を掛け合わせて framework を選定する。

1. **字數帯 による 1 次決策**：
   - 1,500–5,000 字 → 旧 PASONA（5 段階 P/A/So/N/A、So 合體）
   - 3,000–10,000 字 → 新 PASONA（6 段階 P/A/S/O/N/A、Affinity 起点）
   - 5,000–20,000+ 字 → PASBECONA（9 段階、B/E/C 挿入）
   - 字數帯が重なる境界（3,000–5,000 / 5,000–10,000）は次の 2 次決策で確定。

2. **Awareness level による 2 次決策**（`persuasion-psychology-anchor.md` §Schwartz 5 Levels）：
   - Level 1-2（Most / Product Aware）→ 旧 PASONA or 新 PASONA 圧縮版
   - Level 3（Solution Aware）→ 新 PASONA
   - Level 4（Problem Aware）→ 新 PASONA 厚め or PASBECONA
   - Level 5（Unaware）→ PASBECONA（Problem + Affinity + Benefit + Evidence フル長文）
   - **core rule（Schwartz 1966）**：Unaware 読者に直接 Offer closer を使わない。必ず narrative で problem discovery を経由。

3. **トーン境界 による除外判定**：
   - BtoB / 医療 / 金融 / 高額コーチング → 旧 PASONA 非推奨（Agitation 煽り が canonical トーンと整合しない）。
   - 新 PASONA / PASBECONA の Affinity 起点を default とする。

4. **記録**：選定 framework 名、字數目安、awareness level 診断、選定理由を草稿冒頭に明記する。後工程（evaluator / audit）が検証可能な形で残す。
   - 例：「framework = 新 PASONA / 字數 5,000 字 / Schwartz Level 3 (Solution Aware) / 選定理由：商品カテゴリ既知の audience、Affinity 起点で 共感ベースの教育型 LP を構築する」

5. **Framework 決策の例外ケース**：
   - awareness level が複数 segment に分かれる（一部 Level 2、一部 Level 4）→ 高い方に合わせて PASBECONA フル長で Level 4-5 もカバーするか、segment 別に LP を分岐するかを planning-team と協議。
   - 字數制約が外部から課されている（広告媒体の規定字数）→ 字數優先で framework を選び、段階圧縮の方針を記録。
   - 既存 LP のリニューアルで framework 切替が必要 → `copy-audit.md` Phase 3 の framework 切替提案を先行し、本 protocol で新 framework で再構築。

## Phase 2: 段階式草稿

選定 framework の各段を、`long-form-pasona-canon.md` の定義通りに逐段執筆する。

1. **字數比例の設定**（`long-form-pasona-canon.md` §段階間 flow 設計原則 字數比例）：
   - 旧 PASONA：P:A:So:N:A ≒ 2:2:3:2:1
   - 新 PASONA：P:A:S:O:N:A ≒ 1:1:1:1:0.5:0.5（基本）
   - PASBECONA（5,000 字）：P:A:S:B:E:C:O:N:A ≒ 1:1:1:1:2:2:1:0.5:0.5

2. **段階任務の遵守**：各段は `long-form-pasona-canon.md` §旧 PASONA / 新 PASONA / PASBECONA 表 の「該段階の任務」に従う。段階を飛ばさない、任務を混ぜない。

3. **Affinity 段の特別規範**（新 / PASBECONA）：
   - Problem と**同等以上の分量**を割く（`long-form-pasona-canon.md` §段階間 flow 設計原則）。
   - 煽り表現（「このままでは手遅れになります」）を使わない。Affinity は「私も同じでした」で始まる並走型。
   - `persuasion-psychology-anchor.md` §Cialdini Liking が心理学 anchor。
   - JP 案件では 糸井 / 岩崎 voice を参考に「状態提案」「季節感」の情緒共鳴を検討（`voice-and-tone.md` §JP 情緒共鳴傳統）。
   - `copy-ideation-parallel.md` Phase 3 の handoff「Affinity seed」を Phase 2 の起点素材として使う。

4. **Solution → Benefit → Evidence → Contents（PASBECONA 専用）の flow**：
   - 抽象（Benefit）→ 客観（Evidence）→ 具体（Contents）の順序が canonical。逆順は認知負荷を上げる（`long-form-pasona-canon.md` §PASBECONA B/E/C 挿入の論理）。
   - Evidence は 3 種以上を組合（専門家 endorsement / 実績数値 / 顧客の声、`persuasion-psychology-anchor.md` §Cialdini Authority + Social Proof）。

5. **Offer 段**：価格・保証・特典を具体的に提示。**省略禁止**（旧 PASONA 以外）。Narrow down と隣接させる。

6. **Voice の一貫性**：各段で brand voice guide（`voice-and-tone.md` §Brand Voice Guide の Checklist）に対照。headline / body / Offer / CTA で voice が切断しない（`voice-and-tone.md` §Anti-Patterns「Headline と body voice 斷裂」）。
   - Voice 4 軸 position（Formality / Seriousness / Respectfulness / Enthusiasm）が全段一貫しているか。
   - Tone は情境依存で調整可：Problem → matter-of-fact 寄り、Affinity → respectful / 共感寄り、Narrow down → urgent だが扇動ではない、Action → direct。
   - JP 案件で Affinity 段を糸井 / 岩崎 voice で書く場合、Offer / Action 段で急に Anglo 的 direct voice に切替わらないよう注意（`voice-and-tone.md` §JP 情緒共鳴傳統 vs Anglo 分界）。

## Phase 3: 段階間 flow 検証（離脱防止 checkpoint）

`long-form-pasona-canon.md` §段階間 flow 設計原則 に従い、段階移行の境目を重点検査する。

1. **離脱点の明示**：P→A, A→S, S→B, B→E, E→C, C→O, O→N, N→A の各境界で「次の段階への橋渡し」が明示されているか。接続詞 / 小見出し / 問いかけ のいずれかで transition を設計。

2. **各段末の読者質問を自問**：「読者はこの段を読み終えた時、続きを読みたいか？」「読者は今どの疑問を抱えているか？」次段がその疑問に応えているか。

3. **Narrow down → Action の近接**：N と A の間に別段を挟まない。限定性と CTA が離れると緊急性が崩れる。

4. **System 1 / System 2 のバランス**（`persuasion-psychology-anchor.md` §Kahneman System 1 / System 2）：
   - 短文案で System 1 を掴む headline / opener
   - body で System 2 を納得させる（論点・証拠・論理一貫性）
   - closing で再び System 1 の情緒に戻る（Narrow down + Action）
   - どちらか一方に偏ると「感情は動くが購入しない」「論理は納得するが行動しない」になる。

5. **Psychology anchor マッピング検証**（`persuasion-psychology-anchor.md` §PASONA フレームワークの心理学 anchor マッピング）：
   - Problem 段 → Schwartz Level 4→3 移動 + Kahneman System 1 (痛みの瞬間喚起)
   - Affinity 段 → Cialdini Liking + JP 情緒 voice (糸井 / 岩崎)
   - Solution 段 → Schwartz Level 3→2 移動 + System 2 (論理的妥当性)
   - Benefit 段 (PASBECONA) → Kahneman prospect theory (gain frame) + Authority
   - Evidence 段 (PASBECONA) → Cialdini Social Proof + Authority
   - Offer 段 → Kahneman loss aversion + Cialdini Reciprocity
   - Narrow down 段 → Cialdini Scarcity（真実性必須）
   - Action 段 → Cialdini Commitment & Consistency + Nudge default
   - 各段が対応 anchor を**意識的に使って**いるか、単に段階任務を表面的に満たしているだけかを自問する。

## Phase 4: 倫理 self-check

`persuasion-ethics.md` §景品表示法要點 + §FTC Endorsement Guides 要點 + §Dark Pattern 反模式清單 を順に self-check する。最終の gate 判定は evaluator が行う。本 phase は worker 側の self-audit。

1. **景品表示法（JP 市場）**：
   - 優良誤認表示（§5-1）— 「業界最高」「世界初」「No.1」などの最上級表示に根拠があるか。
   - 有利誤認表示（§5-2）— 二重価格表示・虚偽の期間限定・隠れコストがないか。
   - 打消し表示 — メイン訴求を実質的に否定する disclaimer になっていないか。
   - ステマ告示（2023-10-01 施行）— 広告性を隠した表現になっていないか。

2. **FTC Endorsement Guides（米国市場）**：
   - Testimonial の代表性（§255.2）— 非典型 case のみ並べていないか。
   - Material connection disclosure（§255.5）— endorser との関係が開示されているか。

3. **Dark pattern 自查**（`persuasion-ethics.md` §Dark Pattern 反模式清單）：
   - False scarcity（虚偽の限定）→ Narrow down 段は**真実性必須**
   - Fake social proof（捏造 testimonial）→ Evidence 段は検証可能な実績のみ
   - Hidden costs（隠れコスト）→ Offer 段で全料金開示
   - Forced continuity（解約困難な強制継続）→ 「いつでも解約」と書くなら実態も然り

4. **小霜原則**（`persuasion-ethics.md` §小霜和也「嘘をつかない」原則）：
   - 誇大しない / benefit を捏造しない / ユーザーの声を偽造しない / 煽りを基本戦略としない。

5. **ethics self-check 結果を草稿末尾にメタ記述**（evaluator の参照用）。

## Rules

- 段階を飛ばさない。旧 PASONA 以外で Offer を省略しない、Narrow down と Action を隣接させる（`long-form-pasona-canon.md` §Anti-Patterns）。
- Affinity を Agitation の言い換えとして書かない。「不安を感じていませんか？」は Agitation 寄り、「私も同じでした」が Affinity。
- B/E/C 段の creditor を神田 or 衣田 単独と断定しない。共著表記に留める（`long-form-pasona-canon.md` §B/E/C 歸屬の uncertainty）。
- Schwartz awareness level 診断を Phase 1 で明示する。診断なしに framework を選定すると「Level 5 に直接 Offer」のような致命傷が起こる（`persuasion-psychology-anchor.md` §Anti-Patterns）。
- voice guide との整合性を各段で確認する。framework 出力と voice guide が衝突したら voice guide 優先（`voice-and-tone.md` §PASONA / BEAF / キャッチコピー 框架的組合）。
- 最上級表示・二重価格・fake testimonial を使わない。景品表示法 / FTC 違反は evaluator gate で必ず拒否される。
- 旧 PASONA を BtoB / 医療 / 金融 / 高額コーチング で default 選択しない。

## Anti-Patterns

- **1999 年を PASONA 書籍首載として引用する**：2016 年『稼ぐ言葉の法則』が canonical（`long-form-pasona-canon.md` §drift #4）。
- **PASBECONA を 2016 年命名とする**：2021 年『コピーライティング技術大全』が canonical（`long-form-pasona-canon.md` §drift #5）。2016 年は新 PASONA の年。
- **Affinity 段を薄く済ませる**：Problem と同等以上の分量が canonical。薄い Affinity は「共感風の台詞」に堕ちる。
- **PASBECONA で Contents を Benefit の前に置く**：抽象 → 客観 → 具体 の順序違反。スペック先出しは認知負荷増。
- **煽りを長文で通す**：新 PASONA 以降の canonical は Affinity 起点。煽り強化は神田自身が 2016 年に反省した経路。
- **Unaware 読者に直接 Offer を closer として使う**：Schwartz awareness ladder を飛ばすと conversion が崩壊する。
- **倫理 self-check をスキップする**：「evaluator が見るから worker は書くだけ」は責任放棄。景品表示法 / FTC 違反は worker 段階で除去しておくべき。
- **短文案と同じ voice で長文を書く**：短文の圧縮密度を長文で延伸すると同じ表現の自己反復になる（`voice-and-tone.md` §Anti-Patterns「headline と body voice 断裂」の裏面）。
- **最上級表示を根拠提示なしに使う**：「業界 No.1」「世界初」「最安値」は景品表示法 優良誤認 / 有利誤認の頻出違反類型（`persuasion-ethics.md` §Anti-Patterns）。
