# KJ法 収束發想法

Tier 3 standard: 自包含。以川喜田二郎 1967 年『発想法』（中公新書 136）
為 canonical source，1970 年『続・発想法』（中公新書 210）為擴充、1986 年
『KJ法』為完整体系化。本檔為 copywriting-team「選ぶ（收斂）」階段的
標準工具：承接 Mandal-Art / Verbalized Sampling 發散階段產出的 64-81
候選，經由 6 階段運作收斂至 3-5 個優勝。核心原則為「渾沌をして語らしめる」
— 不由上而下分類，而由感性親和性逐層聚類，讓資料自己湧現結構。

- 川喜田二郎 (1970) 『続・発想法 —— KJ法の展開と応用』中公新書 210 — 6 階段運作の精細化と「狭義 / 広義 KJ 法」区分。
- 川喜田二郎 (1986) 『KJ法 —— 渾沌をして語らしめる』中央公論社 — 完整体系化。「渾沌をして語らしめる」原則の定式化。
- Wikipedia 日本語版「KJ法」 https://ja.wikipedia.org/wiki/KJ法 — 次級 dual-verify source：川喜田 1967 初出、狭義 4 階段 / 広義 6 階段、ネパール フィールドワーク起源。
- Swing Root「KJ法とは」 https://swingroot.com/kj-method — 次級 dual-verify source：6 階段の実務手順と表札づくりの感性親和性説明。
- UX Times「KJ法の基礎」 — 次級 dual-verify source：UX リサーチ文脈での 6 階段運用とグループ編成の失敗例。

## Framing

KJ法 是**収束專用**的發想法，由川喜田二郎在 1960 年代ネパール フィールドワーク
的資料整理經驗提煉而成。它與 brainstorming 的互補關係明確：brainstorming
（発散）產出大量素材、KJ法（収束）把素材統整為可行動的洞察。

核心原則「**渾沌をして語らしめる**」(chaos speaks for itself) 規定：
不由研究者事先定分類框架，而由資料彼此的感性親和性**自下而上**聚類。
這與 top-down 分類法（decision tree / taxonomy）的根本差異在於：
KJ法 保留了**資料自己的邏輯湧現空間**，避免研究者既有認知把未發覺的
模式壓平。

在 copywriting pipeline 中，本工具用於：
- 消費者訪談逐字稿的整理（從大量 raw 發言抽取洞察）
- 文案角度的收斂（從 Mandal-Art 81 格或 Verbalized Sampling 64 候選
  篩出 3-5 優勝）
- 競品分析（多家廣告 tagline 的 pattern 抽取）

## 6 階段詳細定義

### 階段 1：テーマ設定 / ブレインストーミング

- **テーマ設定**：確立本次 KJ法 運作的焦點問題（例：「ミドル世代向け
  サプリの 訴求角度は何か」）。テーマが曖昧だと後続階段全部崩れる。
- **ブレインストーミング**（option）：如果素材不足，先以 BS 補齊。
  素材足（例：訪談逐字稿、Mandal-Art 81 格）則直接進入階段 2。

### 階段 2：ラベル作成（カード化）

- **規則**：**一卡一概念**。一張便箋 / 卡片上只寫一個 idea 或一段
  verbatim 引文。多概念混在一卡會破壞後續聚類的粒度。
- **表現**：寫得具體。抽象如「品質向上」不合格，具體如「アミノ酸配合
  量が競合より 2 倍」才合格。
- **產出規模**：50-200 卡典型。過少（<30）資料不足湧現、過多（>300）
  單人難以掌握需拆批次。

### 階段 3：グループ編成（聚類）

- **規則**：**感性親和性**で「似たもの同士」を近づける。**意味カテゴリで
  分類しない**（これが top-down 分類法との根本差）。
- **手順**：
  1. 全カードを机 / ホワイトボードに展開
  2. 2-3 枚を「なんとなく近い」理由で寄せる（**論理ではなく直感**）
  3. 小グループ（2-5 枚）を 10-30 個作る
  4. 小グループ同士を寄せて中グループ、中 → 大へ階層化
- **禁忌**：事先用 persona / 4P / SWOT 等既存框架で分類する。これは
  「渾沌をして語らしめる」に反する。

### 階段 4：表札づくり

- **規則**：各グループに「表札」（見出し）を付ける。表札はグループ内
  全カードの**共通の本質**を一行で言語化する。
- **質の基準**：
  - **悪い例**：「品質関連」（カテゴリ名、抽象）
  - **良い例**：「科学的根拠で選びたい合理派」（グループが語る物語）
- **階層**：小 → 中 → 大の各階層で表札を付ける。大表札が KJ法 最終
  洞察の骨子になる。

### 階段 5：図解化（A 型）

- **規則**：大グループを**空間配置**する。単に並べるのではなく、
  グループ間の**因果・対立・並列・包含**関係を矢印 / 記号で示す。
- **A 型の意味**：Analysis（分析）の A。本図が KJ法 の「見える化」成果物。
- **配置原則**：
  - 上下 → 抽象 / 具体 or 因 / 果
  - 左右 → 時系列 or 対立軸
  - 中心 → 核心テーマ、周辺 → 補助要素
- **LLM 不可**：空間言語は人類の空間認知に依存する。embedding 2D 配置は
  意味的近接を示すが、KJ法 A 型が要求する「因果 / 対立の意識的配置」は
  できない。

### 階段 6：文章化（B 型）

- **規則**：A 型図解を基に、**叙述文**で結論を書く。図解を見ていない
  読者も理解できる連続したテクストにする。
- **B 型の意味**：Breakdown / Building の B。
- **典型長さ**：A4 1-2 枚、400-800 字。
- **構造**：核心洞察 → 主要グループの語り → グループ間の関係 → 含意 /
  次のアクション。

## 狭義 vs 広義 KJ法

- **狭義 KJ法**：階段 2-5（ラベル → グループ編成 → 表札 → A 型図解）。
  資料整理の手法としての KJ法。
- **広義 KJ法**：階段 1-6（テーマ設定から文章化まで）。問題解決 /
  意思決定の総合手法としての KJ法。

copywriting-team が使うのは基本的に**広義 KJ法**。ただし訪談逐字稿の
初回整理など「まず構造を見える化する」目的の場合は狭義で止めても良い。

## 「渾沌をして語らしめる」原則

川喜田 1986 が定式化した KJ法 の根本哲学：

> データを既存の枠組みに押し込むのではなく、データ自身が持つ秩序が
> 湧き上がるのを待つ。研究者は触媒であり、分類者ではない。

運用上の帰結：
- **事前の仮説を固定しない**（仮説検証なら KJ法 ではなく別手法）
- **既存タクソノミーを持ち込まない**（persona / 4P / SWOT 等）
- **グループ編成で違和感が出たら解体して再配置する**（途中で構造が
  変わるのが正常、変わらない方が怪しい）
- **LLM を使う場合、embedding 類似度は「感性親和性の近似値」であって
  代替ではない**。embedding はカテゴリ的近さを過度に評価し、KJ法 が
  大事にする「異質の中の本質的類似」を見落とす。

## 自動化邊界表

| 階段 | LLM 可做 | 半自動（人類 checkpoint） | LLM 不可做 |
|---|---|---|---|
| 階段 1：テーマ設定 | — | ✓ LLM 提案、人類決定 | — |
| 階段 2：ラベル作成（逐字稿→カード） | ✓ 概念抽出 + 1 卡 1 概念分解 | — | — |
| 階段 3：グループ編成 | — | ✓ embedding 初期クラスタ + 人類再配置 | ✗ 感性親和性の最終判断 |
| 階段 4：表札づくり | — | ✓ LLM 候補 3-5 提案、人類選択 | — |
| 階段 5：A 型図解 | — | — | ✗ 空間配置は人類認知必須 |
| 階段 6：B 型文章化 | ✓ A 型 + 表札を入力に叙述化 | — | — |

**規範**：
- LLM を embedding 段階 3 に使う場合は「**初期クラスタの叩き台**」として
  のみ。人類が「これは近くない」「これは分けるべき」と再配置する
  checkpoint を必ず設ける。
- 階段 5 A 型図解を LLM に出力させない。どうしても自動化する場合は
  「テキスト形式の階層リスト」に留める（空間配置なし）。
- 階段 4 表札づくりで LLM 案をそのまま採用しない。LLM はカテゴリ名的な
  表札を出しがち（「品質関連」「価格関連」）で、グループの物語を
  捉えない。

## 應用於 copywriting

### 主要使用場面

- **消費者訪談整理** — 10-30 人の訪談逐字稿を KJ法 で整理 → ペルソナ /
  インサイト発見 → planning-team にフィードバック
- **文案角度收斂** — Mandal-Art 81 格 + Verbalized Sampling 5-10 候補 =
  約 100-150 候補を KJ法 で 3-5 優勝に絞る
- **競品 tagline 分析** — 競合 20-50 社のコピーを KJ法 でクラスタ化 →
  差別化ホワイトスペース発見
- **キャッチコピー 64 候補の収斂** — 谷山 2007「散らかす」で出した 64 本
  から優勝 3-5 本を選ぶ（→ `ideation-taniyama-discipline.md` 「選ぶ」
  階段と統合運用）

### 代表運作（catch copy 64 候補 → 3-5 優勝）

```
【Step 1】テーマ設定
  └── 「〇〇商品の catch copy 優勝を 3-5 本選ぶ」

【Step 2】ラベル作成
  └── 64 本を 64 枚のカードに展開
  └── 各カードに copy 本文 + 発想の角度（Mandal-Art 方向）を記載

【Step 3】グループ編成
  └── embedding で初期 10-15 クラスタ
  └── 人類が「近くない」と感じた 3-5 件を再配置
  └── 小 → 中 → 大 で階層化

【Step 4】表札づくり
  └── 各大グループに「この copy 群が共通に訴える本質」を一行化
  └── 例：「合理派を安心させる数字訴求群」「感性派を物語で引き込む群」

【Step 5】A 型図解
  └── 大グループを配置：X 軸 = 合理 / 感性、Y 軸 = 短文 / 長文
  └── 各群の強弱と空白帯を可視化

【Step 6】B 型文章化 + 優勝選定
  └── 「今回のターゲット（planning-team から受け取り）に最適な群は
       どれか」を叙述
  └── 選定群から 3-5 本を優勝として切り出す
  └── → ideation-taniyama-discipline.md 「磨く」階段へ
```

## Anti-Patterns

- **單步驟 LLM 一次做完**：prompt に「64 件を KJ法 で 5 件に絞って」と
  投げて LLM 独自に完結させる。これは「渾沌をして語らしめる」に反する
  トップダウン圧縮で、KJ法 の名を借りた多数決式要約に過ぎない。
- **embedding でグループ編成を代替する**：embedding cosine 類似度で
  完全自動クラスタを KJ法 と称する。embedding はカテゴリ的類似度で、
  感性親和性（「異質の中の本質的近さ」）を拾わない。必ず人類 checkpoint
  を挟む。
- **既存 framework でグループ編成する**：persona / 4P / AIDMA 段階を
  事前カテゴリに使う。これは川喜田が明確に禁じる「既存枠組みの押し
  込み」。KJ法 の意味を失う。
- **跳過人類 checkpoint**：embedding 初期クラスタ → そのまま表札 → 直接
  文章化、とパイプラインを通す。少なくとも段階 3 と段階 4 で人類（または
  evaluator agent）が「違和感はないか」を確認しないと、LLM 既存学習
  バイアスで意外な本質が潰される。
- **A 型図解を LLM に描かせる**：ASCII 配置や graphviz でも「人類の空間
  認知で因果 / 対立が読み取れる配置」にはならない。A 型は人類作業。
- **表札にカテゴリ名を使う**：「品質関連」「価格関連」など。これは
  グループの物語を言語化していない。表札は「このグループが何を語って
  いるか」の叙述でなければならない。
- **狭義と広義の取り違え**：階段 2-3 で止めて「KJ法 やりました」と報告
  する。それは狭義 KJ法 で、意思決定材料としては不完全。copywriting
  では通常階段 6 まで必要。
