# Mid-Form Copywriting — BEAF Canon

Tier 2 standard：商品ページ・店頭 POP・説明会資料など 数十〜数百字
帯の中文案を構築するための **BEAF (Benefit - Evidence - Advantage -
Feature)** framework を canon として定義する。BEAF は日本の EC 実務
（樂天市場 / Amazon JP）で広く普及している順序であり、canonical
author の特定可能な一次資料は二次資料では未確認のため、**「業界
通用の EC 実務 framework」** として位置づける。

## Primary Sources

- 樂天市場出店者向け教学資料 / Amazon JP セラーセントラル ヘルプ —
  商品ページの推奨構成 として Benefit-first 順序を指導する次級
  來源が多数存在する（複数の樂天 / Amazon 運用代行会社の公式
  ブログ記事が同順序で一致）。
- 日本 EC 実務ブログ（"BEAFの法則 商品ページ" で検索される一群の
  マーケティング会社記事）— 4 段階定義と順序について 3 社以上の
  獨立來源で一致が確認される。
- **Author attribution uncertainty**：BEAF の canonical author
  （誰が命名し体系化したか）の一次資料は二次資料では特定
  できていない。日本 EC 業界に定着した実務 framework として
  扱い、**「業界通用 framework」** と表記する。canonical
  textbook は未確定。
- FAB (Feature-Advantage-Benefit) との関係：FAB は米国セールス
  教育（Xerox 等）由来の古典的 3 段階で、順序は Feature → Advantage →
  Benefit。BEAF は **逆方向 + Evidence 挿入** の 4 段階であり、
  FAB の日本 EC 向け改良形として理解される。

## Framing — 中文案層の位置づけ

BEAF は **数十〜数百字** を射程とする中文案 framework であり、
下記の 2 つの特徴で PASONA 系列 (長文) / キャッチコピー (短文) と
差別化される。

- **機能説得型**：PASONA 系列が「共感→緊迫→行動」の心理 flow で
  誘導するのに対し、BEAF は「この商品はあなたに何をもたらすか →
  証拠 → 競合優位 → スペック」の 客観説得 flow。心理誘導ではなく
  機能説明に軸足を置く。
- **Benefit-first 順序**：読者がスキャン的に読む商品ページでは、
  冒頭で「自分にどんな良い変化が起きるか」を提示しないと読み
  続けられない。順序違反（Feature 先出し）は離脱を招く。

## BEAF の 4 段階定義

| 段階 | 名称 | 該段階の任務 | 典型的文体 |
|---|---|---|---|
| **B** | **Benefit (利益)** | 読者の生活がどう良くなるか、購入後の具体的変化を 1 文目で描く。商品を買うことで実現する「成った姿」を示し、読者の目を留める段。 | 「〜になれる」「〜から解放される」 |
| **E** | **Evidence (証拠)** | Benefit の裏付けとなる客観的データ・受賞歴・第三者評価・顧客レビュー・検査結果を提示する。Benefit への疑念を先回りで解消。 | 「〇〇賞受賞」「レビュー 4.8/5.0」「臨床試験で〜」 |
| **A** | **Advantage (優位)** | 競合商品と比較した独自の強み。同カテゴリの他商品ではなく「なぜこの商品か」の理由付け。機能比較表や独自技術の紹介。 | 「他社比 2 倍の〜」「業界唯一の〜」 |
| **F** | **Feature (特徴)** | 規格・成分・サイズ・カラー・素材・同梱物など客観的基本情報。スペック一覧の役割。 | 「容量 500ml / 原材料：〜 / 重さ：〇g」 |

**順序の canonical 性**：Benefit → Evidence → Advantage → Feature
の順序は、下記 3 つの理由で逆順 (F→A→E→B) に勝る。

- **スキャン読みへの最適化**：商品ページは読者が流し読みする媒体。
  冒頭で Benefit を示さないとスクロールが止まらない。
- **興味 → 納得 → 差別化 → 確認 の認知 flow**：読者は「魅力 → 本当か
  → 他と何が違う → 具体的には何か」の順で疑問を抱く。BEAF はこの
  順序に一致する。
- **購買決定の心理的重量配分**：Feature は購入決定後の確認材料で
  あり、決定前の説得力は弱い。Benefit が決定を促す。

## Benefit-first 順序の重要性

BEAF の最大の canonical 主張は **「先に糖を与え、後に証明を与える」**
順序である。商品ページで Feature を冒頭に置く設計は以下の 2 点で
失敗しやすい。

- **読者は Feature の意味を自力で Benefit に翻訳しない**。「500ml」
  と書かれても「だから朝の水分補給 1 日分」と読者は連想しない。
  書き手側で Feature → Benefit 翻訳を済ませて提示する必要がある。
- **Benefit 不在の商品ページは「広告らしくない広告」になる**。
  スペック羅列は中立的で「売る気がない」ページに見え、心理的
  に購入モチベーションを形成しない。

## 字數範囲と適用用例

**字數**：数十字（店頭 POP）〜 数百字（商品ページ本文）。
1,000 字を超える場合は新 PASONA / PASBECONA を検討すべき境界。

**典型用例**：

- 樂天市場 / Yahoo!ショッピング / Amazon JP の商品ページ本文
- 店頭 POP（スーパー・家電量販店・書店）
- 説明会 / 展示会 のブース配布資料
- B2B 製品カタログの 1 製品 1 ページ説明
- EC サイトの商品一覧ページでのサマリー文

## PASONA 系列との分界

BEAF と PASONA 系列は **説得アプローチの根本が異なる**。

| 軸 | BEAF | PASONA 系列 |
|---|---|---|
| 説得タイプ | 機能説得 (客観) | 心理誘導 (主観) |
| 起点 | Benefit (結果像) | Problem (問題認識) |
| 緊迫感 | 不使用 | Narrow down で明示 |
| 共感 | 不使用 | Affinity (新/PASBECONA) |
| Action | 明示 CTA 不要（購入ボタン隣接が自然） | 段階として明示必須 |
| 字數帯 | 数十〜数百字 | 1,500〜20,000+ 字 |
| 媒体 | 商品ページ / POP | LP / メルマガ / セールスレター |

**選定基準**：

- 商品ページ・POP・カタログ 一製品説明 → BEAF
- LP・メルマガ・セールスレター → PASONA 系列
- 商品ページ内で「追加の物語」を語りたい → BEAF を主軸に
  Problem + Affinity の小節を挿入する hybrid 構成も可

## FAB との差異（業界常見の混同）

**FAB (Feature - Advantage - Benefit)** は米国セールス教育由来の
古典的 3 段階で、BEAF と混同されやすい。正確な差異は以下。

| 軸 | FAB | BEAF |
|---|---|---|
| 段階數 | 3 | 4 |
| 順序 | Feature → Advantage → Benefit | Benefit → Evidence → Advantage → Feature |
| Evidence | なし | あり (第 2 段) |
| 由来 | 米国セールス教育 (Xerox 等) | 日本 EC 実務 |
| 起点の思想 | 商品起点 (Feature から説明) | 顧客起点 (Benefit から描く) |

**canonical 注意**：BEAF を「FAB の順序」で説明する次級來源は
誤りであり、採用しない。BEAF は **FAB の逆順 + Evidence 追加**
であり、両者を入れ替えて覚えてはいけない。

## Anti-Patterns

- **Feature から書き始める**。BEAF の canonical 順序違反。スペック
  羅列が冒頭に来ると読者はスクロールを止めない。
- **Evidence 段を省略する**。Benefit だけでは「広告の誇張」に
  見える。客観的裏付け (受賞・レビュー・数値) が不可欠。
- **BEAF を FAB の順序で実装する**。F → A → B と F → A → E → B は
  別物。Evidence の位置と Benefit の位置を入れ替えると canonical
  違反。
- **Advantage を主観的形容詞で埋める**。「高品質」「最高」などの
  形容詞は Advantage ではなく、競合比較の具体的数値 / 技術 /
  認証が Advantage。
- **Benefit を商品スペックの言い換えで書く**。「500ml 入っている
  ので便利」は Feature の翻訳に過ぎない。「通勤バッグに入れて
  1 日分の水分が賄える」まで具体化して初めて Benefit。
- **BEAF に CTA を必須段として追加する**。BEAF 自体には Action
  段がなく、商品ページの購入ボタンが Action を兼ねる。無理に
  CTA 段を追加すると PASONA との境界が曖昧になる。
